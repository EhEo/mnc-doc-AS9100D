# 1. 목적

본 절차서는 Công ty TNHH M&C ELECTRONICS VIỆT NAM (이하 M&C Electronics VINA)의 품질경영시스템(QMS), 프로세스, 제품 및 서비스의 적합성, 충족성, 효과성을 지속적으로 개선하기 위한 체계와 절차를 규정한다.

본 절차서의 목적:
- AS9100D 10.3 요구사항에 따른 지속적 개선(Continual Improvement) 체계 수립
- PDCA(Plan-Do-Check-Act) 사이클을 기반으로 한 체계적 개선 프로세스 운영
- 데이터 분석, 내부심사, 경영검토, 고객 피드백 등 다양한 원천으로부터의 개선 기회 식별
- 개선 프로젝트의 체계적 관리 및 효과성 측정
- 항공우주 산업의 프로세스 성과 개선 의무 및 모범사례 벤치마킹 이행

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 지속적 개선 (Continual Improvement) | 성과를 향상시키기 위한 반복적 활동 (ISO 9000:2015, 3.3.2). 한 번의 도약이 아닌 지속적이고 점진적인 개선을 포함 |
| PDCA | Plan(계획)-Do(실행)-Check(확인)-Act(조치) 사이클. 모든 개선 활동의 기본 프레임워크 |
| Kaizen | 일본어 '개선(改善)'. 모든 구성원이 참여하는 점진적·지속적 개선 활동 |
| DMAIC | Define(정의)-Measure(측정)-Analyze(분석)-Improve(개선)-Control(관리). Six Sigma 개선 방법론 |
| Lesson Learned | 교훈 사항 — 프로젝트, 문제 해결, 개선 활동 등에서 얻은 경험과 지식을 체계적으로 기록한 것 |
| 개선 기회 (Opportunity for Improvement) | 현재 성과 수준과 바람직한 성과 수준 사이의 격차(Gap)에서 도출되는 개선 여지 |
| KPI (Key Performance Indicator) | 핵심성과지표 — 프로세스 및 시스템 성과를 측정하는 정량적 지표 |
| 벤치마킹 (Benchmarking) | 업계 최우수 사례(Best Practice)와 자사 프로세스를 비교·분석하여 개선 기회를 도출하는 방법 |
| QMS 성숙도 (QMS Maturity) | 품질경영시스템의 발전 단계를 평가하는 수준 — 반응적(Reactive) → 체계적(Systematic) → 선제적(Proactive) → 혁신적(Innovative) |
| CIP (Continual Improvement Plan) | 지속적 개선 계획서 — 개선 목표, 과제, 일정, 자원, 성과 지표를 포함하는 연간/반기 계획 |

# 6. 지속적 개선 체계 (PDCA 사이클)

## 6.1 PDCA 사이클 개요

M&C Electronics VINA의 모든 지속적 개선 활동은 PDCA 사이클을 기본 프레임워크로 운영한다.

```
┌─────────────────────────────────────────────────────┐
│                    PDCA 사이클                        │
│                                                      │
│   ┌──────────┐    ┌──────────┐                       │
│   │  PLAN    │───→│   DO     │                       │
│   │ (계획)   │    │  (실행)  │                       │
│   └──────────┘    └──────────┘                       │
│        ↑               │                             │
│        │               ↓                             │
│   ┌──────────┐    ┌──────────┐                       │
│   │   ACT    │←───│  CHECK   │                       │
│   │  (조치)  │    │  (확인)  │                       │
│   └──────────┘    └──────────┘                       │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## 6.2 단계별 활동

### 6.2.1 PLAN (계획)

| 활동 | 내용 |
|------|------|
| 현상 파악 | 현재 프로세스 성과 데이터 수집 및 분석 |
| 목표 설정 | 개선 목표 및 달성 기준 설정 (SMART 원칙) |
| 원인 분석 | 성과 격차의 근본 원인 분석 (5 Why, 특성요인도, FMEA 등) |
| 대책 수립 | 개선 방안 도출, 실행 계획 수립 (담당자, 일정, 자원) |
| 리스크 평가 | 개선 실행에 따른 리스크 식별 및 완화 방안 (QP-601 연계) |

### 6.2.2 DO (실행)

| 활동 | 내용 |
|------|------|
| 시범 실행 | 소규모 시범(Pilot) 적용 (해당 시) |
| 본 실행 | 계획에 따른 개선 조치 이행 |
| 데이터 수집 | 실행 과정의 데이터 기록 |
| 교육/소통 | 관련 인원 교육 및 변경 사항 공유 |

### 6.2.3 CHECK (확인)

| 활동 | 내용 |
|------|------|
| 결과 측정 | 개선 전/후 성과 데이터 비교 |
| 목표 대비 평가 | 설정 목표 달성 여부 확인 |
| 부작용 점검 | 개선에 따른 의도치 않은 부정적 영향 유무 확인 |
| 효과성 판정 | 개선 효과의 지속 가능성 및 유효성 판정 |

### 6.2.4 ACT (조치)

| 활동 | 내용 |
|------|------|
| 표준화 | 효과 확인된 개선 사항을 절차/표준에 반영 |
| 수평 전개 | 유사 프로세스/제품으로 개선 사항 확대 적용 |
| Lesson Learned 기록 | 개선 활동에서 얻은 교훈 체계적 기록 |
| 잔여 과제 식별 | 미진한 부분에 대한 추가 개선 과제 등록 |
| 차기 PDCA 연결 | 다음 개선 사이클로 연결 |

| 원천 | 내용 | 담당 | 주기 |

| 벤치마킹 | 업계 모범사례 비교 분석 | 품질팀 | 연간 |

## 7.2 개선 기회 평가 및 우선순위

수집된 개선 기회는 다음 기준에 따라 평가하고 우선순위를 결정한다:

| 평가 기준 | 배점 (1~5) | 설명 |
|----------|-----------|------|
| 고객 영향도 | 5 = 매우 높음 | 고객 만족, 클레임 감소에 미치는 영향 |
| 품질 영향도 | 5 = 매우 높음 | 제품/프로세스 품질 향상 기대 효과 |
| 실현 가능성 | 5 = 매우 쉬움 | 기술적 난이도, 자원 소요, 구현 복잡도 |
| 비용 대비 효과 | 5 = 매우 높음 | 투입 비용 대비 기대 효과 |
| 긴급성 | 5 = 매우 긴급 | 조기 개선 필요성 |

**우선순위 결정**: 총점 = Σ(각 기준 배점 × 가중치). 총점 기준 High / Medium / Low 분류.

# 8. 개선 프로젝트 관리

## 8.1 개선 과제 등록

식별되고 우선순위가 결정된 개선 기회는 **개선 과제 대장**에 등록하여 관리한다.

| 등록 항목 | 내용 |
|----------|------|
| 과제 번호 | CI-YYYY-NNN (Continual Improvement-연도-일련번호) |
| 과제명 | 개선 과제의 구체적 명칭 |
| 원천 | 개선 기회 식별 출처 (심사/클레임/데이터 분석 등) |
| 개선 대상 | 대상 프로세스, 제품, 시스템 |
| 현재 수준 | 현행 성과 데이터 (개선 전 기준) |
| 목표 수준 | 개선 목표 (정량적 수치) |
| 개선 방법론 | PDCA, DMAIC, Kaizen, 8D 등 |
| 담당자/팀 | 개선 과제 책임자 및 참여 팀원 |
| 목표 완료일 | 개선 완료 목표 일자 |
| 상태 | 등록 / 진행 중 / 확인 중 / 완료 / 종결 |

## 8.2 개선 방법론 선택

| 개선 유형 | 권장 방법론 | 적용 조건 |
|----------|-----------|----------|
| 단순 개선 | PDCA | 단일 프로세스, 단기 개선 (1~2개월) |
| 현장 즉시 개선 | Kaizen | 작업자 수준의 즉시 실행 가능 개선 |
| 복합 문제 해결 | DMAIC (6 Sigma) | 다수 변수, 통계적 분석 필요, 중장기 (3~6개월) |
| 부적합 재발 방지 | 8D | 고객 클레임, 반복 부적합 (QP-1001 연계) |
| 시스템 구조 개선 | 프로세스 재설계 | QMS 프로세스 구조적 변경 |

## 8.3 개선 과제 실행 및 추적

### 8.3.1 실행 관리

1. **개선 계획서 작성**: 과제별 상세 실행 계획 수립 (목표, 일정, 담당자, 방법)
2. **킥오프 (Kick-off)**: 관련 팀원 대상 과제 목표 및 역할 공유
3. **정기 진척 확인**: 월 1회 이상 진행 상황 점검
4. **중간 보고**: 장기 과제(3개월 이상)의 경우 중간 보고 실시
5. **완료 보고**: 개선 완료 후 결과 보고서 작성

### 8.3.2 추적 관리

- 개선 과제 대장을 월간 업데이트
- 지연 과제에 대해 원인 분석 및 일정 재조정
- 분기별 과제 현황을 경영진에 보고
- 과제 상태: 등록 → 진행 중 → 확인 중 → 완료 → 종결(효과성 확인 후)

## 8.4 Kaizen 제안 제도

### 8.4.1 제안 프로세스

1. **제안 제출**: 전 임직원이 개선 제안서(QF-1002-03)를 작성하여 부서장에게 제출
2. **부서 검토**: 부서장이 실현 가능성 및 효과를 예비 검토
3. **채택 판정**: 품질팀에서 제안 평가 기준에 따라 채택/미채택 판정
4. **실행**: 채택된 제안을 개선 과제로 등록하여 실행
5. **보상**: 채택·실행된 제안에 대해 포상 (사내 포상 규정에 따름)

### 8.4.2 제안 평가 기준

| 평가 항목 | 배점 |
|----------|------|
| 품질/안전 효과 | 30점 |
| 생산성/효율 효과 | 25점 |
| 원가 절감 효과 | 20점 |
| 실현 가능성 | 15점 |
| 창의성/독창성 | 10점 |
| **합계** | **100점** |

- 채택 기준: 60점 이상
- 우수 제안: 80점 이상 (특별 포상)

# 9. 항공우주 특수 요구사항

## 9.1 프로세스 성과 개선 의무

AS9100D 10.3은 조직이 품질경영시스템의 적합성, 충족성 및 효과성을 지속적으로 개선할 것을 요구한다. 항공우주 산업의 특수 요구사항:

| 요구사항 | 이행 방법 |
|---------|----------|
| 제품 적합성 개선 | DPPM 감소 목표 설정 및 추적, 공정 능력(Cpk) 향상 |
| 납기 성과 개선 | OTD 목표 달성, 리드 타임 단축, 납기 예측 정확도 향상 |
| QMS 효과성 개선 | 내부심사 NCR 감소, 시정조치 유효성 향상 |
| 고객만족 개선 | CSI 향상, 고객 클레임 감소, 스코어카드 등급 향상 |
| 공급망 성과 개선 | 외부공급자 품질/납기 성과 향상 |

## 9.2 업계 모범사례 벤치마킹

### 9.2.1 벤치마킹 대상

| 영역 | 벤치마킹 대상 | 방법 |
|------|-------------|------|
| 품질 성과 | 동종 항공우주 부품 제조업체 | 업계 보고서, 고객 정보, IAQG 데이터 |
| 공정 기술 | 업계 선진 제조 기술 (Lean, Industry 4.0) | 기술 세미나, 전시회, 논문 |
| 품질 시스템 | AS9100D 인증 우수 기업 사례 | 인증기관 정보, 컨설팅, 업계 교류 |
| 안전 관리 | 항공우주 안전 관리 모범사례 | FAA/EASA 가이던스, 업계 기준 |

### 9.2.2 벤치마킹 프로세스

1. **대상 선정**: 벤치마킹 대상 영역 및 비교 대상 결정
2. **데이터 수집**: 자사 현행 수준 및 벤치마킹 대상 성과 데이터 수집
3. **격차 분석 (Gap Analysis)**: 자사 성과와 모범사례 간 격차 분석
4. **개선 과제 도출**: 격차 해소를 위한 개선 과제 등록
5. **실행 및 추적**: 개선 과제 실행 후 격차 축소 확인
6. **보고**: 벤치마킹 결과를 경영검토 입력 자료로 활용

## 9.3 QMS 성숙도 평가

### 9.3.1 성숙도 수준 정의

| 수준 | 명칭 | 특징 |
|------|------|------|
| Level 1 | 반응적 (Reactive) | 문제 발생 후 대응, 체계적 개선 미흡 |
| Level 2 | 체계적 (Systematic) | 프로세스 기반 관리, PDCA 운영 |
| Level 3 | 선제적 (Proactive) | 리스크 기반 사전 예방, 데이터 기반 의사결정 |
| Level 4 | 혁신적 (Innovative) | 지속적 혁신, 업계 선도, Best-in-Class |

### 9.3.2 성숙도 평가 방법

- **평가 주기**: 연 1회 (경영검토 전 실시)
- **평가 기준**: ISO 9004:2018 부속서 A '자기 평가 도구' 활용
- **평가 영역**: 리더십, 전략 기획, 지원 관리, 운용 관리, 성과 평가, 개선
- **평가 결과**: 영역별 성숙도 수준 및 종합 성숙도 수준 산출
- **목표**: 매년 최소 1개 영역 이상 성숙도 수준 향상

# 10. 효과성 측정

## 10.1 개선 효과 측정 방법

| 측정 항목 | 방법 | 기준 |
|----------|------|------|
| 정량적 효과 | 개선 전/후 KPI 비교 (DPPM, OTD, Cpk, 수율 등) | 목표 달성 여부 |
| 정성적 효과 | 프로세스 개선 정도, 업무 효율성, 직원 만족도 | 관련자 평가 |
| 재무적 효과 | 불량 비용 감소, 생산성 향상, 원가 절감 | 금액 환산 |
| 지속 가능성 | 개선 효과 유지 기간 (3개월, 6개월, 12개월 추적) | 효과 지속 확인 |

## 10.2 개선 체계 성과 지표 (KPI)

| KPI | 산출 방법 | 목표 |
|-----|----------|------|
| 개선 과제 완료율 | (완료 과제 수 ÷ 등록 과제 수) × 100% | ≥ 80% (연간) |
| 개선 과제 목표 달성률 | (목표 달성 과제 ÷ 완료 과제) × 100% | ≥ 70% |
| Kaizen 제안 건수 | 연간 전 임직원 제안 건수 | ≥ 인원 수 × 1건/년 |
| Kaizen 채택률 | (채택 건수 ÷ 제안 건수) × 100% | ≥ 50% |
| 개선 효과 금액 | 연간 개선 활동의 재무적 효과 합계 | 전년 대비 증가 |
| QMS 성숙도 | 성숙도 평가 종합 점수 | 전년 대비 향상 |

## 10.3 효과성 미달 시 조치

- 개선 과제가 목표 미달인 경우: 원인 분석 후 추가 조치 또는 과제 재기획
- 개선 효과가 지속되지 않는 경우: 표준화 수준 재점검, 교육 강화, 모니터링 강화
- 개선 체계 KPI가 목표 미달인 경우: 경영검토에서 원인 분석 및 대책 결정

# 11. 경영검토 보고

## 11.1 보고 내용 (QP-904 연계)

경영검토 시 다음 지속적 개선 데이터를 입력 자료로 제출한다:

| 보고 항목 | 내용 | 주기 |
|----------|------|------|
| 개선 과제 현황 | 등록/진행/완료/종결 건수, 완료율, 목표 달성률 | 반기 |
| 주요 개선 성과 | 핵심 개선 과제의 정량적/정성적 성과 | 반기 |
| Kaizen 활동 현황 | 제안 건수, 채택률, 실행 현황, 포상 현황 | 반기 |
| QMS 성숙도 평가 결과 | 영역별 성숙도 수준, 전년 대비 변화 | 연간 |
| 벤치마킹 결과 | 격차 분석 결과, 개선 과제 도출 현황 | 연간 |
| 재무적 효과 | 개선 활동 총 비용 절감/생산성 향상 금액 | 연간 |
| 차기 CIP (개선 계획) | 차기 연간/반기 개선 계획(안) | 연간 |

## 11.2 경영검토 출력 반영

경영검토에서 결정된 지속적 개선 관련 사항:
- 연간 개선 방향 및 중점 개선 영역 결정
- 개선 활동 자원(인력, 예산, 설비) 배분 승인
- 품질목표 재설정 시 개선 목표 연계 (QP-602)
- 조직 역량 강화 방안 (교육 계획 연계, QP-703)
- QMS 성숙도 목표 수준 설정
