# 1. 목적

본 절차서는 Công ty TNHH M&C ELECTRONICS VIỆT NAM (이하 M&C Electronics VINA)의 품질경영시스템(QMS) 변경이 필요한 경우, 변경을 체계적으로 기획하고 관리하기 위한 절차를 규정한다.

본 절차서의 목적:

- AS9100D 6.3 요구사항에 따른 QMS 변경의 체계적 기획 및 통제
- 기존 4M-1I-1E(Man, Machine, Material, Method, Information, Environment) 변경관리 체계의 항공우주 규격 확장
- 변경의 목적, 잠재적 결과, QMS 무결성, 자원 가용성, 책임/권한 재배분을 사전 평가
- 일시적 변경과 영구적 변경의 구분 관리 및 추적성 확보
- 고객 사전 승인 및 형상관리(QP-803)와의 연계를 통한 항공우주 제품 안전성 보장
- 변경 후 첫 3로트 집중 관리를 통한 변경 유효성 검증

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| **4M-1I-1E** | Man(인원), Machine(설비/장비), Material(자재/원부자재), Method(방법/공정), Information(정보/시스템), Environment(환경) — 변경 분류 프레임워크. R&D.02.09.04 Rev.11에서 규정한 6가지 변경 범주 |
| **일시적 변경 (Temporary Change)** | 정해진 기간 또는 조건 내에서만 유효한 한시적 변경. 원래 상태로 복귀가 전제됨 (예: 대체 인력, 임시 설비 사용) |
| **영구적 변경 (Permanent/Long-term Change)** | 변경 후 원래 상태로 복귀하지 않는 항구적 변경. QMS 문서 개정이 수반됨 (예: 설비 교체, 공정 재설계) |
| **ECR (Engineering Change Request)** | 설계변경요청서. 변경의 필요성, 내용, 영향 평가를 포함한 공식 변경 요청 |
| **ECN (Engineering Change Notice)** | 설계변경통보서. 승인된 변경 내용을 관련 부서에 공식 통보하는 문서 |
| **SQM (Supplier Quality Manual)** | 고객사 공급업체 품질 매뉴얼. 변경 사전 승인 요건, 제출 서류 등을 규정 |
| **CCB (Configuration Control Board)** | 형상통제위원회. QP-803에 의거하여 형상 변경을 심의·승인하는 위원회 |
| **PPAP (Production Part Approval Process)** | 양산부품승인절차. 변경 후 고객 승인을 위한 제출 서류 패키지 |
| **FAI (First Article Inspection)** | 초도품 검사. AS9102에 따른 최초 생산품 검증 (변경 시 재실시 판단 필요) |
| **4M Survey (4M 조사)** | 변경 실시 전 내부 관련 부서를 대상으로 한 사전 조사. 변경 영향 및 준비 상태 확인 |
| **변경 평가 회의** | 변경 실시 전 관련 부서장이 참석하여 변경 내용, 리스크, 일정, 자원을 심의하는 회의 |
| **HSF (Hazardous Substance Free)** | 유해물질 미사용. 환경 규제 물질 관련 변경 시 MSDS 확인 필수 |

## 6.2 변경 요청 (Step 1)

### 6.2.1 변경 요청 발의

변경 요청은 다음 사유에 의해 발의될 수 있다:

| 발의 사유 | 예시 |
|----------|------|
| 고객 요구 | 사양 변경, 규격 개정, 납기 변경 |
| 내부 개선 | 생산성 향상, 원가 절감, 불량률 감소 |
| 규제/규격 변경 | AS9100D 요건 변경, 환경법규 변경, HSF/MSDS 관련 |
| 부적합 시정 | 반복 부적합 대응, CAR 시정조치에 의한 변경 |
| 공급망 변동 | 공급업체 폐업, 원자재 단종, 대체 자재 필요 |
| 설비/인프라 | 노후 설비 교체, 신규 설비 도입, 레이아웃 변경 |

### 6.2.2 변경 요청서 작성

변경 요청자는 다음 사항을 포함하여 변경 요청서(ECR 또는 4M 변경신청서)를 작성한다:

1. **변경 목적 및 사유**: 변경이 필요한 배경과 기대 효과
2. **변경 내용**: 현재 상태(As-Is)와 변경 후 상태(To-Be)의 구체적 기술
3. **4M-1I-1E 분류**: 해당 변경이 속하는 범주 식별
4. **영향 범위**: 영향받는 제품, 프로세스, 문서, 부서
5. **일시적/영구적 구분**: 변경의 지속 기간 및 복귀 조건 (일시적인 경우)
6. **리스크 사전 평가**: 변경에 따른 잠재적 부정적 영향 식별

## 6.3 4M-1I-1E 분류 및 등급 판정 (Step 2)

### 6.3.1 변경 분류

R&D.02.09.04 Rev.11의 분류 체계에 따라 변경을 다음과 같이 분류한다:

| 분류 | 변경 대상 | 세부 항목 |
|------|----------|----------|
| **Man (인원)** | 작업자, 검사원, 관리자 | 신규 배치, 교대, 전환 배치, 퇴사 대체 |
| **Machine (설비)** | 생산설비, 금형, 치공구, 측정장비 | 신규 도입, 교체, 이설, 개조, 오버홀 |
| **Material (자재)** | 원자재, 부자재, 포장재 | 사양 변경, 공급업체 변경, 대체 자재 |
| **Method (방법)** | 공정, 작업표준, 검사방법 | 공정 순서 변경, 조건 변경, 신규 공법 |
| **Information (정보)** | 도면, 시스템, 데이터 | 도면 개정, ERP 변경, 검사 기준 변경 |
| **Environment (환경)** | 작업환경, 보관환경 | 온습도, 청정도, 조명, 정전기 방지 |

### 6.3.2 변경 등급 판정

| 등급 | 기준 | 승인 권한 | 의사소통 방법 |
|------|------|----------|-------------|
| **A등급 (중대변경)** | 제품 사양 영향, 고객 승인 필요, 형상 변경 수반, 안전 관련 | QMR 승인 + CCB 심의 + 고객 승인 | 회의 (Họp) - 관련 부서장 전원 참석 |
| **B등급 (일반변경)** | 공정 조건 변경, 설비 교체, 자재 동등급 대체 | 부서장 승인 + 품질팀 검토 | 게시판(Notice Board) + 이메일 통보 |
| **C등급 (경미변경)** | 문서 오기 수정, 관리적 변경, QMS에 실질적 영향 없음 | 해당 부서장 승인 | 이메일 통보 |

## 6.4 고객 SQM 확인 및 사전 승인 (Step 3)

### 6.4.1 고객 SQM 확인

변경 실시 전, 개발부서(RD)는 해당 고객의 SQM(Supplier Quality Manual)을 확인하여 다음을 검토한다:

- 사전 승인이 필요한 변경 항목의 범위
- 제출 서류 및 양식 요건
- 사전 통보 기간 요건 (최소 기간)
- PPAP 제출 요건 및 레벨

### 6.4.2 고객 사전 통보 및 승인

A등급 변경 및 고객 SQM에서 요구하는 변경에 대해:

1. **개발부서(RD)**: 변경 내용 및 실행 계획을 작성
2. **영업팀**: 고객에게 변경 내용 및 계획을 **최소 3개월 전**에 제출
3. **고객 승인 접수**: 고객의 서면 승인을 접수한 후에만 변경 실행 가능
4. **PPAP 제출**: 고객 요구 시 해당 레벨의 PPAP 서류 제출

> **주의**: 고객 사전 승인 없이 변경을 실행하는 것은 엄격히 금지한다. 긴급 변경의 경우에도 고객과 사전 협의하고, 최소한 구두 승인 후 서면 승인을 추후 획득한다.

## 6.5 내부 4M Survey 실시 (Step 4)

R&D.02.09.04 Rev.11에 따라, 변경 실행 전 내부 관련 부서를 대상으로 4M Survey를 실시한다.

- 변경 스탬프 적용 기간 및 방법

- 일시적 변경의 경우, 복귀 일자 및 조건을 4M 이력표에 명시한다.

| 판정 | 조건 | 후속 조치 |
|------|------|----------|
| **유효** | 첫 3로트 품질데이터 양호, 고객 클레임 없음, Cpk 충족 | 변경 스탬프 제거, 통상 관리 전환, 변경 마감 |
| **조건부 유효** | 일부 항목 미달이나 경향 개선 중 | 모니터링 연장 (추가 3로트), 원인 분석 |
| **무효** | 품질 악화, 고객 클레임 발생, 부적합 증가 | 변경 원복(Rollback), 원인 분석, 시정조치(CAR 발행) |

## 8.3 일시적 변경의 종료 관리

일시적 변경의 경우 추가로 다음을 관리한다:

- 복귀 예정일 도래 시 원상 복귀 실행 및 확인
- 일시적 변경의 영구화 필요 시 영구적 변경 절차 재수행
- 일시적 변경 기간 초과 시 QMR 보고 및 재승인
- 복귀 후 정상 상태 확인을 위한 품질 검증 실시

# 9. 항공우주 특수 요구사항

## 9.1 고객 사전 승인 (Customer Prior Approval)

AS9100D 및 항공우주 고객의 요구에 따라 다음 변경은 고객 사전 승인이 필수이다:

| 항목 | 통보 시점 | 제출 자료 |
|------|----------|----------|
| 제품 설계 변경 | 실행 **3개월 전** | ECR, 영향 분석, 검증 계획 |
| 제조 공정 변경 | 실행 **3개월 전** | 공정 변경 계획, 리스크 평가, 검증 방법 |
| 공급업체 변경 | 실행 **3개월 전** | 신규 업체 인증 현황, 평가 결과 |
| 생산지/설비 변경 | 실행 **3개월 전** | 이전 계획, 재검증 계획, FAI 계획 |
| 특수공정 변경 | 실행 **3개월 전** | 재인증 계획, 검증 데이터 |

> **중요**: 개발부서(RD)가 변경 내용과 실행 계획을 작성하고, 영업팀을 통해 고객에게 3개월 전에 제출한다. 고객의 서면 승인 없이는 변경을 실행할 수 없다.

## 9.2 형상관리 연계 (QP-803)

변경이 형상 항목(Configuration Item)에 영향을 미치는 경우:

1. QP-803에 따라 형상통제위원회(CCB) 심의를 거친다.
2. 형상 기준선(Baseline)의 변경 여부를 판단한다.
3. 형상 상태 기록(Configuration Status Accounting)을 업데이트한다.
4. ECN 발행 및 관련 형상 문서(도면, BOM, 사양서) 개정을 수행한다.

## 9.3 FAI 재실시 판단

AS9102에 따라 다음 변경 발생 시 FAI 재실시 여부를 판단한다:

| 변경 유형 | FAI 재실시 기준 |
|----------|---------------|
| 제품 설계 변경 (치수/재질) | **Full FAI** 재실시 |
| 금형 변경 (신작/주요 수정) | **Full FAI** 재실시 |
| 공정 순서 변경 | **Partial FAI** (영향 특성 한정) |
| 생산지/설비 변경 | **Full FAI** 재실시 |
| 공정 조건 변경 (범위 내) | FAI 면제 가능 (품질팀 판단) |
| 작업자 변경 | FAI 면제 (기량 인정으로 대체) |

## 9.4 금지물질/유해물질 변경 관리

자재 관련 변경 시 금지물질(HSF) 관리 절차:

1. 변경 자재의 **MSDS (Material Safety Data Sheet)** 확인
2. HSF 기준 적합 여부 검증 (RoHS, REACH, 고객 금지물질 목록 대조)
3. 부적합 시 변경 불가 판정 및 대체 자재 탐색
4. 적합 시 HSF 인증서/시험성적서를 변경 기록에 첨부

## 9.5 리스크 기반 변경 평가

AS9100D의 리스크 기반 사고에 따라 모든 A등급 변경에 대해:

- QP-802 FMEA 업데이트 필요 여부 검토
- 제품 안전성(QP-804) 영향 평가
- 운영 리스크(QP-802 8.1.2) 재평가
- 변경 실패 시 복구 계획(Contingency Plan) 수립
