# 1. 목적

본 절차서는 M&C Electronics VINA의 품질경영시스템(QMS) 운용에 필요한 문서화된 정보의 작성, 갱신, 관리 및 폐기에 관한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 7.5 요구사항에 따른 문서화된 정보의 체계적 관리
- 문서의 적합성, 충족성 및 최신성 보장
- 문서의 가용성, 보호 및 무결성 확보
- 항공우주 산업 규제당국 및 고객 요구사항 충족

---

# 2. 적용 범위

본 절차서는 M&C Electronics VINA의 QMS에 관련된 모든 문서화된 정보에 적용한다.

| 구분 | 적용 대상 |
|------|----------|
| 내부 문서 | 품질매뉴얼, 절차서, 작업지침서, 양식/기록 |
| 외부 문서 | 규격, 표준, 고객 도면/사양, 법적 요구사항, 공급자 인증서 |
| 전자 문서 | DMS(문서관리시스템)에 등록된 전자 파일 |
| 하드카피 | 인쇄 배포된 관리본 및 비관리본 |
| 기록 | QMS 활동의 결과를 포함하는 모든 문서화된 정보 |

**제외**: 개인 업무 메모, 임시 작업 노트 (단, 검사/품질 판정에 영향을 미치는 메모는 기록으로 관리)

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 문서화된 정보 (Documented Information) | 조직이 관리하고 유지해야 하는 정보 및 그것을 포함하는 매체. "유지"는 문서를, "보유"는 기록을 의미 |
| 문서 (Document) | QMS 운용에 필요한 정보를 기술한 것 (매뉴얼, 절차서, 지침서 등). "유지"해야 하는 문서화된 정보 |
| 기록 (Record) | 달성된 결과를 기술하거나 수행된 활동의 증거를 제공하는 문서. "보유"해야 하는 문서화된 정보 |
| 관리본 (Controlled Copy) | 개정 시 회수/갱신이 관리되는 배포본. 적색 "관리본" 스탬프 날인 |
| 비관리본 (Uncontrolled Copy) | 참조 목적으로 배포되며 개정 관리 대상이 아닌 사본. 청색 "비관리본" 스탬프 날인 |
| 폐기 문서 (Obsolete Document) | 현행 개정판으로 대체된 이전 문서. "폐기" 스탬프 날인 후 별도 보관 또는 파기 |
| DMS | 문서관리시스템 (Document Management System) |
| ECN | 설계변경통지서 (Engineering Change Notice) — 형상관리 문서 변경 시 사용 |
| DCR | 문서변경요청서 (Document Change Request) |
| 문서 마스터 | 원본 파일이 보관된 DMS 상의 최신 승인본 |

# 5. 책임과 권한

| 역할 | 책임 |
|------|------|
| **대표이사** | 1계층 품질매뉴얼 최종 승인; 품질방침/목표 문서 승인 |
| **품질책임자 (QMR)** | QMS 문서 체계 전반 관리 총괄; 2계층 절차서 승인; 외부 문서 배포 승인; 문서 관리 현황 경영검토 보고 |
| **품질팀장** | DMS 운영 관리; 문서 마스터 리스트(Document_Master_List) 유지; 문서 배포/회수 실행; 기록 보존 기간 관리; 폐기 문서 처리 |
| **각 부서장** | 소관 부서 문서/기록의 작성, 검토 및 관리; 부서 내 최신 문서 사용 보장; 기록의 식별, 보관 및 인출 관리 |
| **문서 작성자** | 문서 초안 작성; 문서번호 규칙 준수; 개정 이력 기록 |
| **전 임직원** | 최신 승인 문서만 사용; 비인가 문서 복사/배포 금지; 기록의 정확한 작성 및 보관 |

# 6. 문서 체계

## 6.1 문서 계층 구조

```
┌─────────────────────────────────────────┐
│  1계층: 품질매뉴얼 (QM-001)              │  ← 대표이사 승인
│  QMS 전체 프레임워크, AS9100D 4~10 조항   │
├─────────────────────────────────────────┤
│  2계층: 절차서 (QP-xxx) — 30건           │  ← QMR 승인
│  조항별 "무엇을, 누가, 언제" 규정         │
├─────────────────────────────────────────┤
│  3계층: 작업지침서 (WI-xxx) — 15건       │  ← 부서장 승인
│  "어떻게" 수행하는지 상세 기술            │
├─────────────────────────────────────────┤
│  3계층: 양식/기록 (F-xxx) — 21건         │  ← 부서장 승인
│  활동 증거, 데이터 수집 양식              │
└─────────────────────────────────────────┘
```

## 6.2 문서번호 체계

**AS9100D 신규 문서 (QP-xxx 체계)**

| 계층 | 접두사 | 번호 규칙 | 예시 |
|------|--------|----------|------|
| 1계층 | QM | QM-001 | QM-001 품질매뉴얼 |
| 2계층 | QP | QP-{조항}{순번} | QP-706 (7조 6번째) |
| 3계층 지침서 | WI | WI-{상위QP조항}-{순번} | WI-813-01 (QP-813 하위 1번) |
| 3계층 양식 | F | F-{상위QP조항}-{순번} | F-801-01 (QP-801 하위 양식 1번) |

**기존 ISO/IATF 문서 코드 체계 (A.XX.YY.ZZ)**

기존 ISO/IATF 기반 문서는 다음 부서 코드와 계층 체계를 사용하며, AS9100D 전환 시 참조 문서로 활용한다:

| 계층(XX) | 내용 | 예시 |
|---------|------|------|
| 01 | 품질방침/정책 | ISO.01.xx.xx |
| 02 | 절차서 (Procedure) | ISO.02.xx.xx |
| 03 | 규정/지침/작업지시 | ISO.03.xx.xx |
| 04 | 기록/양식 | ISO.04.xx.xx |

주요 부서 코드: ISO(Ban ISO/품질), GA(인사), QM(품질관리), R&D(개발), PUR(구매), WH(창고), PP(생산계획), MO(금형), CNC(CNC가공), ASS(조립), SAL(영업)

## 6.3 개정번호 규칙

| 상태 | 표기 |
|------|------|
| 초판 | Rev. 0 |
| 소개정 (내용 변경 없이 오타/서식 수정) | Rev. 0.1, 0.2 … |
| 정식 개정 (내용 변경) | Rev. 1, 2, 3 … |
| 폐기 | "OBSOLETE" 표기 + 폐기 일자 |

# 7. 문서화된 정보의 작성 및 갱신

## 7.1 문서 작성 절차

```
문서 작성 필요 인식
    │
    ▼
① 문서변경요청서(DCR) 작성 ──→ 신규 작성 또는 개정 사유 기술
    │
    ▼
② 문서 초안 작성 ──→ 문서번호 규칙 준수, 표준 양식 사용
    │
    ▼
③ 검토 (Review) ──→ 관련 부서 합의, 적합성/충족성 확인
    │                  (형상관리 대상 문서: CCB 검토 필수)
    ▼
④ 승인 (Approval) ──→ 계층별 승인권자 서명
    │
    ▼
⑤ DMS 등록 ──→ 문서 마스터 등록, 마스터 리스트 갱신
    │
    ▼
⑥ 배포 (Distribution) ──→ 관리본 배포, 구판 회수
    │
    ▼
⑦ 교육/전파 ──→ 해당 인원에게 변경 사항 전달
```

## 7.2 문서 작성 시 필수 포함 사항

모든 QMS 문서에는 다음 사항을 포함한다:

| 항목 | 내용 |
|------|------|
| 문서번호 | 6.2 문서번호 체계에 따라 부여 |
| 제목 | 문서 내용을 명확히 기술 |
| 개정번호 | 6.3 개정번호 규칙에 따라 부여 |
| 제정일/개정일 | 승인 완료일 기준 |
| 페이지 표시 | "Page X of Y" 형식 |
| 승인 서명란 | 작성-검토-승인 3단계 |
| 개정 이력 | 개정 사유, 변경 내용 요약 |
| 적용 규격 | 해당 AS9100D 조항 명시 |

## 7.3 문서 개정 절차

| 단계 | 활동 | 책임자 | 산출물 |
|------|------|--------|--------|
| 1 | 개정 필요성 식별 (내/외부 심사, 변경 요청, 규격 변경 등) | 모든 임직원 | DCR |
| 2 | DCR 접수 및 타당성 검토 | 품질팀장 | 승인된 DCR |
| 3 | 개정 초안 작성 (변경 부분 밑줄 또는 사이드바 표시) | 문서 작성자 | 개정 초안 |
| 4 | 관련 부서 검토 및 합의 | 관련 부서장 | 검토 기록 |
| 5 | 계층별 승인권자 승인 | 승인권자 | 서명 완료 문서 |
| 6 | DMS 갱신 및 마스터 리스트 업데이트 | 품질팀장 | 갱신된 DMS |
| 7 | 신판 배포 및 구판 회수 | 품질팀 | 배포/회수 기록 |
| 8 | 변경 사항 관련자 교육/전파 | 부서장 | 교육 기록 |

## 7.4 긴급 개정 (Urgent Revision)

안전, 품질 또는 법적 요구사항에 의해 즉시 문서 변경이 필요한 경우:

1. QMR의 구두 승인으로 즉시 시행 가능
2. 긴급 변경 사유를 DCR에 "긴급" 표시하여 기록
3. 시행일로부터 **5영업일 이내** 정식 승인 절차 완료
4. 미완료 시 변경 사항은 자동 철회

# 8. 문서화된 정보의 관리

## 8.1 배포 관리

| 배포 유형 | 관리 방법 |
|----------|----------|
| **전자 배포 (DMS)** | DMS에서 최신 승인본 자동 제공; 접근 권한별 열람/다운로드 제어; 인쇄 시 자동 "비관리본" 워터마크 |
| **하드카피 관리본** | 적색 "관리본" 스탬프 날인; 배포 대장(Distribution Log) 기록; 개정 시 구판 회수 및 신판 재배포 |
| **하드카피 비관리본** | 청색 "비관리본" 스탬프 날인; 참조용으로만 사용; 최신성 보장하지 않음을 수령자에게 고지 |

## 8.2 접근 권한 관리

| 역할 | 접근 수준 |
|------|----------|
| 대표이사/QMR | 전체 문서 읽기/쓰기/승인 |
| 부서장 | 소관 부서 문서 읽기/쓰기, 타 부서 문서 읽기 |
| 담당자 | 소관 업무 문서 읽기/쓰기, 타 업무 문서 읽기 |
| 고객/인증기관 | 사전 승인된 문서에 한해 읽기 (항공우주 요구사항) |
| 외부 공급자 | 해당 공급자 관련 사양/도면에 한해 읽기 |

## 8.3 문서 보관

| 항목 | 기준 |
|------|------|
| 전자 문서 원본 | DMS 서버 (자동 백업: 일 1회, 주 1회 외부 백업) |
| 하드카피 원본 | 품질팀 관리 캐비닛 (잠금 관리, 방화/방습 환경) |
| 폐기 문서 보관 | "OBSOLETE" 스탬프 날인 후 별도 보관함에 최소 1개 이전 개정판 보관 |
| 백업 주기 | 일간 자동 백업 + 주간 외부 미디어 백업 |

## 8.4 외부 출처 문서 관리

| 외부 문서 유형 | 관리 방법 |
|--------------|----------|
| 국제/국가 규격 (AS9100D, ISO 등) | 유효성 연 1회 확인; 개정 시 DMS 갱신 및 영향 평가 |
| 고객 도면/사양 | 접수 즉시 DMS 등록; 개정 관리; 구판 회수; 고객 승인 없이 수정 금지 |
| 공급자 인증서/성적서 | 유효기간 관리; 만료 전 갱신 요청; DMS 등록 |
| 법적/규제 요구사항 | 연 1회 이상 유효성 검토; 변경 시 QMS 영향 평가 |

# 9. 기록 관리

## 9.1 기록 관리 원칙

기록은 다음 요건을 충족하도록 관리한다:

- **식별**: 기록명, 기록번호, 일자, 작성자가 명확히 식별 가능
- **보관**: 검색 가능한 체계로 정리 (부서별/년도별/문서유형별)
- **보호**: 손상, 손실, 열화로부터 보호 (방화 캐비닛, 전자 백업)
- **검색**: 필요 시 신속하게 인출 가능 (DMS 검색 기능 활용)
- **보존 기간**: 규정된 기간 동안 유지
- **폐기**: 보존 기간 경과 후 규정된 방법으로 폐기

## 9.2 보존 기간

| 기록 유형 | 최소 보존 기간 | 근거 |
|----------|--------------|------|

## 9.3 기록 폐기 절차

1. 품질팀장이 보존 기간 경과 기록을 분기별로 확인
2. 폐기 대상 목록 작성 → 기록폐기요청서 작성
3. QMR 승인
4. 기밀 기록: 파쇄기 처리 (전자: 영구 삭제)
5. 일반 기록: 일반 폐기 가능
6. 폐기 기록부에 폐기 일자, 내역, 승인자 기록

# 10. 항공우주 특수 요구사항

## 10.1 고객 및 규제당국 접근권

AS9100D는 고객 및 규제당국이 관련 문서화된 정보에 접근할 수 있도록 요구한다.

| 요구사항 | 관리 방법 |
|---------|----------|
| 고객 접근권 | 계약에 명시된 문서에 대해 열람/사본 제공; 품질팀장 사전 승인 후 제공 |
| 규제당국 접근권 | 인증심사/감독심사 시 요청 문서 즉시 제공; 필요 시 사업장 내 열람 |
| 접근 기록 | 외부 접근 요청 시 요청일, 요청자, 제공 문서, 승인자를 기록 |

## 10.2 형상관리 문서 연계

형상관리 대상 문서(설계도면, BOM, 사양서 등)는 QP-803 형상관리 절차에 따라 추가 관리한다:

- 형상 문서의 변경은 반드시 ECN을 통해 처리
- CCB(형상통제위원회) 승인 필수
- 형상 기준선(Baseline) 변경 이력 유지
- 본 절차서(QP-706)의 일반 문서 관리 요건도 동시 적용

## 10.3 위조 부품 방지 관련 문서

SAE AS6081/AS6174에 따라 위조 부품 방지와 관련된 문서를 관리한다:

- 부품 인증서(CoC), 적합성 증명서, 시험 성적서의 진위 확인
- 의심 또는 확인된 위조 부품 관련 기록 보존 (무기한)
- 공급자 인증 문서의 유효성 검증 기록

## 10.4 특수공정 및 FAI 기록

| 기록 유형 | 특수 관리 요건 |
|----------|--------------|
| FAI 보고서 (AS9102) | AS9102 Part 1/2/3 양식 사용; 제품별 개별 파일 관리; 고객 요청 시 즉시 제출 가능 |
| 특수공정 인증 기록 | 작업자/공정별 자격 인증 유효기간 관리; 만료 전 갱신 프로세스 |
| NDT 기록 | 검사원 자격, 검사 결과, 장비 교정 기록 일괄 관리 |
