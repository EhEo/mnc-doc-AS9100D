# 1. 목적

본 절차서는 M&C Electronics VINA에서 생산하는 항공우주 제품의 전 수명주기에 걸쳐 식별 및 추적성을 보장하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.5.2 요구사항에 따른 제품 식별 및 추적성 관리 체계 수립
- 원자재 입고부터 완제품 출하까지 로트(Lot) 단위 양방향 추적성 확보
- 자재 증명서(Mill Certificate), 공정 기록, 출하 기록 간 연결고리 구축
- 부적합 발생 시 영향 범위의 신속한 파악 및 격리를 위한 추적 체계 확립
- 고객 및 규제 기관의 추적성 요구사항 충족
- **CAR-008** (로트 단위 추적 불가: 원자재 로트-공정-완제품 간 추적 연결고리 없음) **시정조치 반영**

> **배경**: 갭분석 결과, 바코드/라벨 시스템이 미구축되어 있고 추적성 요건을 단품 수준으로만 이해하여, 로트 단위의 체계적 추적이 불가능한 상태였다. 본 절차서는 이 근본 원인을 해소하기 위해 로트 번호 부여 체계, 공정별 이동 기록, ERP 연계를 포함한 종합 추적성 관리 체계를 수립한다.

# 2. 적용 범위

본 절차서는 M&C Electronics VINA의 QMS 내에서 다루는 모든 제품, 자재, 반제품 및 완제품에 적용한다.

**적용 대상:**
- 원자재 및 구매 부품 (수입검사 단계부터)
- 공정 중 반제품 (Work-in-Process, WIP)
- 완제품 (출하 전 최종검사 완료 제품)
- 외주 가공품 (도금, 열처리, 비파괴검사 등 외주 공정 반출/반입)
- 고객 제공 자산 (고객 지급 자재/GFP 포함)
- 수명 제한 부품 및 유효기간 관리 대상 자재

**적용 제외:**
- 사무용 소모품, 일반 간접재 (추적성 요구 없는 자재)
- 제외 사유 및 범위는 품질책임자(QMR)가 승인한다

**적용 사업장:**
- M&C Electronics VINA 본사 (베트남 소재)
- 인증 범위 내 모든 생산/검사/보관 구역

# 6. 제품 식별 체계

## 6.1 부품번호 부여 규칙

부품번호(Part Number)는 제품의 고유 식별을 위한 기본 단위이며, 다음 체계에 따라 부여한다.

**부품번호 구조:**

```
[제품군]-[순번]-[개정]
예: MCE-A1001-R01

MCE     : 회사 코드 (M&C Electronics)
A       : 제품군 코드 (A=항공우주, E=전자, M=기구)
1001    : 순차 부여 번호
R01     : 도면 개정 번호 (Rev. 01)
```

**제품군 코드:**

| 코드 | 분류 | 설명 |
|------|------|------|
| A | 항공우주 부품 | 항공기/우주 장비용 부품 |
| E | 전자 조립품 | PCB, 전자 모듈 등 |
| M | 기구 부품 | 기계 가공 부품, 구조물 |
| S | 서비스 부품 | 수리/정비용 교환 부품 |
| T | 치공구/지그 | 생산용 공구, 지그, 검사 치구 |

## 6.2 로트 번호 부여 규칙

로트 번호는 원자재 입고 시부터 부여되며, 전 공정에 걸쳐 추적의 핵심 단위가 된다.

**로트 번호 구조:**

```
[년도][월][일]-[순번]
예: 260415-001

26      : 년도 (2026)
04      : 월 (4월)
15      : 일 (15일)
001     : 당일 순번 (3자리)
```

**로트 번호 부여 원칙:**
- 동일 자재 + 동일 공급업체 + 동일 입고일 = 동일 로트 번호
- 자재 증명서(Mill Cert)가 상이한 경우 반드시 별도 로트 번호 부여
- 로트 번호는 입고 시 자재/구매팀이 부여하고, ERP에 즉시 등록한다
- 로트 번호는 자재의 전 공정 이동 기록에 반드시 기재한다

## 6.3 시리얼 번호 부여 규칙

시리얼 번호는 개별 추적이 필요한 제품에 부여한다.

**시리얼 번호 구조:**

```
[부품번호]-[로트번호]-[순번]
예: MCE-A1001-R01-260415001-0001

0001    : 로트 내 개체 순번 (4자리)
```

**시리얼 번호 부여 대상:**
- 안전 중요 부품 (Safety Critical Part)
- 수명 제한 부품 (Life-Limited Part)
- 고객이 시리얼 추적을 요구하는 부품
- 항공기 등록번호(A/C Registration Number) 추적이 필요한 부품
- 단가가 기준 금액 이상인 고가 부품 (기준: 품질팀장 결정)

## 6.4 식별 표시 방법

| 방법 | 적용 대상 | 내구성 | 비고 |
|------|----------|--------|------|
| 바코드 라벨 | 모든 자재/반제품/완제품 | 공정 내 | ERP 연계 스캔 |
| 각인/에칭 | 금속 부품 (고객 요구 시) | 영구적 | 부품 기능에 영향 없는 위치 |
| 잉크 마킹 | PCB, 전자 부품 | 반영구적 | 세척 내성 잉크 사용 |
| 태그/꼬리표 | 대형 부품, 외주 반출품 | 공정 내 | 분실 방지 조치 필요 |
| 포장 라벨 | 완제품 포장 | 출하 후 | 고객 요구 양식 준수 |
| 전자적 식별 | ERP 데이터 | 영구적 | 물리적 식별과 반드시 연계 |

**식별 표시 필수 기재 정보:**
- 부품번호 (Part Number)
- 로트 번호 (Lot Number)
- 시리얼 번호 (해당 시, Serial Number)
- 수량
- 검사 상태 (합격/부적합/대기)

# 7. 원자재 식별 및 입고 추적

## 7.1 입고 프로세스

```
공급업체 출하 → 입고 접수 → 자재 증명서 확인 → 로트 번호 부여 → 
바코드 라벨 부착 → ERP 입고 등록 → 수입검사 → 합격 자재 보관
```

## 7.2 자재 증명서 (Mill Certificate) 관리

**입고 시 확인 항목:**

| 확인 항목 | 확인 내용 | 책임자 |
|----------|----------|--------|
| 자재 증명서 수령 | 모든 원자재에 대해 Mill Cert 또는 CoC 첨부 확인 | 자재팀 |
| 내용 검증 | 재질, 화학성분, 기계적 성질이 도면/규격 요구사항과 일치 여부 | 품질팀 |
| 공급업체 확인 | ASL(승인 공급업체 목록) 등재 업체 여부 확인 | 구매팀 |
| 원산지 증명 | 고객 요구 시 원산지 증명서(CoO) 확인 | 구매팀 |
| 자재 증명서 번호 | Mill Cert 번호와 로트 번호의 매핑 기록 | 자재팀 |

**Mill Certificate 연계 기록:**

```
Mill Cert No. ↔ 공급업체 로트 번호 ↔ 당사 로트 번호 ↔ ERP 입고 번호
```

- 자재 증명서 원본(또는 인증 사본)은 문서 보관 절차(QP-706)에 따라 최소 제품 수명 + 3년 또는 고객 요구 기간 중 긴 기간 보관한다.
- Mill Cert 번호와 당사 로트 번호의 매핑 관계는 ERP에 등록하고, 별도 매핑 대장(QF-812-01)에도 기록한다.

## 7.3 입고 식별 라벨

입고 시 모든 자재에 다음 정보가 포함된 바코드 라벨을 부착한다:

| 라벨 기재 항목 | 내용 |
|---------------|------|
| 당사 부품번호 | Part Number |
| 당사 로트 번호 | 입고 시 부여한 로트 번호 |
| 공급업체 로트 번호 | 원래 공급업체의 로트/배치 번호 |
| Mill Cert 번호 | 자재 증명서 번호 |
| 입고일 | YYYY-MM-DD |
| 수량 | 입고 수량 |
| 유효기간 | 해당 시 (Shelf-Life 자재) |
| 검사 상태 | 대기(노란색) / 합격(녹색) / 부적합(빨간색) |

## 7.4 수입검사 후 처리

- **합격**: 녹색 합격 라벨 부착, 승인 보관 구역에 입고, ERP 상태 변경
- **부적합**: 빨간색 부적합 라벨 부착, 격리 구역 이동, QP-818에 따라 처리
- **조건부 합격**: 특채(Concession) 절차에 따라 처리, 추적성 기록에 특채 사유 명기

# 8. 공정 중 식별

## 8.1 로트 트래블러 (Lot Traveler / Route Card)

로트 트래블러는 로트/배치가 각 공정을 이동하면서 동반하는 공정 이동표로, 추적성의 핵심 문서이다.

**트래블러 기재 항목:**

| 항목 | 기재 내용 |
|------|----------|
| 작업지시 번호 | Work Order Number |
| 부품번호 / 개정번호 | Part No. / Rev. |
| 로트 번호 | 원자재 로트 번호와 공정 로트 번호 |
| 투입 수량 / 양품 수량 | 각 공정별 수량 추적 |
| 공정 순서 | 공정명, 공정 번호, 작업 표준서(WI) 번호 |
| 작업자 서명/일시 | 각 공정 완료 시 작업자 확인 |
| 검사자 서명/일시 | 공정검사 수행 시 검사원 확인 |
| 사용 설비 번호 | 사용 장비/설비의 고유 번호 |
| 사용 계측기 번호 | 사용 측정 장비의 관리 번호 |
| 특수공정 조건 | 온도, 시간, 압력 등 공정 파라미터 기록 |
| 부적합 발생 이력 | 공정 중 부적합 발생 시 처분 내용 |

## 8.2 공정 이동 규칙

**기본 원칙:**
- 트래블러가 없는 로트/배치는 다음 공정으로 이동할 수 없다
- 이전 공정의 작업자/검사자 서명이 없는 트래블러는 다음 공정 투입을 금지한다
- 로트 분할(Split) 시 분할 로트 각각에 새 트래블러를 발행하고, 원 로트 번호와의 관계를 기록한다
- 로트 병합(Merge)은 원칙적으로 금지한다. 불가피한 경우 품질팀장 승인 후 시행하며, 양쪽 원 로트 번호를 모두 기록한다

**공정 이동 흐름:**

```
원자재 출고 → [로트 번호 확인, 트래블러 발행]
    ↓
공정 1 (예: 절단) → [작업자 서명, 수량 기록, 설비 번호 기록]
    ↓
공정검사 1 → [검사자 서명, 합격/부적합 판정]
    ↓
공정 2 (예: 가공) → [작업자 서명, 수량 기록]
    ↓
    ... (반복)
    ↓
외주 공정 (해당 시) → [외주 반출/반입 기록, 외주처 성적서 첨부]
    ↓
최종검사 → [검사자 서명, AAM 부여]
    ↓
완제품 입고 → [ERP 완제품 등록, 식별 라벨 부착]
```

## 8.3 외주 공정 식별 관리

외주 가공(도금, 열처리, 비파괴검사 등) 시 추적성 유지:

| 단계 | 관리 항목 | 책임자 |
|------|----------|--------|
| 외주 반출 | 반출 로트 번호, 수량, 반출일 기록. 외주 반출 라벨 부착 | 생산팀 |
| 외주처 전달 | 추적성 요구사항 명시, 로트 혼합 금지 요구 | 구매팀 |
| 외주 반입 | 반입 수량, 외주처 성적서/CoC 확인, 로트 번호 일치 확인 | 품질팀 |
| 기록 연계 | 외주처 성적서 번호를 트래블러에 기록, ERP에 등록 | 품질팀 |

## 8.4 식별 불명 자재 처리

식별 표시가 불명확하거나 분실된 자재 발견 시:

1. 해당 자재를 즉시 **"식별 불명"** 표시(주황색 태그)하여 격리한다
2. 생산팀장에게 즉시 보고한다
3. 가용한 기록(ERP, 트래블러, 보관 위치 등)을 토대로 식별 복원을 시도한다
4. 식별 복원이 불가한 경우 품질팀장 판단 하에 부적합 처리(QP-818)한다
5. 식별 불명 자재를 제품에 사용하는 것은 **절대 금지**한다

# 9. 완제품 식별 및 출하 추적

## 9.1 완제품 식별

최종검사 합격 후 완제품에 다음 식별을 적용한다:

| 식별 항목 | 내용 |
|----------|------|
| 부품번호/개정번호 | Part No. / Rev. |
| 로트 번호 | 해당 완제품 로트 |
| 시리얼 번호 | 해당 시 (시리얼 추적 대상 제품) |
| 합격 표시 (AAM) | 합격 스탬프, 전자서명 또는 합격 라벨 |
| 검사일 | 최종검사 완료일 |
| 고객 주문 번호 | Purchase Order Number |

## 9.2 합격 권한 매체 (Acceptance Authority Media, AAM)

AAM은 제품의 합격/인수 상태를 나타내는 공식 수단이다.

**AAM 종류 및 관리:**

| AAM 유형 | 적용 장소 | 관리 방법 |
|----------|----------|----------|
| 합격 스탬프 | 트래블러, 검사 성적서 | 개인별 고유 스탬프 번호 부여, 스탬프 관리 대장(QF-812-07) 운영 |
| 전자서명 | ERP 내 검사 승인 | 개인별 ERP 계정, 비밀번호 관리, 서명 권한 별도 부여 |
| 합격 라벨 | 완제품 포장 | 일련번호 관리, 미사용 라벨 보관 통제 |

**AAM 관리 원칙:**
- AAM은 검사 권한이 부여된 인원만 사용할 수 있다
- 스탬프는 사용자 본인만 보관하며, 분실 시 즉시 품질팀장에게 보고한다
- 퇴직/이동 시 AAM을 회수하고, 해당 AAM 번호는 재사용하지 않는다
- AAM 부여/회수 이력은 스탬프 관리 대장에 기록한다

## 9.3 출하 추적 기록

출하 시 다음 기록을 생성하고 보관한다:

| 기록 | 내용 | 보관 기간 |
|------|------|----------|
| 출하 검사 기록 | 최종검사 성적서, 외관 검사 기록 | 제품 수명 + 3년 |
| 적합증명서(CoC) | 고객에게 발행하는 제품 적합 증명 | 제품 수명 + 3년 |
| 패킹 리스트 | 출하 품목, 수량, 로트/S/N 목록 | 10년 |
| 출하 전 추적성 검증 | 원자재-공정-완제품 추적 연결 최종 확인 | 10년 |
| 고객 요구 서류 | 고객이 요구하는 추가 증빙 (Mill Cert 사본 등) | 고객 요구 기간 |

**출하 전 추적성 확인 체크리스트:**
- [ ] 원자재 로트 → 완제품 로트 추적 가능 여부 확인
- [ ] 모든 공정 트래블러 완결 여부 확인
- [ ] 자재 증명서(Mill Cert) 매핑 완료 여부 확인
- [ ] 외주 공정 성적서 첨부 여부 확인
- [ ] 특수공정 기록 완비 여부 확인
- [ ] 부적합 처분 기록 완결 여부 확인 (해당 시)
- [ ] 고객 요구 추적성 서류 첨부 여부 확인

# 10. 로트 추적성 매트릭스

## 10.1 추적성 매트릭스 개요

추적성 매트릭스는 원자재부터 완제품까지의 전체 이력을 양방향으로 추적할 수 있도록 구성한 핵심 기록이다.

**매트릭스 구조:**

```
┌─────────────────────────────────────────────────────────────────┐
│                    추적성 매트릭스 (QF-812-05)                    │
├──────────────┬──────────────┬──────────────┬───────────────────┤
│  원자재 정보   │   공정 정보   │  완제품 정보   │   출하 정보       │
├──────────────┼──────────────┼──────────────┼───────────────────┤
│ 원자재 부품번호 │ 작업지시 번호  │ 완제품 부품번호 │ 고객 주문 번호    │
│ 공급업체명     │ 공정별 작업일  │ 완제품 로트 번호│ 출하일           │
│ 공급업체 로트   │ 작업자/검사자  │ 시리얼 번호    │ 출하처(고객)      │
│ 당사 로트 번호  │ 사용 설비     │ 최종검사 결과  │ 패킹 리스트 번호  │
│ Mill Cert No. │ 외주 공정 기록 │ CoC 번호      │ 항공기 등록번호   │
│ 입고일/수량    │ 부적합 이력   │ AAM 기록      │ (해당 시)        │
└──────────────┴──────────────┴──────────────┴───────────────────┘
```

## 10.2 정방향 추적 (Forward Traceability)

**목적:** 특정 원자재 로트가 어떤 완제품에 사용되었는지 추적

**절차:**
1. 원자재 로트 번호로 ERP에서 해당 자재가 출고된 작업지시(WO) 조회
2. 작업지시별로 생산된 완제품 로트 번호 확인
3. 완제품 로트별 출하 기록(고객, 출하일, 패킹 리스트) 확인
4. 필요 시 고객/항공기 등록번호까지 추적

**추적 경로:**

```
원자재 로트 → 작업지시(WO) → 공정 트래블러 → 완제품 로트 → 출하 기록 → 고객
```

**정방향 추적 활용 상황:**
- 원자재 부적합 판명 시 영향받는 완제품 범위 파악
- 공급업체 리콜(Recall) 대응
- 원자재 유효기간 초과 발견 시 영향 범위 확인

## 10.3 역방향 추적 (Reverse Traceability)

**목적:** 완제품에서 출발하여 사용된 원자재, 공정 이력을 추적

**절차:**
1. 완제품 로트/시리얼 번호로 ERP에서 작업지시(WO) 조회
2. 작업지시에 투입된 원자재 로트 번호 확인
3. 원자재 로트의 자재 증명서(Mill Cert), 공급업체 정보 확인
4. 공정별 작업자, 검사자, 사용 설비, 공정 조건 확인

**추적 경로:**

```
고객 클레임 → 출하 기록 → 완제품 로트 → 트래블러 → 원자재 로트 → Mill Cert → 공급업체
```

**역방향 추적 활용 상황:**
- 고객 클레임/불만 접수 시 원인 분석
- 완제품 부적합 발견 시 근본 원인 추적
- 규제 기관 조사 대응

## 10.4 추적성 검증 (Traceability Drill)

추적성 체계의 유효성을 정기적으로 검증하기 위해 **추적성 훈련(Traceability Drill)**을 실시한다.

| 항목 | 내용 |
|------|------|
| 실시 주기 | 분기 1회 이상 |
| 실시 방법 | 임의 로트를 선정하여 정방향/역방향 추적 수행 |
| 합격 기준 | 정방향: 4시간 이내 추적 완료 / 역방향: 4시간 이내 추적 완료 |
| 기록 | 추적성 훈련 기록서(QF-812-06) |
| 부적합 시 | 추적성 체계 개선 조치 수립 및 이행, 재검증 수행 |
| 책임자 | 품질팀장 |

# 11. ERP 시스템 연계

## 11.1 ERP 로트 추적 기능

ERP 시스템 내에 다음 로트 추적 기능을 구축하고 운영한다:

| 기능 | 설명 | 상태 |
|------|------|------|
| 로트 번호 자동 생성 | 입고 시 로트 번호 자동 부여 규칙 설정 | 구축 필요 |
| 로트별 입출고 이력 | 원자재 로트별 입고/출고/재고 현황 추적 | 구축 필요 |
| WO-로트 매핑 | 작업지시와 투입 원자재 로트 간 연결 기록 | 구축 필요 |
| 정방향/역방향 추적 조회 | 로트 번호 입력 시 연관 기록 일괄 조회 | 구축 필요 |
| Mill Cert 전자 보관 | 자재 증명서 스캔본을 로트에 연결하여 보관 | 구축 필요 |
| 유효기간 알림 | Shelf-Life 자재의 유효기간 만료 사전 알림 | 구축 필요 |
| 추적성 리포트 | 출하 시 추적성 매트릭스 자동 생성 | 구축 필요 |

> **참고**: ERP 로트 추적 기능은 IT 담당이 2026-05-23까지 구축 완료하며 (CAR-008 시정조치 일정), 구축 완료 전까지는 수동(엑셀/트래블러)으로 운영한다.

## 11.2 바코드/라벨 시스템

**바코드 라벨 운영 체계:**

| 항목 | 사양 |
|------|------|
| 바코드 형식 | Code 128 또는 QR Code (ERP 호환) |
| 라벨 프린터 | 열전사 바코드 프린터 (입고 구역, 생산 구역 배치) |
| 바코드 스캐너 | 유/무선 바코드 스캐너 (각 공정, 창고 배치) |
| 라벨 내용 | 부품번호, 로트 번호, 수량, 바코드, 검사 상태 (색상 구분) |
| 라벨 재질 | 내열/내유 라벨 (공정 환경에 따라 선정) |

**바코드 스캔 포인트:**

| 스캔 시점 | 위치 | 기록 내용 |
|----------|------|----------|
| 입고 | 수입검사 구역 | 입고 등록, 로트 생성 |
| 자재 출고 | 창고 | 작업지시에 대한 자재 출고 기록 |
| 공정 투입 | 각 공정 시작점 | 공정 시작 기록, 작업자 확인 |
| 공정 완료 | 각 공정 종료점 | 공정 완료 기록, 양품/불량 수량 |
| 외주 반출/반입 | 외주 입출고 구역 | 외주 반출/반입 기록 |
| 최종검사 | 검사 구역 | 최종검사 결과 등록 |
| 완제품 입고 | 완제품 창고 | 완제품 재고 등록 |
| 출하 | 출하 구역 | 출하 기록, 패킹 리스트 연계 |

## 11.3 수동 운영 보완 (ERP 미구축 시 과도기)

ERP 로트 추적 기능 구축 완료 전까지 다음 수동 체계로 운영한다:

- **로트 추적 대장 (엑셀)**: 입고 로트 번호, 작업지시, 완제품 로트 간 매핑 수동 기록
- **트래블러 (종이)**: 공정 이동표를 수기 기록, 완료 후 스캔하여 전자 보관
- **자재 증명서 파일링**: Mill Cert를 로트 번호순으로 물리적/전자적 보관

# 12. 부적합품 식별 및 격리

본 항은 QP-818 (부적합 출력 관리)과 연계하여 부적합품의 식별 및 격리 시 추적성 유지를 규정한다.

## 12.1 부적합품 식별 표시

| 구분 | 식별 방법 | 색상 |
|------|----------|------|
| 부적합 의심 | "검사 대기" 태그 | 노란색 |
| 부적합 확정 | "부적합" 라벨/태그 | 빨간색 |
| 특채(Concession) | "특채 합격" 라벨 | 주황색 |
| 폐기 대상 | "폐기" 라벨 | 빨간색 + 사선(X) |

## 12.2 부적합품 추적성 유지 원칙

- 부적합품의 로트 번호, 시리얼 번호는 **절대 제거하지 않는다**
- 부적합 처분(재작업, 수리, 특채, 폐기) 결과를 트래블러에 기록한다
- 부적합 발생 시 영향 로트 범위를 정방향 추적으로 즉시 파악한다
- 격리 구역에 보관 시 로트별로 분리하여 보관하며, 로트 혼합을 방지한다
- 부적합 처분 기록은 추적성 매트릭스에 포함한다

## 12.3 리콜 대응 추적

고객 또는 규제 기관으로부터 리콜 요청 시:

1. **영향 범위 파악**: 해당 원자재 로트의 정방향 추적 수행 (목표: 24시간 이내 완료)
2. **출하 로트 식별**: 해당 자재가 포함된 완제품 출하 로트 및 고객 목록 확보
3. **고객 통보**: 영향받는 고객에게 즉시 통보
4. **격리 조치**: 사내 재고 중 영향 로트 즉시 격리
5. **기록 보존**: 리콜 대응 전 과정을 문서화하여 보관

# 13. 항공우주 특수 요구사항

## 13.1 수명 제한 부품 (Life-Limited Parts, LLP) 식별

수명이 시간, 사이클 또는 날짜로 제한된 부품에 대해 다음을 관리한다:

| 관리 항목 | 내용 |
|----------|------|
| 식별 표시 | "LLP" 라벨 또는 마킹, 수명 제한 정보 기재 |
| 수명 추적 | 설치일, 사용 시간/사이클, 잔여 수명을 기록으로 관리 |
| 경고 체계 | 수명 도래 전 사전 경고 (ERP 알림 또는 수동 관리) |
| 기록 연계 | 수명 제한 부품 관리 대장(QF-812-08)에 등록 |
| 교환 이력 | 교환 시 구품/신품의 S/N, 교환일, 교환 사유 기록 |
| 고객 통보 | 수명 도래 정보를 고객에게 사전 통보 (해당 시) |

## 13.2 유효기간 관리 (Shelf-Life Management)

보관 기간이 제한된 자재(접착제, 실런트, 화학약품, 일부 전자부품 등)에 대해:

| 관리 항목 | 내용 |
|----------|------|
| 유효기간 표시 | 자재 라벨에 제조일, 유효기간(만료일) 명기 |
| FIFO 원칙 | 선입선출(First-In First-Out) 의무 적용 |
| 유효기간 감시 | 월 1회 유효기간 관리 대장(QF-812-09) 점검 |
| 만료 사전 알림 | 만료 30일 전 자재팀에 알림 (ERP 또는 수동) |
| 만료 자재 처리 | 유효기간 경과 자재는 즉시 격리 → 부적합 처리 (QP-818) |
| 연장 관리 | 공급업체의 공식 유효기간 연장 승인서가 있는 경우에만 연장 가능, 품질팀장 승인 필요 |
| 보관 조건 | 온도, 습도 등 보관 조건 준수 여부 정기 확인 |

**유효기간 자재 보관 구역:**
- 유효기간 자재 전용 보관 구역 지정
- 만료일 순서로 배열 (FIFO 용이)
- 보관 조건(온도/습도) 모니터링 기록 유지

## 13.3 항공기 등록번호 추적

고객이 요구하는 경우, 완제품이 장착될 항공기 등록번호(Aircraft Registration Number)까지 추적 가능하도록 한다:

- 고객 주문서에 항공기 등록번호 또는 장착 대상 정보가 명시된 경우 ERP에 등록
- 출하 기록(CoC, 패킹 리스트)에 항공기 등록번호 기재
- 추적성 매트릭스에 항공기 정보 포함

## 13.4 형상 식별 연계 (QP-803 연계)

제품 형상(Configuration)과 추적성의 연계:

- 부품번호의 개정 번호(Rev.)는 형상 기준선(Baseline)과 일치시킨다
- 도면 개정 시 해당 부품번호의 로트 구분을 명확히 한다 (구 Rev.와 신 Rev. 로트 혼합 금지)
- 형상 변경(ECN) 전후의 로트를 명확히 구분하여 추적 가능하도록 한다
- 형상 식별과 관련한 세부 사항은 QP-803 (형상관리 절차서)을 따른다

## 13.5 고객 자산 (GFP: Government/Customer Furnished Property) 식별

고객이 제공한 자재, 치공구, 시험 장비 등:

- 고객 자산임을 명확히 식별하는 별도 라벨 부착 ("고객 자산 / Customer Property")
- 고객 부여 식별번호 유지 (당사 번호와 병행 관리)
- 고객 자산 관리 대장에 등록하고 추적성 기록 연계
- 고객 자산의 분실, 손상, 부적합 시 고객에게 즉시 통보
