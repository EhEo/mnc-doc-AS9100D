# 1. 목적

본 절차서는 M&C Electronics VINA의 품질경영시스템(QMS)의 적용 범위를 결정하고, 그 경계와 적용 가능성을 명확히 하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 4.3 요구사항에 따른 QMS 범위의 체계적 결정
- QMS 적용 범위의 경계(Boundary)와 적용 가능성(Applicability) 명확화
- 적용 제외(Exclusion) 사유의 정당화 및 문서화
- 인증심사 시 명확한 범위 기술서 제공

# 6. QMS 범위 결정 프로세스

## 6.1 범위 결정 입력

QMS 범위 결정 시 다음 사항을 고려한다 (AS9100D 4.3):

| 입력 사항 | 내용 | 출처 |
|----------|------|------|
| **a) 외부/내부 이슈** | 조직 상황 분석 결과 | QP-401 |
| **b) 이해관계자 요구사항** | 고객, 규제당국, 인증기관 등의 요구 | QP-401 |
| **c) 제품/서비스** | 조직이 제공하는 제품 및 서비스 | 사업 계획 |
| 사업장/기능 | 포함할 물리적 위치 및 조직 기능 | 조직도 |
| 프로세스 | 포함할 핵심/지원/관리 프로세스 | QP-403 |

## 6.2 범위 결정 절차

```
① 조직 상황 분석 결과 검토 (QP-401)
    │
    ▼
② 제품/서비스 유형 식별
    │
    ▼
③ 사업장 및 조직 기능 식별
    │
    ▼
④ 해당 프로세스 식별
    │
    ▼
⑤ AS9100D 요구사항 적용 가능성 검토
    │
    ▼
⑥ 적용 제외 사항 검토 및 정당화 (8절)
    │
    ▼
⑦ 범위 기술서 초안 작성
    │
    ▼
⑧ 경영진 검토 및 승인
    │
    ▼
⑨ 품질매뉴얼(QM-001) 및 인증 신청서에 반영
```

# 8. 적용 제외 관리

## 8.1 적용 제외 원칙

AS9100D 4.3에 따라:
- 조직은 **적용할 수 없는 요구사항이 있는 경우에만** 적용 제외를 결정할 수 있다
- 적용 제외가 적합한 제품/서비스의 적합성 보장 능력 또는 책임에 영향을 미쳐서는 안 된다
- 적용 제외 사유는 **정당화(justification)**되어야 하며 문서화해야 한다

## 8.2 적용 제외 검토

| AS9100D 조항 | 적용 제외 가능 여부 | 정당화 사유 |
|-------------|-------------------|-----------|
| 4~7 (조직상황, 리더십, 기획, 지원) | 제외 불가 | 모든 조직에 적용 |
| 8.3 설계/개발 | 조건부 가능 | 설계 활동을 수행하지 않는 경우에만 가능. 단, 고객 사양의 해석/전환도 설계에 해당 |
| 8.5.1.2 생산공정 검증 | 제외 불가 | 항공우주 필수 (FAI) |
| 8.5.3 고객 소유물 | 조건부 가능 | 고객 제공 자재/장비/지적재산이 전혀 없는 경우에만 |
| 8.5.5 인도 후 활동 | 조건부 가능 | 계약상 인도 후 의무가 전혀 없는 경우에만 |
| 9~10 (성과평가, 개선) | 제외 불가 | 모든 조직에 적용 |

> **중요**: AS9100D에서는 ISO 9001보다 적용 제외가 매우 제한적이다. 항공우주 고유 요구사항(8.1.2~8.1.5, 8.4, 8.5.1.2 등)은 사실상 제외가 불가능하다.

## 8.3 적용 제외 문서화

적용 제외를 결정한 경우 다음을 문서화한다:

| 기록 항목 | 내용 |
|----------|------|
| 제외 조항 | AS9100D 조항 번호 및 제목 |
| 정당화 사유 | 조직의 제품/서비스/프로세스 특성에 기반한 구체적 근거 |
| 영향 평가 | 제외가 제품 적합성에 미치는 영향 없음을 확인 |
| 승인 | 대표이사 승인 |
| 기록 위치 | QM-001 §4.3 및 본 절차서 |

## 9.1 OASIS 등록 범위

| 등록 정보 | 내용 |

| EA/IAF 코드 | 해당 산업 분류 코드 |

## 9.2 고객 요구 범위

일부 항공우주 고객은 공급자에게 특정 프로세스 또는 제품군을 인증 범위에 포함하도록 요구할 수 있다:

- 고객 계약 시 인증 범위 요구사항 확인
- 범위 미포함 프로세스가 고객 요구 대상인 경우 범위 확대 검토
- 범위 변경 시 인증기관과 사전 협의

## 10.1 변경 트리거

|----------|------|
| 제품/서비스 추가·삭제 | 신규 제품군 진입, 기존 제품 단종 |
