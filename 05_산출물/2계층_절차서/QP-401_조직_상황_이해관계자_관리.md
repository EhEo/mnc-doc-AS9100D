# 1. 목적

본 절차서는 M&C Electronics VINA가 품질경영시스템(QMS)의 목적과 전략적 방향에 관련되는 외부 및 내부 이슈를 결정하고, 이해관계자의 요구사항을 파악·관리하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 4.1 요구사항에 따른 조직의 외부/내부 상황 체계적 파악
- AS9100D 4.2 요구사항에 따른 이해관계자 및 요구사항의 결정과 모니터링
- QMS 범위 결정(QP-402) 및 리스크/기회 관리(QP-601)의 입력 자료 확보
- CAR-015 (조직 상황 분석 체계 미흡) 시정조치 반영

---

# 2. 적용 범위

본 절차서는 M&C Electronics VINA의 QMS에 영향을 미치는 모든 외부/내부 이슈 및 이해관계자에 적용한다.

| 구분 | 적용 대상 |
|------|----------|
| 외부 이슈 | 법규/규제, 시장 동향, 기술 변화, 경쟁 환경, 사회·문화·경제 요인 |
| 내부 이슈 | 조직 구조, 역량, 문화, 기술, 인프라, 전략적 방향 |

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 조직 상황 (Context of the Organization) | 조직의 목적과 전략적 방향에 영향을 미치며, QMS의 의도된 결과 달성 능력에 영향을 미치는 외부/내부 이슈의 조합 |
| 외부 이슈 (External Issue) | 조직 외부 환경에서 발생하는 요인 (법규, 시장, 기술, 경쟁, 사회적 환경 등) |
| 내부 이슈 (Internal Issue) | 조직 내부에서 발생하는 요인 (조직 구조, 문화, 역량, 인프라 등) |
| 이해관계자 (Interested Party/Stakeholder) | 조직의 결정이나 활동에 영향을 미치거나, 영향을 받거나, 영향을 받는다고 인식하는 개인 또는 조직 |
| PESTLE 분석 | 정치(Political), 경제(Economic), 사회(Social), 기술(Technological), 법적(Legal), 환경(Environmental) 외부 요인 분석 프레임워크 |
| SWOT 분석 | 강점(Strengths), 약점(Weaknesses), 기회(Opportunities), 위협(Threats) 분석 프레임워크 |

# 6. 조직 상황 파악

## 6.1 외부 이슈 분석

PESTLE 분석 프레임워크를 활용하여 외부 이슈를 체계적으로 파악한다.

| 영역 | 분석 항목 | M&C 관련 예시 |
|------|----------|----------------------|
| **정치 (Political)** | 정부 정책, 무역 규제, 국방 정책, 수출 통제 | 베트남 투자 정책, 한-베트남 FTA, 항공우주 산업 수출 규제 |
| **경제 (Economic)** | 환율, 원자재 가격, 경기 동향, 인건비 | USD/VND 환율 변동, 항공우주 자재 수급, 베트남 인건비 추세 |
| **사회 (Social)** | 노동 시장, 교육 수준, 인구 통계 | 베트남 기술인력 수급, 항공우주 전문인력 부족 |
| **기술 (Technological)** | 기술 혁신, 산업 4.0, 디지털 전환 | 제조 자동화, NDT 기술 발전, 디지털 트윈 |
| **법적 (Legal)** | 법규, 규격 변경, 인증 요구 | AS9100D 개정, 항공우주 안전 법규, 베트남 노동법 |
| **환경 (Environmental)** | 환경 규제, 자원 관리, 기후 | RoHS/REACH 규제, 위험물질 관리, 에너지 효율 |

## 6.2 내부 이슈 분석

| 영역 | 분석 항목 | 검토 내용 |
|------|----------|----------|
| **조직 구조** | 부서 체계, 의사결정 구조, 보고 라인 | 8개 부서(생산/금형/생산계획/인사/품질/영업/개발/구매), Ban ISO 위원회 체계, 해외 공급자는 모기업 또는 고객 지정으로 관리 |
| **역량** | 인적 자원, 기술 역량, 핵심 역량 | 항공우주 전문인력 부족 (AS9100D 전환 교육 필요), 기존 ISO/IATF 심사원 자격 보유, 채용·교육 완료율 ≥90% 목표 |
| **문화** | 조직 문화, 품질 의식, 안전 문화 | ISO 9001/IATF 16949 운영 경험 (2014년~), Ban ISO를 통한 부서별 품질문화 |
| **기술/인프라** | 설비, 장비, IT 시스템, 시설 | ERP 시스템 운영, CMM/2D/XRF 등 정밀측정장비 보유, 소재지: Quế Võ, Bắc Ninh |
| **프로세스** | 현재 프로세스 성숙도, 효율성 | ISO 9001/IATF 16949 기반 프로세스 운영 중 → AS9100D 전환 필요 (특히 8.1.2~8.1.5, 8.5.1.2 항공우주 고유 조항) |
| **재무** | 투자 여력, 인증 비용 | ~151개 공급자 관리 중, AS9100D 인증/컨설팅 비용, 장비 투자 |
| **재무** | 투자 여력, 인증 비용 | AS9100D 인증 비용, 장비 투자, 교육 예산 |

## 6.3 SWOT 종합 분석

PESTLE(외부)과 내부 분석 결과를 SWOT으로 종합한다.

| | 긍정적 | 부정적 |
|---|--------|--------|
| **내부** | **강점(S)**: ISO 9001 인증 보유, 베트남 생산기지 비용 경쟁력, 기존 품질 인프라 | **약점(W)**: 항공우주 경험 부족, AS9100D 전문인력 미보유, 일부 프로세스 미수립 |
| **외부** | **기회(O)**: 항공우주 시장 성장, 글로벌 공급망 다변화 추세, 베트남 항공산업 육성 정책 | **위협(T)**: 인증 요구 강화, 경쟁사 AS9100D 보유, 전문인력 수급 어려움 |

# 7. 이해관계자 파악 및 요구사항 관리

## 7.1 이해관계자 식별

| 이해관계자 | 관계 유형 | QMS 관련 요구사항 |
|-----------|----------|-----------------|
| **항공우주 고객 (OEM/Tier 1)** | 계약/발주 | 제품 품질, 납기 준수(OTD), AS9100D 인증, FAI, 추적성, 특수공정 승인, 위조부품 방지 |
| **항공우주 규제당국 (EASA, FAA 등)** | 규제/감독 | 안전 요구사항, 형식 승인, 감항 규정, 제품 안전 보고 |
| **인증기관 (CB)** | 인증/심사 | AS9100D 적합성, OASIS 등록, 심사 협조 |
| **공급자/외주업체** | 조달/외주 | 납기, 품질 기준, ASL 등록, 추적성 |
| **임직원** | 고용/노동 | 안전한 작업환경, 역량 개발, 적정 보상 |
| **모기업/주주** | 소유/투자 | 사업 성과, 투자 수익, 경영 투명성 |
| **베트남 정부기관** | 법규/인허가 | 투자법, 노동법, 환경법, 세법 준수 |
| **지역사회** | 환경/사회 | 환경 보호, 지역 고용, 사회적 책임 |

## 7.2 이해관계자 요구사항 관리 프로세스

```
① 이해관계자 식별
    │
    ▼
② 요구사항 수집 ──→ 계약, 법규, 규격, 면담, 설문 등
    │
    ▼
③ 관련성 평가 ──→ QMS에 대한 영향도/중요도 평가
    │
    ▼
④ 이해관계자 등록부 기록 ──→ F-401-01
    │
    ▼
⑤ QMS 반영 ──→ 범위(QP-402), 리스크(QP-601), 프로세스(QP-403)에 반영
    │
    ▼
⑥ 모니터링 ──→ 요구사항 변화 추적
    │
    ▼
⑦ 경영검토 보고 ──→ QP-904 입력
```

# 8. 항공우주 특수 요구사항

## 8.1 항공우주 산업 고유 외부 이슈

AS9100D는 일반 ISO 9001 대비 다음 항공우주 고유 이슈를 추가로 고려하도록 요구한다:

| 이슈 영역 | 검토 항목 |
|----------|----------|
| 제품 안전 | 항공우주 제품의 안전 요구사항 변화, 사고/인시던트 동향 |
| 감항 규정 | EASA/FAA 등 감항당국의 규정 변경, 형식 승인 요구 |
| 산업 규격 | AS9100D, AS9102, AS6081 등 규격 개정 동향 |
| 공급망 리스크 | 항공우주 공급망 이슈 (자재 수급, 위조부품 위험, 공급자 인증 현황) |
| OASIS 데이터베이스 | IAQG OASIS 등록 및 인증 현황 모니터링 |

## 8.2 규제 요구사항 추적

항공우주 규제 요구사항의 변경을 추적하고 QMS에 반영하는 프로세스:

| 단계 | 활동 | 책임자 |
|------|------|--------|
| 1 | 규제/규격 변경 정보 수집 (IAQG, EASA, 고객 통보 등) | 품질팀장 |
| 2 | 변경 영향 평가 (QMS, 제품, 공정에 미치는 영향) | QMR |
| 3 | QMS 변경 필요 시 문서 개정/프로세스 변경 실행 | 해당 부서장 |
| 4 | 변경 사항 교육/전파 | 품질팀 |
| 5 | 경영검토 보고 | QMR |

# 9. 모니터링 및 검토

## 9.1 모니터링 방법

| 모니터링 대상 | 방법 | 주기 | 책임자 |
|-------------|------|------|--------|
| 외부 법규/규격 변경 | 관보, 규격기관 뉴스레터, 고객 통보 | 수시 | 품질팀장 |
| 시장/경쟁 환경 | 업계 동향 보고서, 전시회, 벤치마킹 | 분기 | 영업팀장 |
| 고객 요구사항 변화 | 계약 검토, 고객 미팅, 고객 감사 결과 | 수시 | 영업팀장 |
| 내부 역량/인프라 변화 | 인력 현황, 장비 상태, 교육 실적 | 분기 | 각 부서장 |
| 이해관계자 만족도 | 고객 만족도 조사, 임직원 설문, 공급자 피드백 | 연 1회 | QMR |
