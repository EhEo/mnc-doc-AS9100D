# 1. 목적

본 절차서는 M&C Electronics VINA가 품질경영시스템(QMS)에 필요한 프로세스를 식별하고, 프로세스 접근법에 기반하여 QMS를 수립, 실행, 유지 및 지속적으로 개선하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 4.4 요구사항에 따른 프로세스 접근법의 체계적 적용
- QMS에 필요한 프로세스 및 상호작용의 결정
- 프로세스의 효과적 운영과 관리를 위한 기준 수립
- PDCA 사이클과 리스크 기반 사고의 통합

---

# 2. 적용 범위

본 절차서는 M&C Electronics VINA의 QMS를 구성하는 모든 프로세스의 식별, 운영, 모니터링 및 개선에 적용한다.

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 프로세스 (Process) | 입력을 출력으로 변환하는 상호 관련되거나 상호 작용하는 활동의 집합 |
| 프로세스 접근법 (Process Approach) | 프로세스와 그 상호작용을 하나의 시스템으로 관리하여 의도된 결과를 효과적으로 달성하는 방법 |
| PDCA | Plan-Do-Check-Act 사이클 — 프로세스 및 시스템의 지속적 개선을 위한 반복적 방법론 |
| 프로세스 소유자 (Process Owner) | 해당 프로세스의 운영, 성과, 개선에 대한 총괄 책임을 갖는 자 |
| KPI (Key Performance Indicator) | 프로세스 성과를 측정하는 핵심 지표 |
| 핵심 프로세스 (Core Process) | 고객에게 직접 가치를 제공하는 프로세스 (COP: Customer-Oriented Process) |
| 지원 프로세스 (Support Process) | 핵심 프로세스의 운영을 지원하는 프로세스 |
| 관리 프로세스 (Management Process) | QMS 전체를 관리·개선하는 프로세스 |
| 터틀 다이어그램 (Turtle Diagram) | 프로세스의 입력, 출력, 자원, 방법, KPI 등을 체계적으로 기술하는 도구 |

# 6. 프로세스 접근법

## 6.1 프로세스 접근법 원칙

AS9100D는 QMS의 개발, 실행, 효과성 개선 시 프로세스 접근법을 채택하도록 요구한다. 프로세스 접근법의 핵심 원칙:

| 원칙 | 내용 |
|------|------|
| 프로세스 이해 | 활동을 상호 관련된 프로세스로 이해하고 관리 |
| 상호작용 관리 | 프로세스 간 입력/출력의 연결과 의존관계 관리 |
| PDCA 적용 | 각 프로세스에 PDCA 사이클을 적용하여 지속적 개선 |
| 리스크 기반 사고 | 프로세스 계획·운영 시 리스크와 기회를 고려 |

## 6.2 PDCA 사이클 적용

```
    ┌──────────────────────────────────────────┐
    │                  ACT                      │
    │  개선 조치 실행                             │
    │  시스템 갱신                                │
    │                                           │
    ├────────────────────┬─────────────────────┤
    │       PLAN         │        DO            │
    │  목표 수립          │  계획 실행            │
    │  프로세스 기획      │  프로세스 운영         │
    │  리스크/기회 결정   │  자원 배분/활용        │
    │  자원 결정          │  문서화된 정보 유지    │
    ├────────────────────┴─────────────────────┤
    │                  CHECK                    │
    │  성과 모니터링/측정                         │
    │  내부심사                                  │
    │  경영검토                                  │
    └──────────────────────────────────────────┘
```

# 8. 프로세스 관리 요건

## 8.1 프로세스별 관리 사항

AS9100D 4.4.1에 따라 각 프로세스에 대해 다음을 결정한다:

| 관리 항목 | 내용 | 문서화 도구 |
|----------|------|-----------|
| **a) 입력/출력** | 프로세스의 필요 입력과 기대 출력 | 터틀 다이어그램 |
| **b) 순서/상호작용** | 프로세스 간 연결과 의존관계 | 프로세스 상호작용도 |
| **c) 기준/방법/KPI** | 효과적 운영과 관리를 위한 기준 및 측정 지표 | 프로세스 기술서 |
| **d) 자원** | 필요 인적/물적 자원 | 프로세스 기술서 |
| **e) 책임/권한** | 프로세스 소유자 및 역할 배분 | 프로세스 기술서 |
| **f) 리스크/기회** | 프로세스에 영향을 미치는 리스크와 기회 | QP-601 연계 |
| **g) 평가/개선** | 성과 평가 및 개선 필요 사항 | QP-901, QP-1002 연계 |

## 8.2 터틀 다이어그램 (프로세스 기술 도구)

각 핵심/지원 프로세스에 대해 터틀 다이어그램을 작성한다:

```
         무엇으로 (자원/장비)        누가 (역량/인원)
              │                          │
              ▼                          ▼
┌─────┐  ┌───────────────────────────────────┐  ┌─────┐
│입력  │→│           프로세스명                 │→│출력  │
│(Input)│ │                                   │ │(Output)│
└─────┘  └───────────────────────────────────┘  └─────┘
              ▲                          ▲
              │                          │
         어떻게 (방법/절차)          성과지표 (KPI)
```

## 8.3 프로세스 성과 지표 (KPI) 예시

| 프로세스 | KPI | 목표 | 측정 주기 |
|---------|-----|------|----------|
| COP-01 영업/계약검토 | 계약검토 완료율 | 100% | 월간 |
| COP-03 구매 | 공급자 납기 준수율 | ≥ 95% | 월간 |
| COP-04 생산 | 생산 계획 달성률 | ≥ 95% | 주간 |
| COP-05 검사 | 초회 합격률 (First Pass Yield) | ≥ 98% | 월간 |
| COP-06 납품 | 납기 준수율 (OTD) | ≥ 95% | 월간 |
| SP-01 인적자원 | 교육 계획 이행률 | ≥ 90% | 분기 |
| MP-03 성과평가 | 내부심사 계획 이행률 | ≥ 90% | 연간 |
| MP-03 성과평가 | CAR 기한 내 완료율 | ≥ 95% | 분기 |

# 9. QMS 기획

## 9.1 QMS 기획 시 고려 사항

QMS를 기획할 때 다음을 고려한다:

| 고려 사항 | 내용 | 출처 |
|----------|------|------|
| 조직 상황 | 외부/내부 이슈 | QP-401 |
| 이해관계자 | 요구사항 | QP-401 |
| QMS 범위 | 적용 범위/제외 | QP-402 |
| 리스크/기회 | 프로세스별 리스크 | QP-601 |
| 품질목표 | 목표 및 달성 계획 | QP-501, QP-602 |
| 변경 관리 | QMS 변경 필요 시 | QP-603 |

## 9.2 QMS 변경 기획

QMS에 변경이 필요한 경우 (AS9100D 6.3 연계):

| 단계 | 활동 | 책임자 |
|------|------|--------|
| 1 | 변경 필요성 식별 (내/외부 이슈, 심사 결과, 경영검토 등) | 해당 부서장/QMR |
| 2 | 변경 영향 평가 (프로세스 상호작용, 자원, 문서) | QMR |
| 3 | 변경 계획 수립 (일정, 자원, 책임, 검증 방법) | QMR |
| 4 | QMS 무결성 유지 확인 | QMR |
| 5 | 변경 실행 및 문서 갱신 | 해당 부서 |
| 6 | 변경 유효성 확인 | QMR |

# 10. 항공우주 특수 요구사항

## 10.1 프로세스 효과성 평가

AS9100D는 프로세스의 단순 적합성뿐 아니라 **효과성**을 평가하도록 요구한다:

| 평가 항목 | 방법 |
|----------|------|
| KPI 달성도 | 목표 대비 실적 분석 |
| 고객 만족 기여도 | 프로세스 성과와 고객 만족의 상관관계 |
| 리스크 관리 효과 | 식별된 리스크의 완화 수준 |
| 개선 활동 실적 | 프로세스 개선 건수 및 효과 |

내부심사 시 AS9101 PEAR 체크리스트를 활용하여 프로세스 효과성을 평가한다 (QP-903 연계).

## 10.2 외주 프로세스 관리

AS9100D 4.4.1 Note에 따라, 외주(outsourced)로 수행되는 프로세스도 QMS에 포함하여 관리한다:

| 관리 항목 | 내용 |
|----------|------|
| 외주 프로세스 식별 | 외주로 수행되는 프로세스 목록 유지 |
| 관리 유형/정도 | 외주 프로세스의 중요도에 따른 관리 수준 결정 |
| 공급자 관리 | QP-808 외부공급자 관리와 연계 |
| 성과 모니터링 | 외주 프로세스의 KPI 모니터링 |

## 10.3 제품 안전 프로세스

AS9100D 8.1.4에 따른 제품 안전 관리가 QMS 프로세스 체계에 통합되어야 한다:
- 핵심 프로세스 전반에 안전 요구사항 반영
- 안전 관련 리스크의 체계적 관리 (QP-802, QP-804 연계)
- 안전 관련 부적합의 적시 보고 체계
