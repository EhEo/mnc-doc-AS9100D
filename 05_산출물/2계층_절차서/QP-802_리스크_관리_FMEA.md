# 1. 목적

본 절차서는 M&C Electronics VINA의 제품 실현(운용) 프로세스에서 발생할 수 있는 리스크를 체계적으로 식별, 평가, 완화하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.1.2 요구사항에 따른 운영 리스크 관리 프로세스 수립
- FMEA(고장모드 및 영향분석)를 활용한 체계적 리스크 평가
- 제품/공정/서비스의 적합성 보장을 위한 리스크 완화 조치
- CAR-001, CAR-002 (운영 리스크 관리 체계 미흡) 시정조치 반영

> **참고**: 전략적 리스크/기회(AS9100D 6.1)는 QP-601에서 관리한다. 본 절차서는 **운영적·기술적 리스크**(8.1.2)에 집중한다.

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 운영 리스크 (Operational Risk) | 제품 실현 프로세스에서 발생할 수 있는 바람직하지 않은 결과의 가능성 |
| FMEA (Failure Mode and Effects Analysis) | 고장모드 및 영향분석 — 잠재적 고장모드, 원인, 영향을 체계적으로 분석하는 방법 |
| DFMEA (Design FMEA) | 설계 FMEA — 제품 설계 단계에서 수행하는 고장 분석 |
| PFMEA (Process FMEA) | 공정 FMEA — 제조 공정에서 수행하는 고장 분석 |
| 고장모드 (Failure Mode) | 부품/공정이 요구 기능을 수행하지 못하는 방식 |
| RPN (Risk Priority Number) | 위험 우선순위 수 = 심각도(S) × 발생도(O) × 검출도(D) |
| AP (Action Priority) | AIAG-VDA 방식의 조치 우선순위 (High/Medium/Low) |
| 심각도 (Severity) | 고장 영향의 심각한 정도 (1~10) |
| 발생도 (Occurrence) | 고장 원인이 발생할 가능성 (1~10) |
| 검출도 (Detection) | 현재 관리 방법으로 고장을 검출할 능력 (1~10, 낮을수록 우수) |
| 잔여 리스크 (Residual Risk) | 완화 조치 후 남아 있는 리스크 |

# 5. 책임과 권한

| 역할 | 책임 |
|------|------|
| **품질책임자 (QMR)** | 운영 리스크 관리 체계 총괄; FMEA 결과 승인; 리스크 등록부 최종 검토; 경영검토 보고 |
| **개발팀장 (Trưởng bộ phận Phát triển/R&D)** | FMEA 회의 주관 (개발 담당자가 다기능팀 회의 의장); DFMEA 주관; PFMEA 공동 주관; 회의 전 관련 자료(제조 흐름도, 도면, 기술 요건, 과거 불량 분석 자료) 최소 1일 전 팀원 배포 |
| **생산팀장 (Trưởng bộ phận Sản xuất)** | PFMEA 공동 주관; 공정 리스크 식별 및 완화 조치 실행; 공정 능력 관리 |
| **품질팀장 (Trưởng bộ phận Chất lượng/QC)** | FMEA 촉진자(Facilitator) 역할; 검출 관리 평가; 리스크 등록부 유지; 리스크 데이터 분석 |
| **구매팀장** | 공급망 리스크 식별; 공급자 리스크 평가; 대체 공급원 확보 |
| **다기능팀 (CFT — 개발/금형/QC/생산 부서 대표)** | FMEA 워크숍 참여; 각 담당 공정에 대한 전문 의견 제공; 완화 조치 이행; APQP/FMEA/MSA/PPAP/SPC 도구 교육 이수 |

**FMEA 적용 원칙** (기존 R&D.02.06.07 기반):
- PFMEA를 기본으로 적용하며, 자사 설계 변경 제안 시 DFMEA도 수행한다
- FMEA는 "사건 발생 후"가 아닌 "사건 발생 전" 예방 행동이어야 한다
- 팀 전체의 집단 지식을 바탕으로 하며, 개인 판단이 아닌 팀 합의로 진행한다
- 과거 불량 이력, 통계 데이터, 실제 근거에 기반하여 객관적으로 분석한다

# 6. 운영 리스크 관리 체계

## 6.1 리스크 관리 프로세스

```
① 리스크 식별 (Identify)
    │
    ▼
② 리스크 분석 (Analyze) ──→ FMEA, 리스크 매트릭스
    │
    ▼
③ 리스크 평가 (Evaluate) ──→ RPN/AP 산출, 우선순위 결정
    │
    ▼
④ 리스크 완화 (Treat) ──→ 완화 조치 수립/실행
    │
    ▼
⑤ 모니터링/검토 (Monitor) ──→ 잔여 리스크 확인, 주기적 재평가
    │
    ▼
⑥ 기록/보고 (Record) ──→ 리스크 등록부 갱신
```

## 6.2 리스크 식별 트리거

| 트리거 | 시기 | FMEA 유형 |
|--------|------|----------|
| 신규 제품/설계 | 설계 검증 전 | DFMEA |
| 신규 공정/라인 | 양산 시작 전 | PFMEA |
| 설계 변경 (ECN) | 변경 실행 전 | DFMEA 갱신 |
| 공정 변경 (4M) | 변경 실행 전 | PFMEA 갱신 |
| 부적합/고객 불만 | 발생 즉시 | DFMEA/PFMEA 갱신 |
| 공급자 변경 | 변경 승인 전 | 공급망 리스크 평가 |
| 정기 재평가 | 연 1회 | 전체 FMEA 검토 |

# 7. FMEA 실행

## 7.1 FMEA 유형 및 적용

| 유형 | 적용 시기 | 주관 | 목적 |
|------|----------|------|------|
| **DFMEA** | 설계 검증(DVP&R) 전 | 기술팀장 | 설계 고장모드 식별, 설계 강건성 확보 |
| **PFMEA** | 양산 시작 전 / 공정 변경 시 | 생산팀장 | 공정 고장모드 식별, 공정 관리 최적화 |
| **공급망 FMEA** | 공급자 선정/변경 시 | 구매팀장 | 공급 리스크 식별, 대안 확보 |

## 7.2 FMEA 실행 절차

| 단계 | 활동 | 산출물 |
|------|------|--------|
| 1 | 다기능팀(CFT) 구성 (설계/생산/품질/구매) | 팀 구성표 |
| 2 | 분석 범위 정의 (제품/공정/시스템) | 범위 정의서 |
| 3 | 기능/요구사항 식별 | 기능 분석표 |
| 4 | 잠재 고장모드 식별 | 고장모드 목록 |
| 5 | 각 고장모드의 영향 분석 → 심각도(S) 평가 | S 평가 |
| 6 | 잠재 원인 식별 → 발생도(O) 평가 | O 평가 |
| 7 | 현재 관리방법(예방/검출) 식별 → 검출도(D) 평가 | D 평가 |
| 8 | RPN 산출 또는 AP(Action Priority) 결정 | RPN/AP |
| 9 | 조치 우선순위에 따른 완화 조치 수립 | 조치 계획 |
| 10 | 완화 조치 실행 및 효과 확인 | 개선 후 S/O/D 재평가 |
| 11 | FMEA 문서 갱신 및 승인 | 갱신된 FMEA |

## 7.3 RPN 기반 조치 기준

| RPN 범위 | 리스크 수준 | 조치 요구 |
|---------|-----------|----------|
| **200 이상** 또는 S≥9 | **높음 (High)** | **즉시 조치 필수** — 완화 조치 수립/실행, 효과 확인 완료 전 양산 불가 |
| **100~199** | **중간 (Medium)** | 완화 조치 수립 필요 — 실행 계획 및 기한 설정 |
| **50~99** | **낮음 (Low)** | 현 관리 수준 유지 — 모니터링 지속, 필요 시 개선 |
| **1~49** | **매우 낮음** | 수용 가능 — 정기 검토 시 재확인 |

> **중요**: 심각도(S) ≥ 9 (안전 관련)인 경우, RPN 값에 관계없이 **무조건 조치 필수**.

# 8. 리스크 완화 및 관리

## 8.1 완화 전략

| 전략 | 설명 | 적용 예 |
|------|------|--------|
| **회피 (Avoid)** | 리스크 원인 자체를 제거 | 위험 설계 변경, 위험 자재 대체 |
| **저감 (Reduce)** | 발생 가능성 또는 영향 감소 | Poka-Yoke, 자동 검사, 공정 개선 |
| **전가 (Transfer)** | 리스크를 제3자에게 이전 | 보험, 외주 (단, 책임은 유지) |
| **수용 (Accept)** | 잔여 리스크를 수용 | 저위험으로 평가 후 모니터링 |

## 8.2 리스크 등록부 (Risk Register)

리스크 등록부(F-814-01)에 다음 정보를 기록하고 유지한다:

| 항목 | 내용 |
|------|------|
| 리스크 ID | 고유 식별번호 |
| 리스크 출처 | DFMEA/PFMEA/공급망/기타 |
| 리스크 설명 | 고장모드/위험 상황 기술 |
| S / O / D | 심각도, 발생도, 검출도 |
| RPN / AP | 위험 우선순위 |
| 현재 관리방법 | 예방/검출 조치 |
| 완화 조치 | 추가 조치 내용, 담당자, 기한 |
| 잔여 리스크 | 조치 후 재평가 결과 |
| 상태 | Open / In Progress / Closed |

## 8.3 리스크 모니터링

| 활동 | 주기 | 책임자 |
|------|------|--------|
| 리스크 등록부 검토 | 분기 1회 | QMR |
| FMEA 정기 재평가 | 연 1회 | 기술팀장/생산팀장 |
| 고위험 항목 추적 | 월 1회 | 품질팀장 |
| 경영검토 보고 | 반기 | QMR |

# 9. 항공우주 특수 요구사항

## 9.1 AS9100D 8.1.2 고유 요구사항

AS9100D는 ISO 9001에 없는 **운영 리스크 관리**를 별도로 요구한다:

| 요구사항 | 관리 방법 |
|---------|----------|
| 제품 실현 계획 시 리스크 관리 | 신규 제품/공정 착수 시 FMEA 필수 실시 |
| 리스크 관리 책임 부여 | 프로세스별 리스크 소유자(Risk Owner) 지정 |
| 리스크 수용 기준 | 수용 가능 리스크 수준 기준 문서화 |
| 리스크 완화 조치 | 높음/중간 리스크에 대한 완화 계획 수립 필수 |

## 9.2 안전 관련 리스크

심각도(S) ≥ 9인 안전 관련 리스크는 QP-804 (제품/서비스 안전 관리)와 연계하여 추가 관리한다:

- 안전 관련 고장모드는 별도 표시 (Safety Critical)
- 안전 리스크는 수용(Accept) 불가 — 반드시 회피 또는 저감
- 안전 관련 리스크의 변경 시 고객/규제당국 통보 필요
- 안전 리스크 관련 기록은 제품 수명주기 + 15년 보존

## 9.3 특수공정 리스크

특수공정(열처리, 도금, 용접, NDT 등)의 리스크:
- 공정 결과를 사후 검사로 완전히 검증할 수 없으므로 리스크 관리 강화
- 공정 파라미터, 작업자 자격, 장비 상태에 대한 리스크 식별
- 공정 능력(Cpk) 모니터링과 연계

## 9.4 공급망 리스크

| 리스크 유형 | 평가 항목 |
|-----------|----------|
| 품질 리스크 | 공급자 품질 실적, 부적합 이력, 인증 상태 |
| 납기 리스크 | 납기 준수율, 리드타임 안정성 |
| 위조부품 리스크 | 인증 부품 조달 여부, 공급망 추적성 |
| 단일 공급원 리스크 | 대체 공급자 확보 여부 |
| 재무 리스크 | 공급자 재무 안정성 |
