# 1. 목적

본 절차서는 M&C Electronics VINA에서 요구사항에 적합하지 않은 출력(제품, 서비스, 중간 산출물)을 식별, 관리하여 의도하지 않은 사용 또는 인도를 방지하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.7 요구사항에 따른 부적합 출력의 체계적 관리
- 부적합 제품의 의도하지 않은 사용/인도 방지
- 부적합 처분(재작업, 수리, 특채, 폐기)의 체계적 의사결정
- MRB(자재심의위원회)를 통한 전문적 처분 결정

---

# 2. 적용 범위

본 절차서는 M&C Electronics VINA의 QMS 내에서 발생하는 모든 부적합 출력에 적용한다.

| 구분 | 적용 대상 |
|------|----------|
| 원자재/구매품 | 수입검사에서 부적합 판정된 자재 |
| 재공품 | 공정검사에서 부적합 판정된 반제품 |
| 완제품 | 최종검사에서 부적합 판정된 완제품 |
| 출하 후 발견 | 고객 반품, 필드 불량 |
| 외부 공급 서비스 | 외주 가공/처리 결과가 부적합한 경우 |

**참고**: 시스템/프로세스 부적합은 QP-1001 (부적합/시정조치 관리)에서 관리한다. 본 절차서는 **제품/출력**의 부적합에 집중한다.

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 부적합 출력 (Nonconforming Output) | 하나 이상의 요구사항을 충족하지 않는 프로세스의 출력(제품, 서비스, 중간 산출물) |
| 재작업 (Rework) | 부적합 제품을 원래 규격/사양에 적합하게 만드는 조치 |
| 수리 (Repair) | 부적합 제품을 의도된 용도에 사용 가능하게 만드는 조치. 원래 규격과 다를 수 있음 |
| 특채 (Use-as-is / Concession) | 부적합 제품을 원래 사양을 충족하지 않은 상태로 사용/인도를 허가하는 결정 |
| 폐기 (Scrap) | 부적합 제품을 사용 불가로 판정하여 폐기하는 처분 |
| 등급 변경 (Regrade) | 부적합 제품을 원래 용도와 다른 용도로 사용하기 위해 등급을 변경 |
| MRB (Material Review Board) | 자재심의위원회 — 부적합 제품의 처분을 결정하는 다기능 위원회 |
| 봉쇄 (Containment) | 부적합 제품의 확산 방지를 위한 즉각적 격리 조치 |
| 특별 특채 (Special Concession) | 고객 승인이 필요한 특채 결정 |

# 5. 책임과 권한

| 역할 | 책임 |
|------|------|
| **품질책임자 (QMR)** | 부적합 출력 관리 체계 총괄; MRB 의장; 특채/수리 최종 승인; 고객 통보 결정 |
| **품질팀장** | 부적합 접수 및 기록 관리; MRB 소집 지원; 처분 기록 유지; 부적합 데이터 분석 |
| **검사원** | 부적합 발견 시 즉시 보고 및 기록; 부적합 표식 부착; 격리 구역 이동; 재검사 실시 |
| **생산부서장** | 부적합 제품 발생 보고; 재작업/수리 실행; 작업자 교육; 봉쇄 조치 협조 |
| **기술부서장** | 수리 방법 승인; 기술적 처분 검토; 설계 관련 부적합 원인 분석 |
| **MRB 위원** | 부적합 처분 결정 참여; 전문 의견 제공 |

# 6. 부적합 출력의 식별 및 격리

## 6.1 부적합 발견 및 보고

```
부적합 출력 발견 (검사/공정/고객)
    │
    ▼
① 즉시 작업 중단 ──→ 부적합 발생 사실 보고 (구두 + 서면)
    │
    ▼
② 부적합 표식 ──→ 적색 "부적합" 태그/라벨 부착
    │                (부적합 번호, 발견일, 발견자, 부적합 내용)
    ▼
③ 격리 ──→ 부적합품 격리 구역(Red Tag Area)으로 이동
    │        물리적으로 양품과 분리
    ▼
④ 부적합 보고서 작성 ──→ 부적합 내용 상세 기록
    │
    ▼
⑤ 봉쇄 조치 ──→ 동일 로트/배치 확인
    │               재공품, 재고, 출하품 영향 범위 파악
    ▼
⑥ 처분 결정 ──→ 7절 참조
```

## 6.2 부적합 표식 체계

| 표식 유형 | 색상 | 적용 |
|----------|------|------|
| 부적합 태그 (Red Tag) | 적색 | 부적합 판정 즉시 부착 |
| 판정 보류 태그 (Yellow Tag) | 황색 | 검사 보류 또는 MRB 대기 |
| 격리 구역 표시 | 적색 바닥선/표지판 | 부적합품 보관 구역 식별 |
| 폐기 표식 | 적색 "SCRAP" | 폐기 결정 후 부착 |

## 6.3 격리 관리

| 항목 | 기준 |
|------|------|
| 격리 구역 | 생산 구역과 물리적으로 분리된 지정 구역 (Red Tag Area) |
| 접근 관리 | 권한 있는 인원만 접근 가능 |
| 출입 기록 | 격리 구역 입출고 기록 유지 |
| 보관 기간 | 처분 결정 후 5영업일 이내 이행 (특수 사유 제외) |
| 전자 추적 | DMS에 부적합품 상태 등록 및 추적 |

# 7. 부적합 출력의 처분

## 7.1 처분 유형

| 처분 유형 | 정의 | 승인 권한 | 비고 |
|----------|------|----------|------|
| **재작업 (Rework)** | 원래 규격에 적합하게 재가공 | 품질팀장 | 재작업 후 재검사 필수 |
| **수리 (Repair)** | 의도된 용도에 사용 가능하게 조치 (원래 규격과 다를 수 있음) | QMR + 기술부서장 | 수리 절차 사전 승인 필요; 항공우주: 고객 승인 필요 |
| **특채 (Use-as-is)** | 부적합 상태로 사용/인도 허가 | MRB + 고객 승인 | 항공우주: 반드시 고객 승인 필요 |
| **등급 변경 (Regrade)** | 다른 용도로 전환 사용 | QMR | 항공우주 부품에는 제한적 적용 |
| **폐기 (Scrap)** | 사용 불가 → 폐기 | 품질팀장 | 물리적 파괴 또는 사용 불가 표식 |
| **반품 (Return to Supplier)** | 공급자에게 반환 | 구매팀장 + 품질팀장 | 수입검사 부적합 시 |

## 7.2 처분 결정 프로세스

```
부적합 보고서 접수 (품질팀)
    │
    ▼
┌──────────────────┐
│  부적합 심각도     │
│  평가             │
└──────────────────┘
    │
    ├── 경미/명확 ──→ 품질팀장 판단으로 처분 결정
    │                 (재작업 또는 폐기)
    │
    └── 중대/복잡 ──→ MRB 소집 (8절 참조)
         또는
         수리/특채 ──→ MRB + 고객 승인
```

## 7.3 재작업 관리

| 단계 | 활동 | 책임자 |
|------|------|--------|
| 1 | 재작업 가능 여부 기술적 검토 | 기술부서장 |
| 2 | 재작업 지시서 발행 (방법, 기준 명시) | 품질팀장 |
| 3 | 재작업 실행 | 생산부서 |
| 4 | 재검사 (원래 검사 기준과 동일) | 검사원 |
| 5 | 합격 시 양품으로 복귀; 불합격 시 재처분 | 품질팀장 |
| 6 | 재작업 기록 유지 | 품질팀 |

## 7.4 수리 관리

| 단계 | 활동 | 책임자 |
|------|------|--------|
| 1 | 수리 방법 수립 및 문서화 | 기술부서장 |
| 2 | 고객 승인 획득 (항공우주 필수) | QMR/영업팀 |
| 3 | 수리 절차 승인 (MRB) | QMR |
| 4 | 수리 실행 | 생산부서 |
| 5 | 수리 후 검사 (수리 기준에 따라) | 검사원 |
| 6 | 수리 이력 제품에 영구 기록 | 품질팀 |

## 7.5 특채 (Concession) 관리

| 단계 | 활동 | 책임자 |
|------|------|--------|
| 1 | 특채 요청서 작성 (부적합 내용, 사용 근거, 리스크 평가) | 기술부서장 |
| 2 | MRB 검토 및 내부 승인 | MRB |
| 3 | 고객 승인 요청 (항공우주 필수) | QMR/영업팀 |
| 4 | 고객 승인 수령 및 기록 | 품질팀 |
| 5 | 특채 조건에 따른 인도/사용 | 해당 부서 |
| 6 | 특채 이력 영구 보존 | 품질팀 |

> **중요**: AS9100D에서 항공우주 제품의 **수리(Repair)** 및 **특채(Use-as-is)**는 반드시 **고객 승인**이 필요하다. 고객 승인 없이 수리/특채 처분을 실행해서는 안 된다.

# 8. MRB (자재심의위원회)

## 8.1 MRB 구성

| 구분 | 역할 |
|------|------|
| **의장**: QMR | MRB 회의 주관, 최종 처분 결정 승인 |
| **품질팀장** | 부적합 현황 보고, 기록 관리 |
| **기술부서장** | 기술적 처분 검토, 수리 방법 승인 |
| **생산부서장** | 생산 영향 평가, 재작업 가능성 검토 |
| **구매팀장** (해당 시) | 공급자 관련 이슈, 자재 대체 가능성 |
| **고객 대표** (해당 시) | 고객 요구사항 관련 의견 |

## 8.2 MRB 소집 기준

| MRB 소집 필요 | MRB 불필요 (품질팀장 결정) |
|--------------|-------------------------|
| 수리(Repair) 처분 | 단순 재작업 (원래 규격 복원) |
| 특채(Use-as-is) 처분 | 명확한 폐기 대상 |
| 반복 부적합 (동일 불량 3회+) | 공급자 반품 (명확한 규격 미달) |
| 안전 관련 부적합 | |
| 고객 사양 관련 부적합 | |
| 처분 판단이 복잡한 경우 | |

## 8.3 MRB 운영

| 활동 | 내용 |
|------|------|
| 소집 | QMR이 MRB 소집, 최소 24시간 전 통보 |
| 자료 | 부적합 보고서, 검사 데이터, 도면/사양, 이전 처분 이력 |
| 결정 | 만장일치 원칙; 합의 불가 시 QMR 최종 결정 |
| 기록 | MRB 회의록 작성 (참석자, 논의 내용, 결정 사항, 근거) |
| 고객 승인 | 수리/특채 시 고객 승인 절차 진행 |

# 9. 항공우주 특수 요구사항

## 9.1 처분 시 고객 승인

AS9100D 8.7.1에 따라 다음 처분에는 **고객 승인이 필수**:

| 처분 | 고객 승인 | 비고 |
|------|----------|------|
| 재작업 (Rework) | 계약에 명시된 경우 필요 | 원래 사양 복원이므로 일반적으로 불필요 |
| **수리 (Repair)** | **필수** | 수리 방법, 검사 기준에 대해 승인 |
| **특채 (Use-as-is)** | **필수** | 부적합 내용, 사용 근거에 대해 승인 |
| 등급 변경 (Regrade) | 필요 | 용도 변경에 대한 승인 |
| 폐기 (Scrap) | 불필요 | 단, 고객에게 통보 필요한 경우 있음 |

## 9.2 위조 부품 관련

AS9100D 8.1.5에 따라 위조 부품이 의심되거나 발견된 경우:

1. 해당 부품 **즉시 격리** — "의심 위조품" 표식
2. 양품 라인에 절대 투입 금지
3. 사용/수리/재작업/반품 **일체 금지** — 폐기만 허용
4. **24시간 이내** QMR에게 보고
5. 고객, 인증기관, 관련 정부기관에 통보
6. 해당 공급자로부터의 모든 납품 이력 조사
7. 관련 기록 **무기한 보존**

## 9.3 인도 후 발견된 부적합

출하/인도 후에 부적합이 발견된 경우:

| 조치 | 내용 |
|------|------|
| 즉시 고객 통보 | 부적합 내용, 영향 범위, 봉쇄 조치 보고 |
| 출하 이력 조사 | 해당 로트/배치의 전체 출하 이력 추적 |
| 회수/교환 계획 | 필요 시 고객과 협의하여 회수 또는 교환 |
| 시정조치 | QP-1001에 따른 CAR 발행 및 근본 원인 분석 |
