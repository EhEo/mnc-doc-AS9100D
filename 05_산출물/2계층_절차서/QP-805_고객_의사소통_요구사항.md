# 1. 목적

본 절차서는 M&C Electronics VINA가 고객과의 효과적인 의사소통 체계를 수립하고, 제품 및 서비스에 관한 요구사항을 체계적으로 결정하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.2.1/8.2.2 요구사항에 따른 고객 의사소통 체계 수립
- 제품/서비스 요구사항의 완전하고 정확한 결정
- 항공우주 고객의 특수 요구사항 관리
- 고객 피드백의 체계적 수집 및 처리

# 6. 고객 의사소통

## 6.1 의사소통 채널

AS9100D 8.2.1에 따라 다음 사항에 대해 고객과 의사소통한다:

| 의사소통 사항 | 담당 | 채널 |
|-------------|------|------|
| **a) 제품/서비스 정보** | 영업팀 | 제안서, 카탈로그, 기술 자료 |
| **b) 문의, 계약, 주문 처리 및 변경** | 영업팀 (Kinh doanh) | **ERP 시스템**, 이메일, 미팅, 고객 포탈 |
| **c) 고객 피드백 (불만 포함)** | 영업팀/품질팀 | 불만접수 양식, 이메일, 미팅 |
| **d) 고객 재산 취급** | 해당 부서 | 수령 확인서, 상태 보고 |
| **e) 비상 시 대응 조치** | QMR | 긴급 통보, 안전 보고 |

## 6.2 고객 불만/피드백 처리

```
고객 불만/피드백 접수 (영업팀)
    │
    ▼
접수 기록 (F-805-01) ──→ 24시간 이내 접수 확인 회신
    │
    ▼
품질팀 검토 ──→ 부적합 관련: QP-1001 CAR 연계
    │             고객 만족: QP-902 연계
    ▼
대응 조치 수립 및 실행
    │
    ▼
고객에게 결과 회신 ──→ 기한: 고객 요구 또는 10영업일 이내
    │
    ▼
고객 만족 확인
```

# 7. 제품/서비스 요구사항 결정

## 7.1 요구사항 유형

AS9100D 8.2.2에 따라 다음 요구사항을 결정한다:

| 요구사항 유형 | 내용 | 출처 |
|-------------|------|------|
| **a) 고객 명시 요구사항** | 도면, 사양서, PO, SOW에 명시된 요구 | 고객 문서 |
| **b) 고객 명시하지 않았으나 필요한 요구사항** | 의도된 용도에 필요한 기능, 성능 | 기술 검토 |
| **c) 법적/규제 요구사항** | 적용 법규, 규격, 인증 요건 | 법규 검토 |
| **d) 조직이 필요하다고 판단한 요구사항** | 내부 품질 기준, 안전 기준 | 내부 기준 |

## 7.2 요구사항 결정 프로세스

| 단계 | 활동 | 책임자 | 산출물 |
|------|------|--------|--------|
| 1 | 고객 문서 접수 (RFQ, 도면, 사양, SOW) | 영업팀 | 접수 기록 |
| 2 | 요구사항 목록 작성 (명시적 + 묵시적) | 영업팀/기술팀 | 요구사항 체크리스트 |
| 3 | 기술적 검토 (제조 가능성, 설계 요건) | 기술팀장 | 기술 검토 결과 |
| 4 | 법적/규제 요구사항 확인 | 품질팀장 | 적용 법규/규격 목록 |
| 5 | 특수 요구사항 식별 (8절) | QMR | 특수 요구사항 목록 |
| 6 | 납기/생산능력 검토 | 생산팀장 | 납기 확인 |
| 7 | 요구사항 충족 가능 여부 확인 | 영업팀장 | 검토 완료 |
| 8 | 계약검토 진행 (QP-806) | 영업팀 | 계약검토서 |

# 8. 항공우주 특수 요구사항

## 8.1 항공우주 고객 특수 요구사항

| 특수 요구사항 | 관리 방법 |
|-------------|----------|
| 핵심 특성 (Key Characteristics) | 도면/사양에서 식별, 공정/검사 계획에 반영 |
| 특수공정 요구 | 고객 승인 특수공정 목록 확인, 자격 인증 유지 |
| FAI 요구 | AS9102에 따른 초도품 검사 계획 수립 |
| 추적성 요구 | 로트/배치/시리얼 추적 범위 확인 |
| 위임검사원 | 고객 위임검사원 접근/시설 제공 |
| 시험/검증 요구 | 고객 지정 시험/검증 방법 적용 |
| 통보 요구 | 변경, 부적합, 안전 이슈 통보 조건 확인 |
| 문서/기록 보존 | 고객 요구 보존 기간 확인 (표준보다 긴 경우) |

## 8.2 고객 요구사항의 전달 (Flow-Down)

고객 요구사항 중 공급자에게 전달해야 하는 사항:
- 고객 특수 요구사항을 PO/발주서에 포함하여 공급자에게 전달
- 핵심 특성, 추적성 요구, 검사 요건 등 명시
- QP-808 외부공급자 관리와 연계
