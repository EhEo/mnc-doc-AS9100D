# 1. 목적

본 절차서는 M&C Electronics VINA의 품질경영시스템(QMS)에 관련된 내부 및 외부 의사소통을 체계적으로 관리하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 7.4항 의사소통(Communication) 요구사항의 체계적 이행
- 내부 의사소통을 통한 QMS 효과성 향상 및 품질 문화 확립
- 외부 이해관계자(고객, 공급자, 인증기관, 규제기관)와의 효과적 의사소통 보장
- 이상 발생 시 신속하고 체계적인 에스컬레이션 체계 구축
- 항공우주 산업 고유의 고객 불만 즉시 보고, 안전 의사소통, 인도성(On-Time Delivery) 이슈 통보 등 특수 요건 충족

# 2. 적용 범위

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 의사소통 (Communication) | 의도된 정보를 발신자로부터 수신자에게 전달하고 이해를 확인하는 과정 |
| 내부 의사소통 | 조직 내부 인원 간의 정보 교환 (수직적/수평적) |
| 외부 의사소통 | 조직과 외부 이해관계자 간의 정보 교환 |
| 에스컬레이션 (Escalation) | 이상 상황 발생 시 상위 결재/의사결정 라인으로 보고를 단계적으로 상향하는 절차 |
| 이해관계자 (Interested Party) | 조직의 의사결정이나 활동에 영향을 미치거나, 영향을 받을 수 있는 개인 또는 조직 |
| 인도성 (On-Time Delivery, OTD) | 고객이 요구한 납기에 맞추어 제품을 인도하는 성과 지표 |
| GIDEP (Government-Industry Data Exchange Program) | 미국 정부-산업 데이터 교환 프로그램 (안전/품질 Alert 공유) |
| Escape | 부적합 제품이 검출되지 않고 후공정 또는 고객에게 유출된 경우 |

| **각 부서장** | 소속 부서 내/부서 간 의사소통 활성화; 조회/회의 주관; 이상 발생 시 에스컬레이션 보고 |

# 6. 내부 의사소통

## 6.1 의사소통 5W1H 결정

AS9100D 7.4항에 따라, 조직은 QMS 관련 내부 의사소통에 대해 다음을 결정한다:

| 5W1H | 결정 사항 | 세부 내용 |
|------|----------|----------|
| What (무엇을) | 의사소통 내용 | 품질방침/목표, QMS 변경, 부적합/시정조치, 심사 결과, 품질 성과, 안전 이슈 |
| When (언제) | 의사소통 시기 | 정기(회의/조회/게시판), 수시(이상 발생 시 즉시), 변경 시 |
| Who (누가) | 발신자/책임자 | 해당 의사소통 유형별 책임자 (§5 참조) |
| Whom (누구에게) | 수신자/대상 | 전 직원, 관련 부서, 경영진, 특정 인원 |
| How (어떻게) | 의사소통 방법 | 회의, 조회, 게시판, 이메일, 공지, 문서 배포 |

## 6.2 내부 의사소통 유형

### 6.2.1 품질 회의

| 회의 유형 | 주기 | 참석자 | 주관 | 주요 안건 |
|----------|------|--------|------|----------|
| 전사 품질 회의 | 분기 1회 | 경영진 + 부서장 | QMR | 품질목표 달성현황, 부적합 동향, 고객 피드백, 개선 과제 |
| 부서별 품질 회의 | 월 1회 | 부서 내 인원 | 부서장 | 부서 품질 이슈, 부적합 사례, 개선 활동 |
| 일일 생산 회의 | 일 1회 | 생산/품질/기술 | 생산팀장 | 전일 품질 현황, 금일 생산 계획, 품질 주의사항 |
| 긴급 품질 회의 | 수시 | 관련 인원 | QMR/품질팀장 | 중대 부적합, 고객 불만, 안전 이슈 |

**회의록 관리**: 모든 품질 회의는 회의록을 작성하며, 의사결정 사항, 조치 담당자, 완료 기한을 명시한다.

### 6.2.2 조회/조례

| 항목 | 세부 내용 |
|------|----------|
| 주기 | 주 1회 이상 (부서장 재량으로 일일 조회 가능) |
| 내용 | 품질 관련 공지, 부적합 사례 공유, 안전/FOD 주의사항, 품질목표 달성현황 |
| 기록 | 조회 일지 작성 (일자, 참석자, 전달 내용 요약) |

### 6.2.3 게시판/전자 공지

| 매체 | 관리 담당 | 게시 내용 | 갱신 주기 |
|------|----------|----------|----------|
| 오프라인 게시판 | 품질팀 | 품질방침, 품질목표, 부적합 사례, 안전 공지, 인증 현황 | 월 1회 이상 |
| 사내 인트라넷 | IT팀/품질팀 | 절차서/지침서, 품질 공지, 회의록, 교육 자료 | 수시 |
| 이메일 공지 | 각 부서장/품질팀 | 긴급 품질 공지, 규격 변경, 고객 요구 변경 | 수시 |
| 사내 메신저 | 각 부서 | 실시간 업무 의사소통 | 상시 |

### 6.2.4 문서 배포

- 절차서/지침서 제·개정 시 관련 부서에 개정 내용 통보 (QP-706 연계)
- 외부 규격/표준 변경 시 영향 받는 부서에 변경 사항 공지
- 고객 요구사항 변경 시 관련 부서(설계/생산/품질/영업)에 즉시 전달

## 6.3 부서 간 의사소통

| 의사소통 경로 | 주요 내용 | 방법 |
|-------------|----------|------|
| 영업 → 생산/품질 | 수주 정보, 고객 요구사항, 납기 변경 | 수주 회의, 문서 배포 |
| 생산 → 품질 | 공정 이상, 부적합 발생, 작업 변경 | 부적합 보고서, 일일 회의 |
| 품질 → 생산 | 검사 결과, 시정조치 요구, 부적합 통보 | 부적합 보고서, 이메일 |
| 구매 → 품질 | 공급자 변경, 수입검사 이슈 | 공급자 평가 보고, 이메일 |
| 품질 → 경영진 | 품질 성과, 부적합 동향, 심사 결과 | 경영검토, 보고서 |
| 설계 → 생산/품질 | 설계 변경, 도면 개정, 특수 요건 | ECN, 문서 배포 |

# 7. 외부 의사소통

## 7.1 의사소통 5W1H 결정 (외부)

| 5W1H | 결정 사항 | 세부 내용 |
|------|----------|----------|
| What (무엇을) | 의사소통 내용 | 고객 요구/불만, 공급자 품질 요구, 인증/규제 대응, 계약 사항 |
| When (언제) | 의사소통 시기 | 계약/수주 시, 이상 발생 시, 정기 보고 시, 심사 시 |
| Who (누가) | 발신자/책임자 | 유형별 지정 책임자 (§5 참조) |
| Whom (누구에게) | 수신자/대상 | 고객, 공급자, 인증기관, 규제기관 |
| How (어떻게) | 의사소통 방법 | 공식 문서, 이메일, 회의, 전화, 포털 시스템 |

## 7.2 고객 의사소통

| 의사소통 항목 | 내용 | 책임자 | 방법 | 비고 |
|-------------|------|--------|------|------|
| 요구사항 접수 | 견적, RFQ, 도면, 규격, 납기 요구 | 영업팀장 | 이메일/포털 | QP-805/806 연계 |
| 계약 검토 결과 | 계약 이행 능력 확인 결과 통보 | 영업팀장 | 이메일/문서 | QP-806 연계 |
| 품질 보고 | 정기 품질 성과 보고 (PPM, OTD 등) | 품질팀장 | 보고서/이메일 | 고객 요구 주기 |
| 고객 불만 접수/대응 | 불만 접수, 원인 분석, 시정조치 회신 | QMR | 8D 보고서/이메일 | 접수 후 24시간 내 1차 회신 |
| 변경 통보 | 공정/자재/시설 변경 시 사전 통보 | 영업팀장/품질팀장 | PCN(Process Change Notice) | 고객 승인 후 변경 |
| 인도성 현황 | 납기 준수 현황, 지연 발생 시 통보 | 영업팀장 | 이메일/전화 | 지연 예상 시 즉시 |

### 고객 의사소통 프로세스

```
① 고객 요구/문의/불만 접수
   │  영업팀 또는 품질팀 접수
   ▼
② 내부 관련 부서 전달
   │  접수 내용에 따라 관련 부서 배분
   ▼
③ 검토/분석/조치
   │  관련 부서 검토 및 대응 준비
   ▼
④ 고객 회신
   │  공식 회신 (승인된 담당자)
   ▼
⑤ 기록 유지
   고객 의사소통 이력 관리
```

## 7.3 공급자 의사소통

| 의사소통 항목 | 내용 | 책임자 | 방법 |
|-------------|------|--------|------|
| 품질 요구사항 전달 | 구매 규격, 품질 요구, 검사 기준 | 구매팀장 | 구매주문서 + 품질 요구서 |
| 부적합 통보 | 수입검사 부적합, 공정 중 공급자 자재 부적합 | 품질팀장 | SCAR(Supplier Corrective Action Request) |
| 시정조치 요구/추적 | 공급자 시정조치 요구 및 이행 확인 | 품질팀장 | SCAR + Follow-up |
| 공급자 평가 결과 | 정기 평가 결과 통보, 등급 변동 안내 | 구매팀장 | 평가 보고서/이메일 |
| 변경 사전 승인 | 공급자 공정/자재/시설 변경 시 사전 승인 요구 | 구매팀장/품질팀장 | 변경 요청서 |
| 기술 지원 | 도면 해석, 규격 문의, 기술 요청 | 기술팀장 | 이메일/회의 |

## 7.4 인증기관 의사소통

| 의사소통 항목 | 내용 | 책임자 | 방법 |
|-------------|------|--------|------|
| 인증 심사 일정 | 심사 일정 조율, 범위 확인 | QMR | 이메일/공식 서한 |
| 심사 대응 | 심사 중 정보 제공, 문서 제출, 현장 안내 | QMR + 각 부서장 | 현장 대응 |
| 부적합 시정조치 | 심사 부적합에 대한 시정조치 제출 | QMR | 시정조치 보고서 |
| QMS 중대 변경 | 조직구조, 범위, 프로세스 중대 변경 시 통보 | QMR | 공식 서한/이메일 |
| 인증서 유지/갱신 | 인증서 갱신, 범위 변경 등 | QMR | 공식 서한 |

## 7.5 규제기관 의사소통

| 의사소통 항목 | 내용 | 책임자 | 방법 |
|-------------|------|--------|------|
| 안전 관련 보고 | 제품 안전 이슈, 항공안전 당국 보고 의무 사항 | 대표이사/QMR | 공식 보고서 |
| 법규 준수 보고 | 수출 통제, 환경 규제 등 법적 의무 보고 | 관리팀장 | 법정 양식 |
| 규제기관 질의 대응 | 규제기관의 조사/질의에 대한 대응 | 대표이사/QMR | 공식 서한 |
| GIDEP Alert | 안전/품질 Alert 수신 시 해당 조치 이행 | QMR/품질팀장 | GIDEP 시스템 |

# 8. 에스컬레이션 절차

## 8.1 에스컬레이션 개요

이상 상황 발생 시 신속하고 체계적인 보고를 통해 적시에 의사결정이 이루어지도록 단계별 에스컬레이션 체계를 운영한다.

> 참조: 기존 Escalation Procedure (Quy trình thông tin liên lạc, 2022.09.16)의 에스컬레이션 체계를 AS9100D 요구사항에 맞추어 확장

## 8.2 에스컬레이션 등급

M&C Electronics VINA의 실제 에스컬레이션 절차 (Escalation Procedure, Quy trình thông tin liên lạc, 2022.09.16)는 3개 적용 영역 × 3단계 수준으로 운영된다.

### 적용 영역 (3개 도메인)

| 도메인 | 베트남어 명칭 | 설명 |
|-------|------------|------|
| 신제품 개발 현황 | Dự án sản phẩm mới | APQP/신제품 개발 일정·품질 이슈 |
| 품질 현황 | Tình hình chất lượng | 공정 불량, 고객 불만, 인증 이슈 |
| 납기 현황 | Tình hình giao hàng | 납기 지연, 배송 이슈, 생산 계획 미달 |

### 에스컬레이션 3단계 기준

| 등급 | 보고 대상 | 신제품 개발 트리거 | 품질 트리거 | 납기 트리거 |
|------|---------|-----------------|-----------|-----------|
| **Level 1** | 선임 관리자 (Trưởng phòng cấp cao / Senior Manager) | ① KPI 내부 목표 미달 3개월 초과 | ① 내부 KPI 미달 3개월 초과 | ① 내부 KPI 미달 3개월 초과 |
| | | ② APQP 계획 대비 지연 | ② 동일 부적합 반복 발생 | ② 배송 계획 불이행 |
| | | ③ 동일 이슈 반복 발생 | ③ 동일 이슈 반복 발생 | ③ 동일 이슈 반복 발생 |
| **Level 2** | 총괄 이사 (Tổng giám đốc / General Director) | ① Level 1 조건이 3개월 이상 지속 | ① Level 1 조건이 3개월 이상 지속 | ① Level 1 조건이 3개월 이상 지속 |
| | | ② ISO/IATF 인증서 상실 | ② ISO/IATF 인증서 상실 | ② 배송 계획 완전 실패 |
| | | ③ 배송 계획 완전 실패 | | |
| **Level 3** | CEO | ① Level 2 조건이 3개월 이상 지속 | ① Level 2 조건이 3개월 이상 지속 | ① Level 2 조건이 3개월 이상 지속 |
| | | ② 자재 부족으로 생산 중단 위협 | ② 안전에 영향을 미치는 품질 문제 | ② 자재 부족으로 생산 중단 위협 |

## 8.3 에스컬레이션 보고 체계

```
Level 1: 담당자/부서장 → 선임 관리자 (Senior Manager / Trưởng phòng cấp cao)
                                              │ (3개월 이상 미해결 시)
Level 2: 선임 관리자 → 총괄 이사 (General Director / Tổng giám đốc)
                                   │ (3개월 이상 미해결 시)
Level 3: 총괄 이사 → CEO
```

> 에스컬레이션은 신제품 개발(APQP), 품질 현황, 납기 현황 3개 도메인에 독립적으로 적용된다.

## 8.4 에스컬레이션 프로세스

```
① 이상 발생 인지
   │  이상 상황 최초 인지자
   ▼
② 등급 판정
   │  에스컬레이션 등급 기준에 따라 판정
   ▼
③ 초기 보고
   │  등급별 보고 시한 내 보고 (구두 + 서면)
   ▼
④ 긴급 조치
   │  불량품 격리, 작업 중지, 안전 조치 등 즉시 대응
   ▼
⑤ 원인 분석 및 대책 수립
   │  근본 원인 분석, 시정/예방 조치 수립
   ▼
⑥ 이해관계자 통보
   │  고객/공급자/인증기관 등 해당 외부 이해관계자 통보
   ▼
⑦ 조치 실행 및 확인
   │  대책 실행, 효과성 확인
   ▼
⑧ 종결 보고
   에스컬레이션 종결 보고서 작성, 기록 유지
```

## 8.5 에스컬레이션 보고서

에스컬레이션 발생 시 다음 내용을 포함하는 보고서를 작성한다:

| 기록 항목 | 내용 |
|----------|------|
| 발생 일시/장소 | 이상 발생 일시 및 위치 |
| 발견자 | 최초 발견/보고자 |
| 에스컬레이션 등급 | Level 1~4 |

# 9. 항공우주 특수 요구사항

## 9.1 고객 불만 즉시 보고

항공우주 고객의 불만/클레임은 사업 지속성 및 제품 안전에 직결되므로 즉시 대응 체계를 운영한다.

| 항목 | 요구사항 |
|------|---------|
| 접수 시한 | 고객 불만 접수 후 **24시간 이내** 1차 회신 (접수 확인 + 초기 대응) |
| 내부 전파 | 접수 즉시 QMR + 관련 부서장에게 통보 |
| 8D 보고서 | 고객 요구 시한 내 8D 보고서 작성 및 제출 (통상 5~10 영업일) |
| 즉시 격리 | 해당 제품/로트 즉시 격리 및 출하 보류 |
| 경영진 보고 | Level 2 이상의 고객 불만은 경영진에 즉시 보고 |
| Escape 관리 | 부적합 제품이 고객에게 유출(Escape)된 경우, 에스컬레이션 Level 3 이상 적용 |

### 고객 불만 대응 프로세스

```
① 고객 불만 접수 (영업팀/품질팀)
   ▼
② 24시간 내 접수 확인 회신
   ▼
③ 내부 전파 및 에스컬레이션 등급 판정
   ▼
④ 해당 제품/로트 격리
   ▼
⑤ 원인 분석 (8D 방법론)
   ▼
⑥ 시정조치 수립 및 고객 제출
   ▼
⑦ 시정조치 이행 및 효과성 확인
   ▼
⑧ 종결 (고객 승인 후)
```

## 9.2 안전 관련 의사소통

제품 안전에 관련된 의사소통은 최우선 처리하며, 다음 사항을 준수한다:

| 항목 | 요구사항 | 연계 절차 |
|------|---------|----------|
| 안전 이슈 내부 보고 | 안전 위해(Safety Hazard) 발견 시 **즉시** 보고 (에스컬레이션 Level 3 이상) | QP-804 |
| 안전 이슈 외부 보고 | 고객/규제기관에 대한 안전 관련 보고 의무 이행 | QP-804 |
| 안전 Alert 수신 | GIDEP Alert, 고객 안전 통보 등 수신 시 즉시 검토 및 해당 조치 | QP-804 |
| 안전 의사결정 | 안전 관련 의사결정은 반드시 승인 권한자(QMR 이상) 확인 | QP-804 |
| 안전 기록 보존 | 안전 관련 의사소통 기록은 제품 수명 기간 + 3년 이상 보존 | QP-706 |

## 9.3 인도성(OTD) 이슈 통보

항공우주 산업에서 납기 준수는 핵심 성과 지표이며, 인도성 이슈는 고객에게 사전 통보해야 한다.

| 항목 | 요구사항 |
|------|---------|
| 납기 지연 예상 시 | 지연 예상 **즉시** 영업팀 → 고객 통보 (예상 지연 기간, 사유, 회복 계획) |
| 납기 지연 확정 시 | 공식 납기 변경 요청 (Delivery Schedule Change Notice) 발행 |
| 인도성 현황 보고 | 고객 요구 시 OTD 실적 정기 보고 (월별/분기별) |
| 내부 모니터링 | 생산 계획 대비 실적 일일 모니터링, 이상 발생 시 에스컬레이션 |
| 회복 계획 | 납기 지연 시 회복 계획(Recovery Plan) 수립 및 고객 공유 |

## 9.4 공급자 품질 이슈 의사소통

항공우주 공급망 관리를 위한 공급자 품질 이슈 의사소통:

| 항목 | 요구사항 |
|------|---------|
| 수입검사 부적합 | 부적합 발생 즉시 공급자에게 SCAR 발행 |
| 공급자 시정조치 추적 | SCAR 회신 기한 관리 (통상 10~15 영업일) |
| 특채(Concession) 요청 | 공급자 부적합에 대한 특채 시 고객 승인 필요 여부 확인 |
| 공급자 변경 통보 | 고객 승인 공급자(ASL) 변경 시 고객 사전 통보/승인 |
| 위조부품 의심 | 위조부품 의심 시 즉시 에스컬레이션 Level 3, 고객/공급자 동시 통보 |
