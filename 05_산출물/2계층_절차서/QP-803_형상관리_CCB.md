# 1. 목적

본 절차서는 Công ty TNHH M&C ELECTRONICS VIỆT NAM (이하 M&C Electronics VINA)의 항공우주 제품에 대한 형상관리(Configuration Management) 체계를 수립하고 운영하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.1.3 요구사항에 따른 형상관리 프로세스 수립
- 형상 항목(CI)의 체계적 식별, 변경 통제, 상태 기록 및 감사 수행
- 형상통제위원회(CCB)를 통한 변경 심의 및 승인 체계 확립
- 설계 기준선(Baseline)의 수립과 이력 관리를 통한 제품 무결성 보장
- **CAR-003** (형상관리 체계 미수립: 설계 기준선, 형상 식별자, 변경통제 절차 부재) **시정조치 반영**
- **CAR-004** (ERP 시스템과 형상관리 연계 없음: 도면 개정 이력과 생산 BOM 버전 불일치) **시정조치 반영**

> **참고**: 설계/개발 변경 관리(AS9100D 8.3.6)의 세부 절차는 QP-807에서 다룬다. 본 절차서는 **형상관리 체계(8.1.3)** 전반을 규정하며, 설계 변경 시 형상관리 측면의 요구사항을 정의한다.

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 형상 (Configuration) | 기능적·물리적 특성이 기술 문서에 기술된 대로의 제품 구성 상태 |
| 형상 항목 (Configuration Item, CI) | 형상관리 대상으로 선정된 하드웨어, 소프트웨어, 문서 또는 이들의 조합. 독립적으로 식별·관리되는 단위 |
| 형상 기준선 (Configuration Baseline) | 특정 시점에서 공식 승인된 형상 항목의 기술적 기준. 이후 모든 변경의 기준점 |
| 기능 기준선 (Functional Baseline, FBL) | 기능 요구사항 검토 시점에서 승인된 성능/기능 사양 기준선 |
| 할당 기준선 (Allocated Baseline, ABL) | 예비설계검토(PDR) 시점에서 승인된 CI별 요구사항 배분 기준선 |
| 제품 기준선 (Product Baseline, PBL) | 상세설계검토(CDR) 이후 확정된 제품의 물리적·기능적 형상 기준선 |
| CCB (Configuration Control Board) | 형상통제위원회 — 형상 변경 요청을 심의·승인하는 다기능 위원회 |
| ECR (Engineering Change Request) | 설계 변경 요청서 — 형상 변경의 필요성 및 근거를 기술한 요청 문서 |
| ECN (Engineering Change Notice) | 설계 변경 통보서 — CCB 승인 후 변경 실행을 지시하는 공식 문서 |
| ECP (Engineering Change Proposal) | 설계 변경 제안서 — 변경의 기술적 세부사항 및 영향 분석을 포함한 제안 문서 |
| 형상 감사 (Configuration Audit) | 형상 항목이 승인된 기준선과 일치하는지 확인하는 공식 검증 활동 |
| FCA (Functional Configuration Audit) | 기능 형상 감사 — 형상 항목이 기능/성능 요구사항을 충족하는지 검증 |
| PCA (Physical Configuration Audit) | 물리적 형상 감사 — 실물 제품이 기술 문서(도면, BOM)와 일치하는지 검증 |
| 형상 상태 기록 (CSA, Configuration Status Accounting) | 형상 항목의 현재 상태, 변경 이력, 기준선 정보를 기록·유지하는 활동 |

# 5. 책임과 권한

## 5.1 조직별 책임

| 역할 | 책임 |
|------|------|
| **품질책임자 (QMR)** | 형상관리 체계 총괄 감독; CCB 결정사항 최종 승인; 형상 감사 결과 경영검토 보고 |
| **기술팀장 (개발/Phát triển)** | CCB 의장; 형상 항목 식별 주관; 기준선 수립; ECR/ECN 기술 검토; 설계 변경 영향 분석 |
| **품질팀장 (Chất lượng)** | CCB 위원; 형상 감사(FCA/PCA) 주관; 형상 상태 기록 관리; 변경 후 품질 검증 |
| **생산팀장 (Sản xuất)** | CCB 위원; 변경의 생산 영향 평가; 변경 사항 생산 현장 반영; 금형/치공구 변경 관리 |
| **구매팀장 (PUR)** | CCB 위원; 변경의 공급망 영향 평가; 공급자 변경 통보; 구매 사양 변경 관리 |
| **생산계획팀장 (KHSX)** | 변경에 따른 생산 일정 조정; 재고/WIP 처리 계획 수립 |
| **금형팀장 (Khuôn)** | 금형/지그/치구 변경 실행; 금형 형상 이력 관리 |
| **영업팀장 (Kinh doanh)** | 고객 승인 필요 변경 시 고객 통보 및 승인 획득; 납기 영향 커뮤니케이션 |
| **형상관리 담당자 (CM Officer)** | 형상 항목 등록/관리; ECR/ECN 접수/배포; 형상 상태 기록부 유지; CCB 회의 준비 및 기록 |

## 5.2 CCB (형상통제위원회) 구성

| 구분 | 직위 | 역할 |
|------|------|------|
| 의장 | 기술팀장 (개발부서장) | CCB 회의 주재, 변경 검토 총괄 |
| 상임위원 | 품질팀장 | 품질 영향 평가, 형상 감사 연계 |
| 상임위원 | 생산팀장 | 생산 영향/실행 가능성 평가 |
| 상임위원 | 구매팀장 | 공급망/원가 영향 평가 |
| 간사 | 형상관리 담당자 | 회의 준비, 기록, ECR/ECN 관리 |
| 임시위원 | 관련 부서 담당자 | 안건에 따라 금형, 생산계획, 영업 등 참여 |

**CCB 운영 원칙**:
- 정기 회의: 월 1회 (매월 첫째 주 수요일)
- 긴급 회의: 안전/감항성 관련 변경, 고객 긴급 요청 시 24시간 이내 소집
- 의결 정족수: 의장 + 상임위원 3인 중 2인 이상 참석
- 의결 방법: 참석 위원 과반수 동의 (안전 관련 사항은 만장일치)

# 6. 형상관리 프로세스

## 6.1 형상관리 프로세스 흐름도

```
┌─────────────────────────────────────────────────────────────────┐
│                    형상관리 프로세스 흐름도                        │
│              Configuration Management Process Flow               │
└─────────────────────────────────────────────────────────────────┘

  ① 형상 식별 (Configuration Identification)
     │  · CI 선정 및 번호 부여
     │  · BOM, 도면, 사양서에 CI 번호 할당
     │  · 형상 기준선(Baseline) 수립
     │
     ▼
  ② 형상 관리/변경 통제 (Configuration Control)
     │  · ECR 접수 → 영향 분석
     │  · CCB 심의/승인
     │  · ECN 발행 → 변경 실행
     │
     ├──→ [CCB 반려] → ECR 보완 또는 종결
     │
     ▼
  ③ 형상 상태 기록 (Configuration Status Accounting)
     │  · 기준선 이력 기록
     │  · 변경 이력 추적 (ECR/ECN 이력)
     │  · 현재 형상 상태 유지
     │
     ▼
  ④ 형상 감사 (Configuration Audit)
     │  · FCA: 기능/성능 요구사항 충족 검증
     │  · PCA: 실물과 기술 문서 일치 검증
     │  · 부적합 발견 시 → 시정조치 (QP-1001)
     │
     ▼
  ⑤ 기준선 갱신 / 기록 보존
     · 감사 결과 반영
     · 형상관리 기록부 갱신
     · 다음 주기 형상관리 환류

  [전 단계 공통] ERP 시스템 연계 — 도면↔BOM 정합성 자동 검증
                 (CAR-004 시정조치)
```

## 6.2 형상관리 활동 트리거

| 트리거 | 시기 | 형상관리 활동 |
|--------|------|-------------|
| 신규 제품/설계 | 설계 착수 시 | CI 식별, 기능 기준선(FBL) 수립 |
| 예비설계검토 (PDR) | PDR 완료 후 | 할당 기준선(ABL) 수립 |
| 상세설계검토 (CDR) | CDR 완료 후 | 제품 기준선(PBL) 수립 |
| 설계 변경 요청 | ECR 접수 시 | ECR → CCB → ECN 프로세스 |
| 공정/4M 변경 | 변경 발생 시 | 공정 형상 변경 통제 |
| 고객 요구사항 변경 | 변경 접수 시 | 요구사항 변경 → 기준선 재평가 |

| FAI (초도품 검사) | FAI 실행 시 | PCA 연계 수행 |

# 7. 형상 식별 (Configuration Identification)

## 7.1 형상 항목(CI) 선정 기준

다음 기준에 해당하는 항목을 형상 항목(CI)으로 선정하여 관리한다:

| 기준 | 설명 |
|------|------|
| 안전 핵심 항목 | 안전/감항성에 영향을 미치는 부품, 어셈블리, 소프트웨어 |
| 주요 인터페이스 항목 | 다른 시스템/부품과 물리적·기능적 인터페이스를 갖는 항목 |
| 고객 지정 항목 | 고객이 형상관리를 요구하는 항목 |
| 규제 관련 항목 | 감항 인증, 규제 요구사항에 직접 관련된 항목 |
| 기술적 리스크 항목 | 기술적 복잡도가 높거나 성능에 핵심적인 항목 |
| 호환성/교환성 항목 | 호환성/교환성(Interchangeability) 유지가 필요한 항목 |

## 7.2 형상 항목 번호 체계

형상 항목에 고유 식별 번호를 부여하며, 다음 체계를 적용한다:

```
CI-[제품군]-[유형]-[일련번호]-[개정번호]

예시:
CI-AES-HW-001-A   →  AES 제품군, 하드웨어, 001번, 개정 A
CI-AES-SW-012-B   →  AES 제품군, 소프트웨어, 012번, 개정 B
CI-AES-DC-005-A   →  AES 제품군, 문서, 005번, 개정 A
```

| 코드 | 유형 |
|------|------|
| HW | 하드웨어 (부품, 어셈블리, 금형, 지그) |
| SW | 소프트웨어 (임베디드, 시험 프로그램) |
| DC | 문서 (도면, 사양서, 시험 절차서) |
| FW | 펌웨어 |

## 7.3 형상 문서 관리

| 문서 유형 | CI 번호 부여 대상 | 관리 체계 |
|----------|-----------------|----------|
| 제품 도면 | 모든 항공우주 제품 도면 | 도면번호 + CI 번호 매핑, 개정 이력 관리 |
| BOM (Bill of Materials) | 제품별 BOM | BOM 버전과 도면 개정 연계 관리 (ERP 연동) |
| 제품 사양서 | 성능/재료/시험 사양서 | CI 번호 부여, 기준선 포함 |
| 소프트웨어 | 임베디드 SW, 시험 프로그램 | 버전 관리 (형상관리 도구 사용) |
| 공정 문서 | 공정도, 작업표준서, QC 공정표 | 생산 형상으로 별도 관리 |

## 7.4 기준선(Baseline) 수립

| 기준선 유형 | 수립 시점 | 포함 내용 | 승인 권한 |
|------------|----------|----------|----------|
| 기능 기준선 (FBL) | 요구사항 검토 완료 | 고객 요구사항, 기능 사양 | CCB 의장 + QMR |
| 할당 기준선 (ABL) | PDR 완료 | CI별 할당 요구사항, 인터페이스 정의 | CCB |
| 제품 기준선 (PBL) | CDR 완료 / FAI 완료 | 상세 설계, 제조 도면, BOM, 시험 절차 | CCB + 고객 (해당 시) |

기준선 수립 시 다음 문서를 포함하여 형상 기준선 패키지를 구성한다:
- 해당 CI의 도면, 사양서, BOM
- 적용 규격/표준 목록
- 인터페이스 정의 문서
- 시험/검증 계획
- 기준선 승인 기록

# 8. 형상 관리 (변경 통제, Configuration Control)

## 8.1 변경 통제 프로세스 흐름

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  변경     │    │  영향     │    │  CCB     │    │  ECN     │    │  변경     │
│  요청     │───→│  분석     │───→│  심의     │───→│  발행     │───→│  실행     │
│ (ECR)    │    │          │    │  /승인    │    │          │    │  /검증    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
     │                              │                               │
     │                         [반려/보완]                          │
     │                              │                               ▼
     │                              ▼                         ┌──────────┐
     │                        ECR 보완                        │  기준선   │
     │                        또는 종결                        │  갱신     │
     │                                                        │  /기록    │
     │                                                        └──────────┘
     │
     └──→ [긴급 변경] ──→ CCB 의장 단독 승인 ──→ ECN 발행 ──→ 사후 CCB 추인
```

## 8.2 ECR (설계 변경 요청) 절차

### 8.2.1 ECR 발행

- **발행 가능자**: 모든 부서의 임직원
- **접수처**: 형상관리 담당자 (CM Officer)
- **양식**: F-803-01 (설계변경요청서, ECR)

### 8.2.2 ECR 필수 기재 사항

| 항목 | 내용 |
|------|------|
| ECR 번호 | 자동 채번 (ECR-YYYY-NNN 형식) |
| 변경 대상 CI | CI 번호, 도면번호, 현재 개정번호 |
| 변경 사유 | 품질 개선, 고객 요구, 원가 절감, 부적합 시정 등 |
| 변경 내용 (상세) | 현재 형상 → 변경 후 형상 비교 |
| 긴급도 | 일반/긴급/초긴급 |
| 요청자/요청 부서 | 성명, 부서, 일자 |

### 8.2.3 변경 분류

| 분류 | 정의 | 승인 권한 |
|------|------|----------|
| Class I (중대 변경) | 형태/기능/성능/신뢰성/안전/호환성에 영향 또는 고객 승인 필요 변경 | CCB 심의 + QMR 승인 + 고객 승인 |
| Class II (경미 변경) | 형태/기능/성능에 영향 없는 경미 변경 (표기법, 편집 등) | CCB 의장 승인 |

## 8.3 영향 분석

ECR 접수 후 관련 부서는 다음 항목에 대한 영향 분석을 수행한다:

| 분석 항목 | 담당 | 내용 |
|----------|------|------|
| 기술적 영향 | 기술팀 | 성능, 신뢰성, 인터페이스, 호환성, 안전/감항성 |
| 생산 영향 | 생산팀 | 공정 변경, 설비/금형 변경, 리드타임, 재작업 |
| 품질 영향 | 품질팀 | 검사 기준 변경, FAI 재실시 필요 여부, 리스크 |
| 원가 영향 | 구매팀 | 자재비, 가공비, 금형비, 폐기 비용 |
| 일정 영향 | 생산계획 | 납기, 재고/WIP 처리, 전환 일정 |
| 고객 영향 | 영업팀 | 고객 통보/승인 필요 여부, 계약 영향 |

## 8.4 CCB 심의 및 승인

### 8.4.1 심의 절차

1. 형상관리 담당자가 ECR 및 영향 분석 결과를 CCB 위원에게 사전 배포 (회의 3일 전)
2. CCB 회의에서 변경의 기술적/경제적/일정 타당성 심의
3. 심의 결과: **승인 / 조건부 승인 / 반려 / 보류**
4. 결과를 F-803-03 (CCB 회의록)에 기록

### 8.4.2 승인 기준

| 판정 | 조건 |
|------|------|
| 승인 | 영향 분석 완료, 리스크 수용 가능, 이행 계획 적절 |
| 조건부 승인 | 추가 검증/시험 조건 부여 후 승인 |
| 반려 | 기술적/경제적 타당성 부족, 대안 검토 필요 |
| 보류 | 추가 정보/분석 필요 |

## 8.5 ECN (설계 변경 통보) 발행

CCB 승인 후, 기술팀이 ECN을 발행하여 변경을 공식 지시한다.

### 8.5.1 ECN 필수 기재 사항

| 항목 | 내용 |
|------|------|
| ECN 번호 | ECN-YYYY-NNN 형식 |
| 연관 ECR 번호 | 변경 요청 추적 |
| 변경 대상 CI / 도면 | 변경 대상 목록, 현재/변경 후 개정번호 |
| 변경 내용 (상세) | 변경 전/후 비교 (도면, BOM, 사양) |
| 적용 시점 | 즉시, 재고 소진 후, 특정 로트부터 |
| 호환성 정보 | 기존 제품과의 호환성/교환성 여부 |
| 배포처 | 관련 부서 목록 |

### 8.5.2 변경 실행 및 검증

| 단계 | 활동 | 담당 |
|------|------|------|
| 1 | 도면/BOM/사양서 개정, ERP 반영 | 기술팀 + IT |
| 2 | 공정 문서(작업표준, QC 공정표) 변경 | 생산팀 |
| 3 | 금형/치공구 변경 (해당 시) | 금형팀 |
| 4 | 검사 기준 변경, 측정기 교정 확인 | 품질팀 |
| 5 | 변경 후 초도품 제작 및 검증 | 생산팀 + 품질팀 |
| 6 | FAI 재실시 (Class I 변경 시) | 품질팀 |
| 7 | 변경 완료 확인 및 ECN 종결 | 형상관리 담당자 |

> **CAR-004 시정조치**: 모든 ECN 발행 시 ERP 시스템에 도면 개정번호와 BOM 버전을 동시 갱신한다. 도면-BOM 정합성은 ECN 종결 전 반드시 검증하며, 불일치 발생 시 ECN을 종결할 수 없다.

## 8.6 긴급 변경 절차

안전/감항성 관련 또는 라인 정지를 유발하는 긴급 변경 시:

1. ECR 접수 → CCB 의장이 단독으로 긴급 승인 가능
2. 24시간 이내 ECN 발행 및 변경 실행
3. 72시간 이내 CCB 긴급 회의를 소집하여 사후 추인
4. 미추인 시 변경 철회 및 원복 조치

# 9. 형상 상태 기록 (Configuration Status Accounting)

## 9.1 형상 상태 기록의 목적

형상 항목의 현재 상태와 변경 이력을 체계적으로 기록·유지하여, 어느 시점에서든 형상 항목의 형상 상태를 정확히 파악할 수 있도록 한다.

## 9.2 형상관리 기록부

형상관리 담당자는 F-815-01 (형상관리 기록부)를 유지하며, 다음 정보를 기록한다:

| 기록 항목 | 내용 |
|----------|------|
| CI 현황 | CI 번호, 명칭, 유형, 현재 개정번호, 기준선 소속 |
| 기준선 이력 | FBL/ABL/PBL 수립 일자, 승인자, 포함 CI 목록 |
| 변경 이력 | ECR/ECN 번호, 변경 전/후 형상, 변경 일자, 승인자 |
| 현재 유효 형상 | 각 CI의 최신 유효 개정번호 및 적용 상태 |
| 편차/특채 이력 | 임시 편차(Deviation), 특채(Waiver) 이력 |

## 9.3 ERP 연계 관리

> **CAR-004 시정조치 핵심 사항**

| 요구사항 | 구현 방법 |
|----------|----------|
| 도면-BOM 정합성 | ERP 내 도면 개정번호 필드와 BOM 버전 필드 연계; 불일치 시 경고 발생 |
| 자동 동기화 | ECN 승인 시 ERP에 변경 내용 자동 반영 (또는 24시간 이내 수동 반영 후 검증) |
| 이력 추적 | ERP에서 CI별 변경 이력 조회 가능 (도면, BOM, 공정도 통합 이력) |
| 정합성 보고서 | 월 1회 도면-BOM 정합성 보고서 생성 및 형상관리 담당자 검토 |

## 9.4 형상 상태 보고

| 보고 유형 | 주기 | 내용 | 보고 대상 |
|----------|------|------|----------|
| 형상 상태 요약 | 월 1회 | CI 현황, ECR/ECN 현황, 기준선 변경 | CCB, QMR |
| 도면-BOM 정합성 | 월 1회 | ERP 도면↔BOM 버전 일치 확인 | 기술팀장, 생산팀장 |
| 변경 이력 보고 | 분기 1회 | 분기별 변경 건수, 유형별 통계, 리드타임 분석 | 경영검토 입력 |
| 고객 요청 보고 | 수시 | 고객 요청 시 해당 제품의 형상 이력 제공 | 고객 |

# 10. 형상 감사 (Configuration Audit)

## 10.1 형상 감사 유형

### 10.1.1 FCA (기능 형상 감사, Functional Configuration Audit)

| 항목 | 내용 |
|------|------|
| 목적 | 형상 항목이 승인된 기능/성능 요구사항을 충족하는지 검증 |
| 시기 | 설계 검증/확인 완료 시, FAI 연계 |
| 검증 대상 | 시험 결과 vs 사양서 요구사항, 기능 시험 기록, 시뮬레이션 결과 |
| 수행 주체 | 품질팀 (기술팀 지원) |

**FCA 확인 사항**:
- 모든 기능/성능 요구사항에 대한 시험 결과가 존재하는가?
- 시험 결과가 사양서 요구사항을 충족하는가?
- 미충족 항목에 대한 편차(Deviation) 승인이 있는가?
- 시험 조건이 적절하며 재현 가능한가?

### 10.1.2 PCA (물리적 형상 감사, Physical Configuration Audit)

| 항목 | 내용 |
|------|------|
| 목적 | 실물 제품이 승인된 기술 문서(도면, BOM, 사양서)와 정확히 일치하는지 검증 |
| 시기 | 양산 이관 전, FAI 연계, 정기(반기 1회) |
| 검증 대상 | 실물 vs 도면 치수/재질, 실물 vs BOM 구성, 마킹/표시 |
| 수행 주체 | 품질팀 (생산팀, 기술팀 지원) |

**PCA 확인 사항**:
- 실물 제품의 치수/재질/외관이 최신 도면과 일치하는가?
- 실물 부품 구성이 최신 BOM과 일치하는가?
- 제품 마킹(Part No., Rev., S/N 등)이 정확한가?
- 사용 자재의 성적서(Mill Cert.)가 사양과 일치하는가?

## 10.2 형상 감사 절차

| 단계 | 활동 | 담당 |
|------|------|------|
| 1 | 감사 계획 수립 (대상 CI, 일정, 감사원) | 품질팀장 |
| 2 | 기준 문서 확보 (최신 도면, BOM, 사양서) | 형상관리 담당자 |
| 3 | FCA 수행: 시험 결과 vs 요구사항 대조 | 품질팀 |
| 4 | PCA 수행: 실물 vs 기술 문서 대조 | 품질팀 + 생산팀 |
| 5 | 부적합 사항 기록 | 품질팀 |
| 6 | 시정조치 요구 (부적합 시) → QP-1001 연계 | 품질팀장 |
| 7 | 시정조치 이행 확인 | 품질팀 |
| 8 | 감사 보고서 작성 및 CCB/QMR 보고 | 품질팀장 |

## 10.3 형상 감사 주기

| 감사 유형 | 주기 | 비고 |
|----------|------|------|
| 정기 PCA | 반기 1회 | 양산 제품 대상 |
| FAI 연계 FCA/PCA | 초도품 검사 시 | 신규 제품, Class I 변경 후 |
| 고객 요구 감사 | 고객 요청 시 | 고객 감사 대응 |
| 특별 감사 | 중대 부적합 발생 시 | 형상 관련 부적합 발견 시 |

# 11. 항공우주 특수 요구사항

## 11.1 고객 승인 요구사항

| 요구사항 | 세부 내용 |
|----------|----------|
| 고객 승인 변경 | Class I 변경 중 고객 계약에서 승인을 요구하는 변경은 반드시 고객 서면 승인 획득 후 실행 |
| 고객 통보 | 고객에게 형상 변경 사항 및 영향을 적시에 통보 (영업팀 주관) |
| 고객 감사 대응 | 고객의 형상 감사 요청 시 형상 기록 및 실물 확인을 지원 |
| 제품 출하 시 형상 정보 | 납품 시 해당 제품의 형상 상태(적용 도면 Rev., BOM 버전)를 인도 문서에 명기 |

## 11.2 감항 영향 평가

형상 변경 시 감항성(Airworthiness)에 대한 영향을 평가한다:

| 평가 항목 | 내용 |
|----------|------|
| 감항성 영향 판정 | 모든 ECR에 대해 감항성 영향 여부를 판정 (CCB 심의 시 필수 확인 사항) |
| 안전 핵심 항목 변경 | 안전 핵심 항목(Safety-Critical Item)의 형상 변경 시 리스크 평가(QP-802) 연계 |
| 인증 영향 평가 | 변경이 기존 형식 인증(Type Certificate)에 영향을 미치는지 확인 |
| 규제당국 통보 | 감항성에 영향을 미치는 변경 시 해당 규제당국(EASA, FAA 등) 통보 요건 확인 |

## 11.3 형상 문서 보존

항공우주 요구사항에 따라 형상관리 관련 기록의 보존 기한을 다음과 같이 규정한다:

| 기록 유형 | 보존 기한 | 비고 |
|----------|----------|------|
| 형상관리 기록부 | 제품 수명 + 10년 | 최소 보존 기한, 고객 요구 시 연장 |
| ECR/ECN 기록 | 제품 수명 + 10년 | 전자/종이 이중 보관 권장 |
| CCB 회의록 | 제품 수명 + 10년 | 의결 내용 및 참석자 기록 포함 |
| 기준선 패키지 | 제품 수명 + 10년 | FBL/ABL/PBL 각각 보존 |
| 형상 감사 보고서 | 제품 수명 + 10년 | FCA/PCA 결과 포함 |
| 도면/BOM 개정 이력 | 제품 수명 + 10년 | 모든 개정판 보존 (폐기판 포함) |

> **참고**: "제품 수명"은 최종 납품된 제품의 운용 수명 종료 시점을 의미하며, 고객/규제당국이 별도 요구 시 해당 기한을 적용한다.
