# 1. 목적

본 절차서는 M&C Electronics VINA가 고객에게 제공할 제품 및 서비스에 대한 요구사항을 계약/주문 수락 전에 체계적으로 검토하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.2.3/8.2.4 요구사항에 따른 계약검토 체계 수립
- 조직이 요구사항을 충족할 능력이 있는지 확인
- 고객 요구와 조직 능력 간의 차이 해결
- 계약 변경의 체계적 관리

# 6. 계약검토 프로세스

## 6.1 계약검토 절차

> **참고**: 기존 주문 처리 절차(SAL.02.00.01, 2017-04-01)의 ERP 기반 프로세스를 유지하면서 AS9100D 항공우주 검토 항목을 추가한다.

```
고객 PO/주문 접수 (영업팀 / Kinh doanh)
    │  ← 시스템(ERP), 이메일, PO 등
    ▼
① 주문 정보 확인 및 ERP 입력
    │  ← 영업 직원이 ERP/SOP 파일에 주문 정보 입력
    │  ← 생산계획(KHSX) 및 관련 부서에 전달
    ▼
② 생산계획 부서 피드백
    │  ← 생산 일정/능력 확인 후 회신 (ERP, 이메일)
    ▼
③ 납기 확인 및 고객 소통
    │  ├── 계획대로 가능 → 고객에게 납기 확인
    │  └── 불가능 → 고객과 납기 변경 협의
    ▼
④ AS9100D 추가 검토 (항공우주 주문 시)
    │  ← 기술팀(Phát triển): 기술 요구사항, 핵심 특성, 특수공정
    │  ← 품질팀(Chất lượng): FAI 요구, 추적성, 검사 요건
    │  ← 구매팀(PUR): 자재 조달, 공급자 인증 상태
    ▼
⑤ 부서장(Trưởng Phòng) 승인
    │  ├── OK → 기록 보관 (ERP, 증빙 서류)
    │  └── NG → 보완/수정 후 재승인
    ▼
⑥ 기록 보관 (Lưu Hồ Sơ)
    ← PO, 인수인계서, 납품서, 판매요청서 등 ERP + 증빙
```

## 6.2 계약검토 항목

AS9100D 8.2.3.1에 따라 다음 사항을 검토한다:

| 검토 항목 | 검토 내용 | 검토 부서 |
|----------|----------|----------|
| **a) 고객 명시 요구사항** | 도면, 사양, 수량, 납기, 포장, 운송 조건 | 영업/기술 |
| **b) 고객 미명시 필요 요구사항** | 의도된 용도에 필요한 성능, 기능 | 기술 |
| **c) 조직 자체 요구사항** | 내부 품질 기준, 공정 능력 | 품질 |
| **d) 법적/규제 요구사항** | 적용 법규, 규격, 인증 | 품질 |
| **e) 이전 계약과의 차이** | 변경 사항 식별 및 해결 | 영업 |
| 생산 능력 | 설비, 인력, 용량 | 생산 |
| 자재 조달 | 자재 가용성, 리드타임 | 구매 |
| 특수공정 | 특수공정 능력, 자격 인증 | 기술/품질 |
| 리스크 | 계약 이행 리스크 | QMR |

## 6.3 계약검토 수준

| 계약 유형 | 검토 수준 | 승인 |
|----------|----------|------|
| 신규 고객/신규 제품 | 전체 검토 (모든 항목) | QMR 승인 |
| 기존 고객/기존 제품 신규 PO | 표준 검토 (변경 중심) | 영업팀장 승인 |
| 반복 주문 (동일 조건) | 간소화 검토 (납기/수량만) | 영업 담당자 |
| 계약 변경 | 변경 영향 검토 | 변경 규모에 따라 |

# 7. 요구사항 변경 관리

## 7.1 변경 유형

| 변경 출처 | 예시 |
|----------|------|
| 고객 변경 요청 | 사양 변경, 수량 변경, 납기 변경 |
| 내부 변경 필요 | 공정 변경, 대체 자재, 설계 변경 |
| 법규/규격 변경 | 적용 규격 개정, 법규 변경 |

## 7.2 변경 처리 절차

| 단계 | 활동 | 책임자 |
|------|------|--------|
| 1 | 변경 요청 접수/식별 | 영업팀 |
| 2 | 변경 영향 평가 (기술/생산/품질/비용/납기) | 관련 부서 |
| 3 | 변경 수용 여부 결정 | 영업팀장 (중대 변경: QMR) |
| 4 | 고객과 변경 합의 | 영업팀 |
| 5 | 관련 문서 갱신 (도면, 작업표준, 검사기준 등) | 해당 부서 |
| 6 | 관련 인원에게 변경 전달 | 영업팀/품질팀 |

# 8. 항공우주 특수 요구사항

## 8.1 항공우주 계약검토 추가 항목

| 항목 | 검토 내용 |
|------|----------|
| AS9100D 인증 요구 | 고객이 AS9100D 인증을 요구하는지 확인 |
| FAI 요구 | 초도품 검사 범위 및 시기 확인 |
| 특수공정 승인 | 고객 승인 특수공정 요구 확인 (NADCAP 등) |
| 핵심 특성 | 고객 지정 핵심 특성(KC) 식별 |
| 추적성 범위 | 로트/배치/시리얼 추적 요구 수준 |
| 위임검사 | 고객 위임검사원의 접근 조건 |
| 변경 통보 | 공정/자재/공급자 변경 시 고객 사전 승인 요구 |
| 기록 보존 | 고객 요구 기록 보존 기간 (표준보다 긴 경우) |
| 수출 통제 | ITAR, EAR 등 수출 규제 해당 여부 |

## 8.2 리스크 기반 계약검토

신규 또는 고위험 계약에 대해:
- 기술적 리스크 (신규 기술, 엄격한 공차, 특수 재질)
- 일정 리스크 (짧은 납기, 병행 개발)
- 공급망 리스크 (특수 자재, 단일 공급원)
- 재무 리스크 (고 투자, 장기 계약)
