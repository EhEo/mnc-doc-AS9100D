# 1. 목적

본 절차서는 Công ty TNHH M&C ELECTRONICS VIỆT NAM (이하 M&C Electronics VINA)의 제품, 공정, 자재, 설비 및 인원에 대한 변경을 체계적으로 관리하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 8.5.6 요구사항에 따른 생산 및 서비스 제공의 변경 관리
- 변경이 요구사항에 대한 지속적 적합성에 미치는 영향을 검토하고 통제
- 기존 4M(Man, Machine, Material, Method) 변경관리 체계를 항공우주 요구사항 수준으로 격상
- ECR(Engineering Change Request) → ECN(Engineering Change Notice) 프로세스를 통한 변경 통제 체계 확립
- 형상관리(QP-803) 및 리스크 관리(QP-802)와 연계된 통합 변경관리 실현
- 고객 사전 승인, 규제기관 통보 등 항공우주 고유 변경 요건 충족

> **참고**: 설계/개발 변경의 기술적 세부 사항은 QP-807(설계/개발 관리)에서, 형상 기준선 변경 통제는 QP-803(형상관리)에서 다룬다. 본 절차서는 **생산/서비스 제공 단계의 변경 통제(8.5.6)** 전반을 규정한다.

# 2. 적용 범위

본 절차서는 M&C Electronics VINA의 항공우주 제품 생산 및 서비스 제공 과정에서 발생하는 모든 변경에 적용한다.

| 구분 | 적용 대상 |
|------|----------|
| 설계 변경 | 도면, 사양서, BOM, 소프트웨어 변경 (QP-807/QP-803 연계) |
| 공정 변경 | 제조 공정, 작업 방법, 공정 순서, 공정 파라미터 변경 |
| 자재 변경 | 원부자재 변경, 대체 자재 적용, 공급업체 변경 |
| 설비 변경 | 생산 설비, 금형, 지그, 치구, 검사 장비 변경/이전 |
| 인원 변경 | 특수 공정 작업자, 검사원, 핵심 기술인력 변경 |

**적용 부서**: 개발(Phát triển), 생산(Sản xuất), 금형(Khuôn), 생산계획(KHSX), 품질(Chất lượng), 구매(PUR), 영업(Kinh doanh)

**제외 사항**: 일상적 설비 보전(예방 정비 계획에 따른 정기 교체), 동일 규격 소모품 교체는 본 절차의 변경 대상에서 제외한다.

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 4M 변경 | Man(인원), Machine(설비), Material(자재), Method(공정/방법)의 변경. 기존 ISO/IATF 체계에서 사용하던 변경 분류 방식 |
| ECR (Engineering Change Request) | 변경 요청서 — 변경의 필요성, 근거, 예상 영향을 기술하여 변경을 요청하는 문서 |
| ECN (Engineering Change Notice) | 변경 통보서 — 변경 심의/승인 후 변경 내용 및 실행 지시를 공식 통보하는 문서 |
| PCN (Process Change Notice) | 공정 변경 통보서 — 공정 변경 시 고객 및 관련 부서에 통보하는 문서 |
| Deviation | 일시적 이탈 허가 — 아직 생산되지 않은 제품에 대해 규정된 요구사항으로부터 이탈을 사전 허가하는 것 |
| Concession (특채) | 규정된 요구사항에 부적합한 제품의 사용 또는 출하를 허가하는 것 |
| 형상 영향 변경 | 형상 기준선(Baseline)에 영향을 미치는 변경으로, CCB 심의가 필수인 변경 |
| 변경 유효성 확인 | 변경 실행 후 변경된 제품/공정이 요구사항을 충족하는지 확인하는 활동 |
| Class I 변경 | 형태, 적합, 기능(Form, Fit, Function)에 영향을 미치는 주요 변경 |
| Class II 변경 | 형태, 적합, 기능에 영향이 없는 경미한 변경 |

# 6. 변경 분류

## 6.1 변경 유형별 분류

### 6.1.1 설계 변경 (Design Change)
- 도면, 사양서, BOM, 소프트웨어의 기술적 변경
- 형상 기준선 영향 여부에 따라 Class I/II 분류
- ECR → CCB 심의(Class I) → ECN 발행 → 실행

### 6.1.2 공정 변경 (Process Change)
- 제조 공정 순서, 방법, 파라미터 변경
- 작업 표준서(WI), 공정도, QC 공정도 변경 수반
- 특수 공정 변경 시 재인증(Requalification) 필요

### 6.1.3 자재 변경 (Material Change)
- 원부자재 규격, 등급, 제조사 변경
- 대체 자재(Alternate Material) 적용
- 공급업체 변경 (QP-808 외부공급자 관리 연계)

### 6.1.4 설비 변경 (Machine/Equipment Change)
- 생산 설비 신규 도입, 교체, 이전(Relocation)
- 금형, 지그, 치구의 신규 제작 또는 대폭 수리
- 검사/시험 장비 변경

### 6.1.5 인원 변경 (Man Change)
- 특수 공정(용접, 솔더링, 도금 등) 인증 작업자 변경
- 핵심 검사원 변경
- 신규 작업자 배치 시 자격 확인(QP-703 연계)

## 6.2 변경 등급 분류 기준

| 등급 | 기준 | 예시 |
|------|------|------|
| Class I (주요 변경) | 형태, 적합, 기능(FFF)에 영향 / 호환성에 영향 / 안전·신뢰성에 영향 / 고객 요구사항에 영향 | 치수 공차 변경, 재질 변경, 공정 추가/삭제, 핵심 공급업체 변경 |
| Class II (경미한 변경) | FFF 영향 없음 / 호환성 유지 / 안전·신뢰성 영향 없음 | 도면 표기법 수정, 비핵심 치수 공차 완화, 작업 편의 개선 |

# 7. 변경 요청 및 검토 (ECR→ECN 프로세스)

## 7.1 변경 요청 (ECR)

### 7.1.1 변경 요청 발의
- **발의 주체**: 모든 부서 임직원이 변경 필요성을 인지한 경우 ECR 발행 가능
- **변경 요청 사유**:
  - 고객 요구사항 변경
  - 설계 개선 또는 오류 시정
  - 공정 개선 또는 문제 해결
  - 원가 절감
  - 규격/규제 변경 대응
  - 공급업체 사유 (단종, 품질 이슈)
  - 내부/외부 심사 부적합 시정

### 7.1.2 ECR 필수 기재 사항

| 항목 | 내용 |
|------|------|
| ECR 번호 | ECR-YYYY-NNN (연도-일련번호) |
| 변경 대상 | 해당 도면/사양서/공정/자재/설비 식별번호 |
| 변경 사유 | 변경이 필요한 구체적 이유 |
| 현재 상태 (As-Is) | 현재 규격/공정/자재 상태 기술 |
| 변경 후 상태 (To-Be) | 변경 후 목표 규격/공정/자재 상태 기술 |
| 예상 영향 | 품질, 납기, 원가, 안전, 형상 영향 예비 평가 |
| 변경 분류 | 설계/공정/자재/설비/인원 |
| 등급 (Class I/II) | 예비 등급 분류 (검토 시 확정) |
| 요청일 / 요청자 | 일자, 부서, 성명 |

### 7.1.3 ECR 접수 및 예비 검토
1. 기술팀(설계 변경) 또는 생산팀(공정/4M 변경)에서 ECR 접수
2. ECR 완전성 확인 (필수 항목 기재 여부)
3. 변경 등급 예비 분류 (Class I/II)
4. Class I 판정 시 → CCB 심의 일정 배정
5. Class II 판정 시 → 해당 부서장 검토로 진행

## 7.2 변경 심의

### 7.2.1 Class I 변경 — CCB 심의
- CCB 소집 (QP-803 §8 형상 관리 참조)
- 다기능 팀(Cross-Functional Team) 참여: 기술, 생산, 품질, 구매, 생산계획
- 심의 항목:
  - 기술적 타당성
  - 영향 분석 결과 (§8 참조)
  - 고객 승인 필요 여부
  - 규제 영향
  - 형상 기준선 영향
  - 실행 일정 및 전환 방안
- 심의 결과: 승인 / 조건부 승인 / 반려 / 보완 요청

### 7.2.2 Class II 변경 — 간이 검토
- 해당 부서장 + 품질팀장 검토
- 기술적 타당성 및 품질 영향 확인
- FFF 영향 없음 재확인 후 승인

## 7.3 ECN 발행

| 항목 | 내용 |
|------|------|
| ECN 번호 | ECN-YYYY-NNN (연도-일련번호), ECR 번호 참조 |
| 변경 내용 | 구체적 변경 사항 (도면/BOM/공정/규격 변경 내용) |
| 영향 받는 문서 | 개정 대상 도면, 사양서, 작업 표준서, QC 공정도 목록 |
| 적용 시점 (Effectivity) | 로트 번호, 시리얼 번호, 또는 일자 기준 전환 시점 |
| 재고 처리 방안 | 기존 재고/반제품/완제품 처리 방법 |
| FAI 재수행 여부 | Class I 변경 시 FAI 재수행 필요 여부 판정 (§10 참조) |
| 고객 통보/승인 | 고객 사전 승인 완료 여부, 승인 문서 번호 |
| 배포 대상 | ECN 배포 부서 및 확인 서명 |

# 8. 변경 영향 분석

## 8.1 영향 분석 범위

모든 Class I 변경 및 제품 적합성에 영향을 줄 수 있는 Class II 변경에 대해 영향 분석을 수행한다.

| 분석 영역 | 분석 내용 |
|----------|----------|
| 제품 적합성 | 제품이 규격/사양 요구사항을 지속 충족하는지 여부 |
| 형상 영향 | 형상 기준선 변경 여부 (QP-803 연계) |
| 안전 영향 | 제품/서비스 안전에 미치는 영향 (QP-804 연계) |
| 호환성/교환성 | 기존 제품과의 호환성, 교환 가능 여부 |
| 공정 능력 | 변경 후 공정 능력(Cpk) 유지 가능 여부 |
| 납기 영향 | 변경 소요 시간, 생산 중단 기간, 납기 영향 |
| 원가 영향 | 자재비, 가공비, 검사비 등 원가 변동 |
| 공급망 영향 | 공급업체 변경, 리드 타임 변동 |
| 규제/인증 영향 | 항공 당국, 고객 인증 재취득 필요 여부 |

## 8.2 리스크 평가 (QP-802 연계)

변경에 따른 리스크는 QP-802(운영 리스크 관리) 절차에 따라 평가한다.

| 리스크 수준 | 심각도 × 발생도 | 조치 |
|------------|----------------|------|
| 고위험 (High) | RPN ≥ 100 또는 심각도 ≥ 8 | 별도 리스크 완화 계획 수립 필수, CCB 심의 |
| 중위험 (Medium) | 40 ≤ RPN < 100 | 리스크 완화 조치 실행 후 승인 |
| 저위험 (Low) | RPN < 40 | 표준 변경 프로세스 진행 |

## 8.3 FMEA 업데이트

- Class I 변경 시 해당 공정/설계 FMEA 업데이트 필수
- 변경에 따른 신규 고장 모드(Failure Mode) 식별 및 대책 수립
- FMEA 업데이트 결과를 ECN에 첨부

# 10. 변경 유효성 확인

## 10.1 유효성 확인 방법

변경 실행 후 변경된 제품/공정이 요구사항을 충족하는지 확인한다.

| 확인 방법 | 적용 조건 | 기준 |
|----------|----------|------|
| FAI 재수행 | Class I 변경 (FFF 영향) | AS9102 요구사항 충족 (QP-811) |
| 부분 FAI | Class I 변경 (일부 특성만 영향) | 변경 영향 특성에 대해서만 FAI |
| 초도품 검사 (간이) | Class II 변경 | 변경 항목에 대한 전수 검사 |
| 공정 능력 확인 | 공정 변경 | Cpk ≥ 1.33 확인 |
| 기능 시험 | 기능 영향 변경 | 고객 규격 시험 항목 합격 |

## 10.2 FAI 재수행 판단 기준 (QP-811 연계)

AS9102에 따라 다음 경우 FAI를 재수행한다:

| FAI 재수행 필요 | FAI 재수행 불필요 |
|----------------|-----------------|
| 설계(도면/사양) 변경으로 FFF 영향 | 도면 표기법 수정 등 비기술적 변경 |
| 제조 공정의 중대 변경 | 동일 공정 내 경미한 파라미터 조정 |
| 제조 장소(사업장) 이전 | 동일 사업장 내 라인 이동 |
| 2년 이상 생산 중단 후 재개 | 정기적 생산 유지 상태 |
| 자재/원자재 변경 (FFF 영향) | 동일 규격 동일 제조사 로트 변경 |

## 10.3 유효성 확인 보고

- 유효성 확인 결과를 ECN에 기록하고 변경 이력에 반영
- 부적합 발생 시 QP-818(부적합 출력 관리) 및 QP-1001(시정조치) 연계 처리
- 유효성 확인 완료 후 변경 종결(Close-out) 처리

# 11. 항공우주 특수 요구사항

## 11.1 고객 사전 승인 (Customer Prior Approval)

항공우주 고객의 변경 관련 요구사항:

- 고객 계약/PO에 명시된 변경 통보/승인 요건을 **의무적으로** 준수
- Class I 변경 시 고객에게 변경 요청서(ECR) 또는 PCN을 제출하고, 고객의 서면 승인을 받은 후에만 변경 실행
- 고객이 요구하는 양식 및 절차가 있는 경우 고객 양식을 우선 사용
- 고객 승인 대기 기간 동안 변경 전 상태로 생산 유지
- **고객 미승인 시 변경 실행 절대 금지**

## 11.2 형상관리 연계 (QP-803)

- 모든 Class I 변경은 형상관리 프로세스와 연동하여 관리
- 형상 기준선(Baseline) 변경 시 CCB 심의 필수
- ECN 발행과 동시에 형상 상태 기록(CSA) 업데이트
- 형상 감사(Configuration Audit) 시 변경 이력 검증 포함

## 11.3 규제기관 통보

- 항공 당국(EASA, FAA 등) 인증 대상 제품의 변경 시 규제 요건에 따른 사전 통보/승인
- 고객을 통한 규제기관 통보가 필요한 경우 영업팀이 고객과 협의하여 처리
- 규제 승인 미완료 시 변경 실행 보류

## 11.4 외부공급자 변경 통제

- 외부공급자의 제품/공정 변경에 대해서도 동일한 변경 관리 절차 적용 요구
- 공급업체는 변경 실행 전 M&C Electronics VINA에 사전 통보 의무
- 승인 공급업체 목록(ASL) 변경 시 QP-808(외부공급자 관리) 연계

## 11.5 추적성 확보

- 변경 전/후 제품에 대한 명확한 로트 번호 또는 시리얼 번호 식별
- 변경 적용 시점(Effectivity Point)을 기준으로 추적 가능한 이력 유지
- 변경 이력은 제품 수명주기 종료 후 최소 10년 보존 (또는 고객 요구 기간)

# 12. 긴급 변경(Deviation/Concession) 처리

## 12.1 Deviation (일시적 이탈 허가)

생산 전, 규정된 요구사항으로부터의 이탈이 불가피한 경우:

| 항목 | 요구사항 |
|------|---------|
| 발의 | 해당 부서에서 이탈 필요성 및 근거를 기술하여 Deviation Request 발행 |
| 범위 | 적용 로트/수량/기간을 명확히 한정 |
| 영향 분석 | 이탈이 제품 안전, 기능, 적합성에 미치는 영향 분석 |
| 승인 | QMR 승인 필수, Class I 수준의 이탈 시 고객 승인 필수 |
| 기록 | Deviation 번호, 적용 범위, 승인 내역 기록 보존 |
| 기한 | 적용 기한 만료 시 자동 종료, 연장 시 재승인 필요 |

## 12.2 Concession (특채)

이미 생산된 부적합 제품의 사용/출하 허가:

| 항목 | 요구사항 |
|------|---------|
| 발의 | 부적합 발생 시 QP-818에 따라 부적합 처리 후 특채 검토 |
| 조건 | 안전, 기능, 신뢰성에 영향이 없음을 입증 |
| 고객 승인 | **항공우주 제품은 고객 서면 승인 필수** (자체 특채 불가) |
| 식별 | 특채 제품에 대한 명확한 식별 표시 및 추적 |
| 기록 | Concession 번호, 승인 근거, 고객 승인 문서, 영향 분석 기록 |
| 후속 조치 | 근본 원인 분석 및 시정조치 실행 (QP-1001 연계) |

## 12.3 긴급 변경 프로세스

긴급 상황(생산 라인 정지, 납기 위험 등)으로 정규 ECR→ECN 프로세스 수행이 불가능한 경우:

1. **긴급 승인**: QMR 또는 대표이사의 구두/이메일 긴급 승인으로 변경 실행 가능
2. **사후 문서화**: 긴급 변경 실행 후 **48시간 이내** 정규 ECR/ECN을 소급 작성
3. **고객 통보**: 고객 승인 대상인 경우 즉시 고객에게 긴급 변경 사실 통보
4. **유효성 확인**: 정규 변경과 동일한 수준의 유효성 확인 수행
5. **재발 방지**: 긴급 변경 사유 분석 및 정규 프로세스로의 전환 계획 수립
