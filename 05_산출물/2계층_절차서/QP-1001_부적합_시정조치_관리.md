# 1. 목적

본 절차서는 M&C Electronics VINA에서 발생하는 모든 부적합 사항을 체계적으로 관리하고, 근본 원인 분석을 통한 효과적인 시정조치를 수행하기 위한 절차를 규정한다.

본 절차서의 목적:
- AS9100D 10.2 요구사항에 따른 부적합 및 시정조치의 체계적 관리
- 부적합의 재발 방지를 위한 근본 원인 분석 및 시정조치 실행
- 8D 방법론을 활용한 구조적 문제 해결
- 시정조치의 유효성 확인 체계 확립
- 부적합 데이터의 추세 분석을 통한 예방적 개선

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 부적합 (Nonconformity) | 요구사항의 불충족. 제품, 프로세스, 시스템에 적용 |
| 시정 (Correction) | 발견된 부적합을 제거하기 위한 즉시 조치 (응급 처치) |
| 시정조치 (Corrective Action) | 부적합의 원인을 제거하여 재발을 방지하기 위한 조치 |
| CAR (Corrective Action Request) | 시정조치 요청서 — 부적합 기록, 원인 분석, 시정조치 계획/실행/확인을 포함 |
| 8D | 8 Disciplines — 체계적 문제해결 방법론 (팀 구성 → 문제 정의 → … → 종결) |
| 근본 원인 (Root Cause) | 부적합의 발생을 유발한 근본적인 원인 |
| 봉쇄 조치 (Containment Action) | 부적합의 영향 확산을 방지하기 위한 즉각적 격리/차단 조치 |
| 유효성 확인 (Effectiveness Verification) | 시정조치가 부적합의 재발을 효과적으로 방지하는지 확인하는 활동 |
| 탈출 포인트 (Escape Point) | 부적합이 검출되었어야 하나 검출되지 못한 프로세스 단계 |
| RCCA | Root Cause and Corrective Action — 근본 원인 분석 및 시정조치 |

# 6. 부적합 관리

## 6.1 부적합 발견 및 보고

```
부적합 발견
    │
    ▼
① 즉시 보고 ──→ 구두 보고 + 부적합 보고서 작성
    │              (제품 부적합: QP-818 연계)
    ▼
② 봉쇄 조치 ──→ 부적합 제품 격리/표식
    │              영향 범위 파악 (재공품, 재고, 출하품)
    ▼
③ CAR 발행 여부 판단 ──→ 6.2 CAR 발행 기준 참조
    │
    ├── CAR 필요 → 7절 시정조치 프로세스
    │
    └── CAR 불필요 → 시정(Correction)만 실시 후 기록
```

## 6.2 CAR 발행 기준

| CAR 발행 필수 | CAR 선택적 |
|-------------|-----------|
| Major 부적합 (시스템적/반복적) | Minor 부적합 (단발적/경미) |
| 고객 불만/반품 | 경미한 공정 이탈 (즉시 시정 가능) |
| 내부심사/인증심사 NCR | 단순 서류 미비 |
| 안전 관련 부적합 | OFI (관찰사항) |
| 동일 부적합 3회 이상 반복 | |
| 고객이 CAR을 요구한 경우 | |
| 위조 부품 의심/발견 | |

## 6.3 부적합 분류

| 분류 | 정의 | CAR 등급 |
|------|------|---------|
| **Critical** | 안전에 영향을 미치거나, 규제 요구사항 위반 | Major CAR (즉시 조치) |
| **Major** | 시스템적 실패 또는 제품 기능에 직접 영향 | Major CAR |
| **Minor** | 단발적 일탈, 기능에 간접적 영향 | Minor CAR |
| **OFI** | 부적합은 아니나 개선 권고 | CAR 불필요 |

# 7. 시정조치 관리 (CAR)

## 7.1 CAR 프로세스 흐름

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  1. 부적합    │     │  2. CAR 발행 │     │  3. 봉쇄 조치│
│  발견/보고    │────→│  (품질팀)    │────→│  (담당 부서) │
└──────────────┘     └──────────────┘     └──────────────┘
                                                │
                         ┌──────────────────────┘
                         ▼
                  ┌──────────────┐
                  │  4. 근본 원인 │
                  │  분석 (RCA)   │
                  └──────────────┘
                         │
                         ▼
                  ┌──────────────┐     ┌──────────────┐
                  │  5. 시정조치  │────→│  6. 시정조치  │
                  │  계획 수립    │     │  실행         │
                  └──────────────┘     └──────────────┘
                                              │
                                              ▼
                                       ┌──────────────┐
                                       │  7. 유효성   │
                                       │  확인        │
                                       └──────────────┘
                                              │
                                              ▼
                                       ┌──────────────┐
                                       │  8. CAR 종결 │
                                       └──────────────┘
```

## 7.2 CAR 발행

CAR 발행자는 다음 정보를 기재하여 품질팀에 제출한다:

| 항목 | 내용 |
|------|------|
| CAR 번호 | 자동 부여 (CAR-YYYY-NNN) |
| 발행일 | 부적합 발견일 |
| 발행자 | 부적합 발견자/보고자 |
| 부적합 출처 | 내부심사/검사/고객불만/공급자 등 |
| 부적합 상세 | 무엇이, 어디서, 언제, 어떻게 발견되었는지 객관적 기술 |
| 관련 규격/절차 | 불충족된 AS9100D 조항, 절차서, 사양 등 |
| 증거 자료 | 사진, 검사 데이터, 기록 사본 등 |
| CAR 등급 | Major / Minor |
| 담당 부서 | 시정조치 책임 부서 |

## 7.3 봉쇄 조치 (Containment)

담당 부서장은 CAR 접수 후 **24시간 이내**에 봉쇄 조치를 실행한다.

| 봉쇄 조치 유형 | 내용 |
|--------------|------|
| 부적합품 격리 | 부적합 제품의 물리적 격리 및 "부적합" 표식 |
| 재고 점검 | 동일 로트/배치의 재고 전수 확인 |
| 재공품 확인 | 생산 라인의 해당 공정 재공품 확인 |
| 출하품 확인 | 출하 완료 제품 중 영향 범위 파악 → 고객 통보 필요 여부 판단 |
| 공정 중단 | 부적합 원인이 공정에 있는 경우 해당 공정 일시 중단 |

## 7.4 근본 원인 분석 (Root Cause Analysis)

### 7.4.1 분석 도구

| 분석 도구 | 적용 |
|----------|------|
| 5 Why 분석 | 기본 분석 도구 — 모든 CAR에 적용 |
| 특성요인도 (Fishbone/Ishikawa) | 복잡한 부적합, 다수 원인 가능성 |
| FTA (Fault Tree Analysis) | 안전 관련 부적합, 시스템 고장 |
| FMEA (Failure Mode & Effects Analysis) | 공정/설계 관련 부적합 |
| 시계열 분석 (Timeline Analysis) | 사건 경과 파악 필요 시 |

### 7.4.2 근본 원인 분류

| 원인 범주 | 예시 |
|----------|------|
| 인적 요인 (Man) | 교육 부족, 절차 미숙지, 부주의 |
| 장비 요인 (Machine) | 장비 고장, 교정 불량, 공구 마모 |
| 자재 요인 (Material) | 자재 불량, 규격 미달, 위조 부품 |
| 방법 요인 (Method) | 절차 미비, 작업표준 부적합, 공정 설계 오류 |
| 환경 요인 (Environment) | 온습도 부적합, 청정도 미달, 진동/충격 |
| 측정 요인 (Measurement) | 측정 오류, 장비 정밀도 부족, MSA 불합격 |

### 7.4.3 탈출 포인트 분석 (Escape Point Analysis)

AS9100D는 부적합이 **검출되었어야 할 프로세스 단계**를 식별하도록 요구한다:

1. 부적합이 발생한 프로세스 단계 식별
2. 부적합이 검출되어야 했던 검사/검증 단계 식별
3. 검출 실패 원인 분석
4. 검출 체계 개선 조치 수립

## 7.5 시정조치 계획

담당 부서장은 근본 원인 분석 결과에 기반하여 시정조치 계획을 수립한다.

# 8. 8D 문제해결 방법론

고객 불만, 반복 부적합, 복잡한 문제에 대해 8D (Eight Disciplines) 방법론을 적용한다.

## 8.1 8D 단계

| 단계 | 활동 | 산출물 |
|------|------|--------|
| **D0** | 문제 인식 및 긴급 대응 조치 (ERA) | 긴급 대응 보고 |
| **D1** | 팀 구성 — 관련 전문가로 다기능팀 편성 | 팀 구성표 |
| **D2** | 문제 정의 — IS/IS NOT 분석, 영향 범위 정량화 | 문제 기술서 |
| **D3** | 봉쇄 조치 (ICA) — 고객 보호를 위한 임시 조치 | 봉쇄 조치 기록 |
| **D4** | 근본 원인 분석 — 5 Why, 특성요인도, 검증 실험 | 근본 원인 보고 |
| **D5** | 시정조치 선택 — 영구 시정조치(PCA) 결정 및 검증 | 시정조치 계획 |
| **D6** | 시정조치 실행 — PCA 적용 및 봉쇄 조치 해제 | 실행 기록 |
| **D7** | 재발 방지 — 시스템/절차/교육 개선, 유사 공정 확장 적용 | 재발 방지 대책 |
| **D8** | 종결 및 팀 격려 — 유효성 확인, 교훈 공유 | 8D 최종 보고서 |

## 8.2 8D 적용 기준

| 8D 적용 필수 | 일반 CAR 처리 |
|-------------|-------------|
| 고객 불만/반품 (고객이 8D 요구 시) | 내부 발견 Minor 부적합 |
| 안전 관련 부적합 | 단순 절차 미준수 |
| 동일 부적합 3회 이상 반복 | 단발적 부적합 |
| 다부서 협력 필요한 복잡한 문제 | 단일 부서 해결 가능 |

# 9. 항공우주 특수 요구사항

## 9.1 적시 보고 (Timely Reporting)

AS9100D는 다음 경우 **적시에(timely)** 고객 및 관련 당사자에게 보고하도록 요구한다:

| 보고 대상 | 보고 시점 | 보고 내용 |
|----------|----------|----------|
| 부적합 제품 납품 발견 시 | 발견 즉시 (24시간 이내) | 부적합 내용, 영향 범위, 봉쇄 조치, 시정조치 계획 |
| 위조 부품 의심/발견 시 | 발견 즉시 | 부품 정보, 발견 경위, 공급자 정보 |
| 안전 관련 부적합 | 발견 즉시 | 안전 영향 평가, 즉각 조치 사항 |
| 인증기관 보고 필요 사안 | 해당 기관 요구 기한 | 인증 관련 중대 부적합 |

## 9.2 유사 제품/공정 확장 (Flow-Down)

시정조치 수립 시, 부적합의 원인이 다른 제품, 공정, 부서에도 존재할 수 있는지 반드시 검토한다:

1. 동일 자재/공급자를 사용하는 다른 제품 확인
2. 동일 공정/장비를 사용하는 다른 생산 라인 확인
3. 유사 설계/사양을 적용하는 다른 프로젝트 확인
4. 확장 적용이 필요한 경우 별도 조치 계획 수립

## 9.3 고객 승인 (Customer Approval)

다음 경우 시정조치 또는 처분에 대해 고객 승인을 받아야 한다:

| 고객 승인 필요 사항 | 비고 |
|-------------------|------|
| Use-as-is (특채 사용) 결정 | 고객 사양 불충족 제품 |
| Repair (수리) 허용 | 수리 방법 및 기준에 대한 승인 |
| 부적합 납품 후 회수/교환 계획 | 출하 후 발견된 부적합 |
| 공정 변경을 수반하는 시정조치 | 고객 승인 공정의 변경 |

## 9.4 위조 부품 관련 부적합

위조 부품이 의심되거나 발견된 경우:
1. 해당 부품 즉시 격리 및 "의심 위조품" 표식
2. 24시간 이내 품질팀장 및 QMR에게 보고
3. 고객, 인증기관, 관련 정부기관에 통보
4. 해당 공급자로부터의 모든 납품 부품 검토
5. GIDEP (Government-Industry Data Exchange Program) 등에 보고 (해당 시)
6. 위조 부품 관련 기록은 무기한 보존

# 10. 유효성 확인 및 종결

## 10.1 유효성 확인 방법

| 확인 방법 | 적용 |
|----------|------|
| 데이터 모니터링 | 시정조치 후 일정 기간 동일 부적합 미발생 확인 |
| 후속 심사 | Major CAR의 경우 현장 확인 (QP-903 연계) |
| 검사 데이터 비교 | 시정조치 전후 불량률/검사 데이터 비교 |
| 프로세스 확인 | 변경된 절차/공정의 실행 상태 확인 |

## 10.2 유효성 확인 기준

| 기준 | 내용 |
|------|------|
| 재발 여부 | 유효성 확인 기간 중 동일 부적합 미발생 |
| 근본 원인 제거 | 식별된 근본 원인이 효과적으로 제거되었음을 증거로 확인 |
| 탈출 포인트 개선 | 검출 체계가 강화되어 유사 부적합 검출 가능 확인 |
| 확장 적용 | 유사 제품/공정에 대한 시정조치도 효과적으로 적용 확인 |

## 10.3 CAR 종결

품질팀장이 유효성을 확인하고 QMR이 승인 시 CAR이 종결된다.

| CAR 등급 | 종결 승인 | 종결 조건 |
|---------|----------|----------|
| Major | QMR 승인 | 시정조치 완료 + 유효성 확인 + 고객 승인(해당 시) |
| Minor | 품질팀장 승인 | 시정조치 완료 + 유효성 확인 |
| 고객 CAR | 고객 승인 | 고객의 8D 보고서 승인 |

# 11. 데이터 분석 및 추세 관리

## 11.1 부적합 데이터 분석

품질팀장은 부적합 데이터를 월간/분기별로 분석하여 경영검토에 보고한다.

| 분석 항목 | 분석 방법 |
|----------|----------|
| 부적합 건수 추세 | 월별/분기별 추세 차트 |
| 부적합 유형별 분포 | 파레토 분석 (Pareto Chart) |
| 부서별/공정별 분포 | 부서/공정별 부적합 건수 비교 |
| 반복 부적합 | 동일 원인 반복 발생 건 식별 |
| CAR 처리 기한 준수율 | 기한 내 완료율 추적 |
| 시정조치 유효성 | 유효성 확인 합격률 |

## 11.2 KPI

| 지표 | 목표 | 보고 주기 |
|------|------|----------|
| CAR 기한 내 완료율 | ≥ 95% | 월간 |
| 시정조치 유효성 합격률 | ≥ 90% | 분기 |
| 동일 부적합 재발률 | ≤ 5% | 분기 |
| 고객 불만 대응 기한 준수율 | 100% | 월간 |

