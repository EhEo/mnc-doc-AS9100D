# 1. 목적

본 지침서는 QP-706 (문서화된 정보 관리 절차서)의 하위 문서로서, M&C Electronics VINA의 DMS(문서관리시스템)를 사용하여 품질경영시스템 문서를 등록, 검색, 개정, 배포, 보존하는 구체적인 실무 방법을 규정한다.

본 지침서의 주요 목적:
- DMS 사용자가 문서를 정확하고 효율적으로 관리할 수 있도록 표준 절차 제공
- AS9100D 7.5 요구사항에 부합하는 문서화된 정보의 체계적 관리
- 최신판 문서의 가용성 보장 및 구판/폐지판의 의도하지 않은 사용 방지
- 문서 보안 및 접근 권한의 적절한 관리

본 지침서는 M&C Electronics VINA의 DMS에 등록 및 관리되는 모든 문서화된 정보에 적용한다.

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| DMS | 문서관리시스템 (Document Management System), 전자적으로 문서를 관리하는 시스템 |
| 문서화된 정보 | 조직이 관리하고 유지해야 하는 정보 및 그 매체 (AS9100D 7.5) |
| 통제본 (Controlled Copy) | 개정 시 자동으로 최신판이 배포되는 관리 대상 문서 |
| 비통제본 (Uncontrolled Copy) | 개정 시 갱신 의무가 없는 참조용 문서 |
| 메타데이터 | 문서의 속성 정보 (문서번호, 제목, 개정번호, 작성자, 승인일 등) |
| 구판 (Obsolete) | 최신 개정판으로 대체되어 더 이상 유효하지 않은 이전 버전 |
| 폐지판 (Void) | 공식적으로 폐지된 문서 |
| 접근 권한 | DMS 내 문서에 대한 조회, 편집, 승인, 삭제 등의 사용자별 권한 |

# 5. DMS 접근 및 로그인

## 5.1 접근 방법

| 항목 | 내용 |
|------|------|
| 접근 URL | http://[내부서버주소]/dms (사내 네트워크에서만 접근 가능) |
| 지원 브라우저 | Chrome 최신판 (권장), Edge 최신판 |
| 외부 접근 | VPN 연결 후 접근 (IT팀 VPN 설정 필요) |
| 모바일 접근 | 조회만 가능 (등록/개정은 PC에서만 수행) |

## 5.2 계정 발급

1. 신규 사용자는 소속 부서장의 DMS 계정 신청서(F-706-05)를 작성하여 IT팀에 제출한다.
2. IT팀은 부서장 승인 확인 후 1영업일 이내에 계정을 발급한다.
3. 초기 비밀번호는 시스템에서 자동 생성되며, 최초 로그인 시 변경을 요구한다.
4. 비밀번호 요건: 8자 이상, 영문 대소문자 + 숫자 + 특수문자 조합

## 5.3 로그인 및 보안

1. 사번(Employee ID)과 비밀번호로 로그인한다.
2. 비밀번호는 90일마다 변경한다 (시스템 자동 알림).
3. 5회 연속 로그인 실패 시 계정이 잠기며, IT팀에 해제를 요청한다.
4. 타인에게 계정 정보를 공유하지 않는다.
5. 로그아웃하지 않고 자리를 비울 경우 30분 후 자동 로그아웃된다.

# 6. 문서 등록 방법

## 6.1 문서번호 부여 규칙

| 계층 | 문서번호 형식 | 예시 | 설명 |
|------|-------------|------|------|
| 1계층 | QM-001 | QM-001 | 품질매뉴얼 (1건) |
| 2계층 | QP-{조항}{순번} | QP-818 | 절차서, 8.1항 8번째 |
| 3계층 지침서 | WI-{상위조항}-{순번} | WI-811-01 | 지침서, QP-811 하위 1번째 |
| 3계층 양식 | QF-{상위조항}-{순번} | QF-810-03 | 양식, QP-810 하위 3번째 |
| 외부 문서 | EXT-{연도}-{순번} | EXT-2026-001 | 외부 출처 문서 |

## 6.2 문서 등록 절차

### 단계 1: 문서 작성 완료

1. 문서 작성자가 문서를 최종 작성한다.
2. 문서에 헤더 정보(문서번호, 제목, 개정번호, 작성일)를 포함한다.

### 단계 2: 검토 및 승인

1. 작성자가 DMS에서 "문서 등록 요청"을 선택한다.
2. 검토자(부서장 또는 지정 검토자)에게 검토 요청이 자동 발송된다.
3. 검토자가 검토 후 "승인" 또는 "반려(사유 기재)"를 선택한다.
4. 승인권자(품질책임자 또는 지정 승인권자)가 최종 승인한다.

### 단계 3: 메타데이터 입력

DMS 등록 시 다음 메타데이터를 입력한다:

| 필드 | 필수/선택 | 입력 내용 |
|------|----------|----------|
| 문서번호 | 필수 | 문서번호 부여 규칙에 따라 입력 |
| 문서 제목 | 필수 | 한국어 제목 + 영어 부제 |
| 문서 계층 | 필수 | 1계층/2계층/3계층/외부문서 |
| 문서 유형 | 필수 | 매뉴얼/절차서/지침서/양식/외부문서 |
| 적용 규격 조항 | 필수 | AS9100D 조항 번호 |
| 보안 등급 | 필수 | 대외비/일반/공개 |
| 작성 부서 | 필수 | 문서 작성 주관 부서 |
| 작성자 | 자동 | 로그인 계정 기준 자동 입력 |
| 개정번호 | 필수 | Rev. 0 (신규 시) |
| 개정일 | 자동 | 승인일 기준 자동 입력 |
| 상위 문서 | 선택 | 상위 절차서 문서번호 (3계층 문서 시) |
| 배포 대상 | 필수 | 배포 대상 부서/개인 |

### 단계 4: 파일 업로드

1. 승인 완료된 문서 파일을 DMS에 업로드한다.
2. 파일 형식: PDF (최종 배포용), 원본 파일(편집용, 접근 제한)
3. 파일명 규칙: `{문서번호}_Rev{개정번호}_{문서제목약칭}.pdf`
   - 예: `QP-818_Rev0_부적합출력관리.pdf`
4. 업로드 용량 제한: 단일 파일 50MB 이하

# 7. 문서 검색 및 조회

## 7.1 검색 방법

| 검색 유형 | 검색 필드 | 사용 예시 |
|----------|----------|----------|
| 문서번호 검색 | 문서번호 입력 | "QP-818" 입력하여 해당 문서 검색 |
| 제목 검색 | 문서 제목 키워드 | "부적합" 입력하여 관련 문서 검색 |
| 조항 검색 | AS9100D 조항 번호 | "8.7" 입력하여 해당 조항 관련 문서 검색 |
| 부서 검색 | 작성 부서명 | "품질팀" 입력하여 품질팀 문서 검색 |
| 복합 검색 | 여러 조건 조합 | 문서번호 + 개정번호 + 상태 조합 |

## 7.2 조회 및 다운로드

1. 검색 결과에서 문서를 선택하면 메타데이터와 미리보기를 확인할 수 있다.
2. "다운로드" 버튼으로 PDF 파일을 다운로드한다.
3. 다운로드한 파일은 비통제본으로 간주한다 (DMS 내 원본만 통제본).
4. 조회 이력은 시스템에 자동 기록된다.

## 7.3 최신판 확인

1. DMS에서 문서 상태가 "유효(Active)"인 문서만 최신판이다.
2. "구판(Obsolete)" 또는 "폐지(Void)" 상태 문서는 참조 목적으로만 열람 가능하다.
3. 문서 상세 화면에서 "개정 이력" 탭을 통해 모든 개정 버전을 확인할 수 있다.

# 8. 문서 개정 절차

## 8.1 개정 사유

문서 개정이 필요한 경우:
- 규격/법규 변경에 따른 반영
- 프로세스 변경 또는 개선
- 내부심사/외부심사 부적합 시정
- 고객 요구사항 변경
- 조직 구조 변경
- 오류 수정

## 8.2 개정 절차 흐름

```
개정 요청
    │
    ▼
개정 요청서 작성 (F-706-03)
    │
    ▼
문서관리 담당자 접수 및 타당성 검토
    │
    ▼
해당 문서 작성자/주관부서에 개정 지시
    │
    ▼
문서 개정 작성 (변경 사항 명확 표기)
    │
    ▼
검토자 검토
    │
    ▼
승인권자 승인
    │
    ▼
DMS 등록 (신규 개정판 업로드, 메타데이터 갱신)
    │
    ▼
자동 배포 (통제본 수신자에게 알림 발송)
    │
    ▼
구판 회수 (인쇄본 해당 시)
```

## 8.3 개정 표기 방법

1. 개정된 내용은 문서 내에 세로선(│) 또는 밑줄로 표기한다.
2. 개정 이력 표에 개정 사항을 요약 기재한다.
3. 개정번호를 1 증가시킨다 (Rev. 0 → Rev. 1).
4. 대폭 개정(전면 재작성) 시 개정번호를 초기화할 수 있다 (품질책임자 승인 필요).

## 8.4 구판 처리

1. DMS에서 이전 개정판은 자동으로 "구판(Obsolete)" 상태로 변경된다.
2. 구판 문서는 DMS에서 열람은 가능하나, 다운로드 시 "OBSOLETE" 워터마크가 표시된다.
3. 인쇄된 구판 통제본은 즉시 회수하여 파쇄한다.
4. 회수가 완료되면 회수 대장(F-706-04)에 기록한다.

# 9. 문서 배포 및 회수

## 9.1 전자 배포 (기본)

1. DMS 등록/개정 승인 시 배포 대상에게 자동 이메일 알림이 발송된다.
2. 알림 내용: 문서번호, 제목, 개정번호, 개정 요약, DMS 링크
3. 배포 대상자는 DMS에서 직접 문서를 조회한다.
4. DMS 내 문서가 유일한 통제본이다.

## 9.2 인쇄본 배포 (예외)

인쇄본 배포가 필요한 경우 (현장 비치용, 외부 제출용 등):

| 단계 | 내용 | 책임 |
|------|------|------|
| 1 | 인쇄본 배포 요청서 작성 | 요청자 |
| 2 | 문서관리 담당자 승인 | 문서관리 담당 |
| 3 | 인쇄 및 "통제본" 스탬프 날인 | 문서관리 담당 |
| 4 | 통제본 배포 대장(F-706-02)에 기록 | 문서관리 담당 |
| 5 | 배포처에 서명 수령 확인 | 수령자 |

## 9.3 문서 회수

1. 문서 개정 또는 폐지 시 인쇄된 통제본을 회수한다.
2. 회수 절차:
   - 문서관리 담당자가 통제본 배포 대장 기반으로 회수 대상을 확인한다.
   - 회수 통보를 발송한다 (이메일 + DMS 알림).
   - 수령자는 3영업일 이내에 인쇄본을 반납한다.
   - 반납된 인쇄본은 파쇄 처리한다.
   - 회수 완료를 대장에 기록한다.
3. 미회수 건은 품질팀장에게 보고하고, 추가 회수 조치를 취한다.

# 10. 외부 출처 문서 관리

## 10.1 외부 문서 유형

| 유형 | 예시 | 관리 방법 |
|------|------|----------|
| 국제 규격 | AS9100D, ISO 9001, SAE 규격 | DMS 등록, 개정 모니터링 |
| 고객 규격/도면 | 고객 제공 사양서, 도면, 검사 기준 | DMS 등록, 고객 개정 통보 시 갱신 |
| 법규/규제 | 항공 안전 법규, 환경 법규 | DMS 등록, 정기 검토 (연 1회) |
| 공급업체 문서 | 성적서, 인증서, MSDS | DMS 등록, 유효기간 관리 |

## 10.2 외부 문서 등록 절차

1. 외부 문서 수신 시 문서관리 담당자에게 등록을 요청한다.
2. 외부 문서번호를 부여한다 (EXT-{연도}-{순번}).
3. 메타데이터에 원본 문서번호, 발행 기관, 수신일을 기재한다.
4. 원본 파일(PDF)을 DMS에 업로드한다.
5. 해당 문서의 사용 부서를 배포 대상으로 지정한다.

## 10.3 외부 문서 개정 관리

1. 외부 문서의 개정 여부를 정기적으로 확인한다 (최소 연 1회).
2. 고객 문서는 고객으로부터 개정 통보 수신 시 즉시 갱신한다.
3. 국제 규격은 발행 기관 웹사이트 또는 구독 서비스를 통해 모니터링한다.
4. 개정된 외부 문서는 DMS에 신규 등록하고, 이전 버전을 구판 처리한다.

# 11. 문서 보안 등급 관리

## 11.1 보안 등급 분류

| 등급 | 정의 | 접근 범위 | DMS 설정 |
|------|------|----------|----------|
| **대외비 (Confidential)** | 회사 핵심 정보, 고객 기밀 문서 | 지정 인원만 접근 가능 | 개별 접근 권한 설정 |
| **일반 (Internal)** | 사내 업무용 문서 | 전 직원 조회 가능 | 부서별 접근 권한 |
| **공개 (Public)** | 외부 공유 가능 문서 | 제한 없음 | 전체 공개 |

## 11.2 보안 등급 적용

1. 문서 등록 시 작성자가 보안 등급을 지정한다.
2. 보안 등급 판단 기준:
   - 고객이 기밀로 지정한 문서 → 대외비
   - 기술 노하우, 영업 비밀 포함 문서 → 대외비
   - 일반 운영 절차, 표준 양식 → 일반
   - 회사 소개, 인증서 등 → 공개
3. 보안 등급 변경은 문서관리 담당자와 품질팀장 승인이 필요하다.

## 11.3 접근 권한 관리

| 권한 수준 | 권한 내용 | 부여 대상 |
|----------|----------|----------|
| 관리자 (Admin) | 전체 기능 (등록/개정/삭제/권한 설정) | 문서관리 담당자, IT관리자 |
| 승인자 (Approver) | 조회 + 문서 승인 | 부서장, 품질책임자 |
| 편집자 (Editor) | 조회 + 문서 작성/개정 요청 | 문서 작성 담당자 |
| 조회자 (Viewer) | 조회 + 다운로드 | 일반 사용자 |

# 14. 주의사항

## 14.1 인쇄본 관리

1. DMS에서 인쇄한 문서는 원칙적으로 비통제본이다.
2. 통제본으로 인쇄하려면 반드시 문서관리 담당자의 승인을 받고, **"통제본" 스탬프**를 날인한다.
3. "통제본" 스탬프 관리:
   - 스탬프는 문서관리 담당자만 보유하고 사용한다.
   - 스탬프 색상: 청색 (적색은 "폐지" 스탬프로 사용)
   - 스탬프 위치: 문서 첫 페이지 우측 상단
4. 스탬프가 없는 인쇄본은 모두 비통제본으로 간주한다.
5. 비통제본에는 "비통제본 - 최신판 여부를 DMS에서 확인하십시오" 표기를 권장한다.

## 14.2 문서 무단 변경 금지

1. DMS에 등록된 문서의 내용을 무단으로 변경하지 않는다.
2. 문서 변경은 반드시 개정 절차(8절)를 통해 수행한다.
3. 인쇄본에 수기 추가/수정을 하지 않는다.
4. 무단 변경 발견 시 품질팀장에게 보고하고, 시정조치를 취한다.

## 14.3 문서 폐기 주의

1. DMS에서 문서를 삭제(폐지)할 수 있는 권한은 관리자(Admin)에게만 부여한다.
2. 문서 폐지 시 품질책임자(QMR)의 승인이 필요하다.
3. 폐지된 문서는 DMS에서 "VOID" 상태로 보존하며, 완전 삭제하지 않는다.
4. 보존 기간 만료 전에는 어떠한 문서도 삭제하지 않는다.
