## 1. 목적

본 지침서는 (주)에이로텍의 항공우주 제품에 대한 형상관리(Configuration Management)를 체계적으로 수행하기 위한 구체적 작업 방법을 규정한다. 제품의 설계, 제조, 유지보수 전 수명주기에 걸쳐 형상의 식별, 통제, 기록, 감사를 일관되게 수행함을 목적으로 한다.

## 2. 적용범위

- 당사가 설계/제조하는 모든 항공우주 제품 및 부품
- 고객 도면에 의한 수주 생산 제품
- 형상관리 대상으로 지정된 지그, 치구, 금형, 시험 장비
- 제품 관련 기술 문서(도면, BOM, 사양서, 공정 문서)

## 4. 용어정의

| 용어 | 정의 |
|------|------|
| **CI (Configuration Item)** | 형상관리 대상으로 지정된 하드웨어, 소프트웨어, 문서 등의 단위 항목 |
| **CCB (Configuration Control Board)** | 형상 변경을 심의/승인하는 의사결정 위원회 |
| **Baseline (기준선)** | 특정 시점에서 공식 승인된 형상항목의 상태. 이후 변경은 공식 절차를 통해서만 수행 |
| **ECR (Engineering Change Request)** | 설계변경 요청서 — 변경의 필요성과 영향을 기술 |
| **ECN (Engineering Change Notice)** | 설계변경 통보서 — 승인된 변경의 구체적 내용과 적용 시점을 통보 |
| **FCA (Functional Configuration Audit)** | 기능 형상 감사 — 제품이 사양의 기능 요구사항을 충족하는지 검증 |
| **PCA (Physical Configuration Audit)** | 물리 형상 감사 — 제품이 도면/BOM과 물리적으로 일치하는지 검증 |
| **Effectivity (유효성)** | 변경이 적용되는 시리얼 번호, 로트 번호, 또는 시점 |

## 5. CI(형상항목) 식별 방법

### 5.1 형상항목 선정 기준

다음 중 하나 이상에 해당하는 항목을 CI로 지정한다:

| 기준 | 설명 | 예시 |
|------|------|------|
| 안전 관련 | 제품 안전에 영향을 미치는 항목 | 안전 장치, 구조 부품 |
| 기능 핵심 | 핵심 기능 수행에 필수적인 항목 | 주요 조립체, 핵심 부품 |
| 인터페이스 | 다른 시스템/부품과의 접점 | 커넥터, 장착 브래킷 |
| 고객 지정 | 고객이 형상관리를 요구한 항목 | 고객 도면 지정 부품 |
| 규제 관련 | 인증/규제 적합성에 관련된 항목 | 인증 대상 부품 |

### 5.2 식별자 부여 규칙

형상항목 식별자(CI Identifier) 체계:

```
CI-[제품군]-[순번]-[개정]
예: CI-MNC-0001-A
```

| 구성 요소 | 규칙 |
|-----------|------|
| CI | 형상항목 접두사 (고정) |
| 제품군 | 제품군 코드 (MNC, AER 등) — 3자리 영문 |
| 순번 | 4자리 일련번호 (0001~9999) |
| 개정 | 영문 대문자 순서 (A, B, C, ...) |

### 5.3 도면/BOM/사양서 기준선 설정

1. CI로 지정된 각 항목에 대해 관련 문서를 목록화한다:
   - 도면 (Drawing Number + Revision)
   - BOM (Bill of Materials + Revision)
   - 재료 사양서 (Material Specification)
   - 공정 사양서 (Process Specification)
2. 각 문서의 현재 유효 개정 번호를 **형상항목 등록 대장(CI Register)**에 기록한다.
3. 등록 대장은 기술팀이 관리하며, 변경 시 CCB 승인 후 업데이트한다.

## 6. 형상 기준선(Baseline) 수립 단계

### 6.1 기준선 유형

| 유형 | 시점 | 내용 | 승인자 |
|------|------|------|--------|
| **기능 기준선 (Functional Baseline)** | 설계 개념 확정 시 | 기능 요구사항, 성능 사양 | 고객/경영진 |
| **개발 기준선 (Allocated Baseline)** | 상세 설계 완료 시 | 상세 도면, 부품 목록, 시험 사양 | CCB |
| **제품 기준선 (Product Baseline)** | 초도품 검증 완료 시 | 양산 도면, BOM, 공정 문서, FAI 완료 | CCB + 고객 |

### 6.2 기준선 수립 절차

1. **기준선 대상 문서 확정**: CI 등록 대장 기반으로 기준선에 포함될 문서 목록을 작성한다.
2. **문서 상태 확인**: 모든 대상 문서가 최신 승인판인지 확인한다.
3. **기준선 패키지 작성**: 문서 목록 + 각 문서의 개정 번호 + 승인 일자를 기준선 기록지에 정리한다.
4. **CCB 승인**: CCB 회의에서 기준선을 심의하고 승인한다.
5. **기준선 공지**: 승인된 기준선을 관련 부서에 통보한다.
6. **DMS 등록**: 기준선 기록을 DMS에 등록하고, 해당 문서들을 "기준선 확정" 상태로 전환한다.

> **주의**: 기준선 수립 이후의 모든 변경은 반드시 ECR → CCB 승인 → ECN 절차를 거쳐야 한다.

## 7. ECR/ECN 작성 방법

### 7.1 ECR (Engineering Change Request) 작성

ECR은 변경이 필요하다고 판단될 때 누구나 발행할 수 있다.

**ECR 양식(QF-803-01) 기입 요령:**

| 필드 | 기입 방법 |
|------|-----------|
| ECR 번호 | ECR-[연도]-[순번] (예: ECR-2026-012) |
| 요청일 | ECR 작성일 |
| 요청자 | 성명, 부서, 연락처 |
| 대상 CI | 변경 대상 형상항목 식별자 및 명칭 |
| 변경 사유 | 변경이 필요한 이유를 구체적으로 기술 |
| 현재 상태 | 변경 전 현재 형상(도면 번호/Rev 등) |
| 요청 변경 | 변경하고자 하는 내용을 구체적으로 기술 |
| 긴급도 | 긴급(Emergency) / 일반(Normal) |

### 7.2 영향 분석 체크리스트

ECR에는 아래 영향 분석을 첨부한다:

| 항목 | 확인 내용 | 영향 유무 |
|------|-----------|-----------|
| 형상/치수 | 도면 치수 또는 공차 변경 여부 | Y / N |
| 재료 | 원자재 규격 또는 등급 변경 여부 | Y / N |
| 공정 | 제조 공정 추가/삭제/변경 여부 | Y / N |
| 호환성 | 기존 제품과의 호환성 영향 여부 | Y / N |
| 인터페이스 | 타 부품/시스템과의 접합부 영향 여부 | Y / N |
| 시험/검사 | 시험 규격 또는 검사 기준 변경 여부 | Y / N |
| 비용 | 제조 원가, 자재비 변동 여부 | Y / N |
| 납기 | 생산 일정 영향 여부 | Y / N |
| FAI | FAI 재수행 필요 여부 | Y / N |
| 고객 승인 | 고객 사전 승인 필요 여부 | Y / N |
| 인증 | 인증/적합성에 미치는 영향 여부 | Y / N |
| 재고 | 기존 재고품/재공품 처리 방안 | Y / N |

### 7.3 ECN (Engineering Change Notice) 작성

CCB에서 ECR이 승인되면 기술팀이 ECN을 발행한다.

**ECN 양식(QF-803-02) 기입 요령:**

| 필드 | 기입 방법 |
|------|-----------|
| ECN 번호 | ECN-[연도]-[순번] (예: ECN-2026-012) |
| 관련 ECR | 원 ECR 번호 참조 |
| 발행일 | ECN 발행일 |
| 적용 시점 (Effectivity) | 적용 시작 시리얼/로트/날짜 명시 |
| 변경 전 (Was) | 변경 전 형상을 도면/BOM 기준으로 명시 |
| 변경 후 (Is) | 변경 후 형상을 도면/BOM 기준으로 명시 |
| 영향 문서 목록 | 개정이 필요한 도면, BOM, 사양서, 공정 문서 목록 |
| 재고품 처리 | 기존 재고/재공품의 처리 방안 (폐기/재작업/현상 사용) |
| FAI 요구 | FAI 재수행 여부 (Full/Partial/불요) |
| 배포 부서 | ECN 통보 대상 부서 목록 |

### 7.4 ECN 후속 조치

1. 관련 도면/BOM을 개정하고 개정 이력을 기록한다.
2. 기준선 기록을 업데이트한다.
3. CI 등록 대장의 개정 번호를 갱신한다.
4. ERP의 BOM/라우팅을 변경한다.
5. 필요 시 FAI를 재수행한다 (WI-802-01 참조).
6. 변경 내용을 관련 부서 및 외주처에 통보한다.

## 8. CCB 회의 준비 및 운영

### 8.1 CCB 구성

| 역할 | 직위 | 책임 |
|------|------|------|
| 의장 | 기술팀장 | 회의 주재, 최종 결정 |
| 품질 대표 | 품질팀장 | 품질 영향 평가, FAI 판단 |
| 생산 대표 | 생산팀장 | 공정/일정 영향 평가 |
| 구매 대표 | 구매 담당 | 자재/외주 영향 평가 |
| 기록자 | 기술팀 담당 | 회의록 작성 |

### 8.2 회의 준비

1. ECR 접수 후 **5영업일 이내** CCB 회의를 소집한다 (긴급 ECR은 24시간 이내).
2. 회의 자료를 사전 배포한다:
   - ECR 원본 + 영향 분석 체크리스트
   - 관련 도면/BOM (변경 전)
   - 비용/일정 영향 분석 자료
3. 참석자에게 최소 **2영업일 전** 자료를 배포한다.

### 8.3 회의 운영

1. 의장이 ECR 내용을 설명한다.
2. 각 부서 대표가 영향 분석 결과를 발표한다.
3. 변경의 적절성, 대안, 리스크를 토론한다.
4. 결정 사항을 기록한다:
   - **승인**: ECN 발행 지시 (담당자, 기한 지정)
   - **반려**: 반려 사유 기록, 요청자에게 통보
   - **보류**: 추가 검토 사항 지정, 재심의 일정 수립
5. 회의록(QF-803-03)을 작성하고 참석자 서명을 받는다.

### 8.4 긴급 변경 처리

안전 또는 생산 중단에 관련된 긴급 변경의 경우:

1. 의장과 품질 대표의 **구두 승인**으로 즉시 변경을 시행할 수 있다.
2. 변경 시행 후 **48시간 이내** 정식 CCB를 소집하여 사후 승인한다.
3. 긴급 ECN에는 "EMERGENCY"를 표기한다.

## 9. 형상 감사(FCA/PCA) 수행 절차

### 9.1 FCA (Functional Configuration Audit)

**목적**: 제품이 기능 사양의 모든 요구사항을 충족하는지 검증

**수행 절차:**

1. 기능 요구사항 목록을 기준선에서 추출한다.
2. 각 요구사항에 대응하는 시험/검증 결과를 수집한다.
3. 검증 매트릭스를 작성한다:

| 요구사항 번호 | 요구사항 내용 | 검증 방법 | 검증 결과 문서 | 적합 여부 |
|--------------|-------------|-----------|--------------|-----------|
| REQ-001 | (기능 요구사항 내용) | 시험/분석/검사 | (성적서 번호) | 적합/부적합 |

4. 모든 요구사항이 "적합"인 경우 FCA 합격으로 판정한다.
5. 부적합 항목이 있으면 시정조치 후 재검증한다.

### 9.2 PCA (Physical Configuration Audit)

**목적**: 실물 제품이 승인된 도면/BOM과 물리적으로 일치하는지 검증

**수행 절차:**

1. 제품 기준선의 도면/BOM 최신판을 준비한다.
2. 실물 제품과 도면을 대조한다:
   - 부품 구성이 BOM과 일치하는가?
   - 치수가 도면과 일치하는가? (FAI 결과 활용)
   - 재질이 사양과 일치하는가? (자재 증명서 확인)
   - 마킹/라벨이 도면 요구와 일치하는가?
3. 대조 결과를 PCA 체크리스트에 기록한다.
4. 모든 항목이 일치하면 PCA 합격으로 판정한다.
5. 불일치 항목은 NCR을 발행하고 시정조치한다.

### 9.3 형상 감사 시기

| 시기 | 감사 유형 | 비고 |
|------|-----------|------|
| 초도품 생산 완료 시 | FCA + PCA | 제품 기준선 수립 전 필수 |
| 주요 설계 변경 후 | FCA 또는 PCA | 변경 범위에 따라 결정 |
| 고객 요청 시 | FCA 및/또는 PCA | 고객 요구에 따라 수행 |
| 정기 감사 | PCA | 연 1회 이상 권장 |

## 10. ERP 형상 데이터 관리 방법

### 10.1 BOM 관리

1. ECN이 발행되면 ERP의 BOM을 해당 Effectivity에 맞게 변경한다.
2. BOM 변경 시 아래 항목을 확인한다:
   - 부품 번호(Part Number) 정확성
   - 수량(Quantity) 정확성
   - 개정(Revision) 번호 일치
   - 유효 일자(Effectivity Date) 설정
3. 변경 전/후 BOM을 출력하여 ECN에 첨부한다.

### 10.2 도면 관리

1. 개정된 도면을 DMS에 등록한다.
2. 이전 개정판은 "구판(Obsolete)" 상태로 전환한다 (삭제하지 않는다).
3. 도면 배포 대장에 최신 개정 번호를 업데이트한다.

### 10.3 추적성 유지

- 모든 형상 변경 이력은 CI 등록 대장에 누적 기록한다.
- ECR → CCB 결정 → ECN → 도면/BOM 개정의 연결 고리를 유지한다.
- ERP의 변경 이력(Change Log)을 활용하여 추적한다.

## 11. 주의사항

1. **기준선 없이 변경 금지**: 기준선이 수립된 이후에는 어떠한 형상 변경도 ECR/CCB/ECN 절차 없이 수행할 수 없다. 비공식 변경("빨간 펜 수정" 등)은 절대 불가하다.
2. **고객 승인 필수 항목**: 고객 도면 기반 제품의 형상 변경은 반드시 고객 사전 승인을 득해야 한다.
3. **Effectivity 명확화**: ECN에 적용 시점을 모호하게 기술하면 안 된다. "즉시"가 아닌, 구체적 시리얼/로트/날짜를 명시한다.
4. **구판 관리**: 작업 현장에 구판 도면이 남아있지 않도록 회수 관리를 철저히 한다.
5. **기록 보존**: 모든 ECR, ECN, CCB 회의록은 제품 수명주기 동안 보존한다.
6. **교차 참조**: 형상 변경이 FAI, 특수공정, 외주 관리 등 타 절차에 미치는 영향을 반드시 확인한다.
