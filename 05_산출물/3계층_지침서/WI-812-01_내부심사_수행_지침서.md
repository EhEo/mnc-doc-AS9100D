# 1. 목적

본 지침서는 QP-903 (내부심사 관리 절차서)의 하위 문서로서, M&C Electronics VINA의 내부심사원이 내부심사를 계획, 준비, 수행, 보고, 추적하는 과정에서 필요한 구체적인 실무 지침을 제공한다.

본 지침서의 주요 목적:
- 내부심사의 일관되고 객관적인 수행 방법 표준화
- AS9100D 9.2 요구사항에 부합하는 내부심사 역량 확보
- ISO 19011에 근거한 심사 원칙 및 기법 적용
- 심사 결과의 효과적인 보고 및 시정조치 추적 체계 수립

---

# 2. 적용 범위

본 지침서는 M&C Electronics VINA의 품질경영시스템(QMS) 전 범위에 대한 내부심사에 적용한다.

| 구분 | 적용 대상 |
|------|----------|
| 시스템 심사 | QMS 전 조항 (AS9100D 4~10) 적합성 심사 |
| 프로세스 심사 | 개별 프로세스의 효과성 심사 |
| 제품 심사 | 제품/서비스의 요구사항 적합성 심사 |
| 특별 심사 | 주요 변경, 고객 불만, 반복 부적합 발생 시 |

**심사 주기**: 연 1회 이상 전 조항 심사 (QP-903에서 정한 연간 심사 계획에 따름)

# 4. 용어 및 정의

| 용어 | 정의 |
|------|------|
| 내부심사 (Internal Audit) | 조직 자체에 의해 또는 대리인에 의해 수행되는 체계적이고 독립적인 심사 |
| 심사원 (Auditor) | 심사를 수행하는 자격을 갖춘 인원 |
| 수석심사원 (Lead Auditor) | 심사팀을 이끌고 심사를 총괄 관리하는 심사원 |
| 피심사부서 (Auditee) | 심사를 받는 부서 또는 프로세스 책임자 |
| 심사 기준 (Audit Criteria) | 심사의 적합성 판단 기준이 되는 규격, 절차서, 법규 등 |
| 심사 증적 (Audit Evidence) | 심사 기준과 관련되고 검증 가능한 기록, 사실 진술, 관찰 사항 |
| 심사 발견사항 (Audit Finding) | 심사 증적을 심사 기준에 대비하여 평가한 결과 |
| OFI (Opportunity for Improvement) | 개선 기회, 부적합은 아니나 개선이 권고되는 사항 |
| 객관적 증거 (Objective Evidence) | 어떤 것의 존재 또는 진실을 뒷받침하는 데이터 |

# 5. 내부심사원 자격 요건

## 5.1 자격 기준

| 구분 | 수석심사원 | 심사원 |
|------|----------|--------|
| 학력 | 전문대 이상 또는 동등 경력 | 고졸 이상 또는 동등 경력 |
| 심사 교육 | AS9100D 내부심사원 교육 이수 (16시간 이상) | AS9100D 내부심사원 교육 이수 (16시간 이상) |
| 심사 경험 | 내부심사 3회 이상 참여 | 수석심사원 지도 하 1회 이상 참여 |
| 근무 경력 | 당사 1년 이상 또는 품질 관련 3년 이상 | 당사 6개월 이상 |
| 자격 승인 | 품질책임자(QMR) 승인 | 품질팀장 승인 |

## 5.2 자격 유지

1. 내부심사원은 연 1회 이상 심사에 참여하여 자격을 유지한다.
2. 2년간 심사 미참여 시 자격을 정지하고, 재교육 후 자격을 갱신한다.
3. 품질팀장은 내부심사원 자격 대장을 관리하고 연 1회 자격 현황을 검토한다.
4. AS9100D 규격 개정 시 보수 교육을 실시한다.

## 5.3 독립성 원칙

- 심사원은 **자기 업무 영역을 심사하지 않는다**.
- 심사 배정 시 직접적 업무 관계, 상하 보고 관계를 배제한다.
- 독립성 확보가 어려운 경우 품질책임자(QMR)가 대안을 결정한다.

# 6. 심사 계획 수립

## 6.1 연간 심사 계획

1. 품질팀장은 매년 1월 (또는 경영검토 후)에 연간 내부심사 계획을 수립한다.
2. 연간 계획에 포함할 사항:
   - 심사 대상 프로세스/부서
   - AS9100D 조항 배분 (전 조항 1회 이상 심사 보장)
   - 예정 심사 일정 (월별)
   - 예정 심사팀 구성
3. 다음 사항을 고려하여 심사 빈도/깊이를 조정한다:
   - 이전 심사 결과 (부적합 다발 영역은 심사 강화)
   - 고객 불만 이력
   - 프로세스 변경 사항
   - 해당 프로세스의 중요도 및 리스크 수준

## 6.2 개별 심사 계획

1. 수석심사원은 해당 심사 시작 2주 전까지 개별 심사 계획을 수립한다.
2. 개별 심사 계획에 포함할 사항:

| 항목 | 내용 |
|------|------|
| 심사 목적 | 적합성/효과성 확인, 특별 심사 사유 등 |
| 심사 범위 | 대상 프로세스, AS9100D 조항, 부서 |
| 심사 기준 | AS9100D, 해당 절차서, 고객 요구사항, 법규 |
| 심사 방법 | 문서 검토, 현장 관찰, 인터뷰, 기록 확인 |
| 심사 일정 | 일자, 시간, 장소 |
| 심사팀 구성 | 수석심사원, 심사원 배치 |

3. 개별 심사 계획은 품질팀장 승인 후 피심사부서에 사전 통보한다.

# 7. 심사 준비

## 7.1 문서 사전 검토

수석심사원 및 심사원은 심사 전 다음 문서를 검토한다:

| 검토 문서 | 검토 포인트 |
|----------|-----------|
| AS9100D 해당 조항 | 요구사항 이해, 적합성 판단 기준 확인 |
| 해당 절차서/지침서 | 프로세스 흐름, 책임, 기록 요건 파악 |
| 이전 심사 보고서 | 이전 부적합 사항 및 시정조치 현황 확인 |
| 이전 시정조치 기록 | 시정조치 완료 여부 및 효과성 확인 |
| 고객 불만/CAR 이력 | 관련 부적합 이력 및 재발 방지 현황 |

## 7.2 체크리스트 작성

1. 수석심사원은 심사 대상 조항별 체크리스트를 작성한다.
2. 체크리스트에 포함할 사항:
   - AS9100D 조항 번호 및 요구사항 요약
   - 확인 질문 (개방형 질문 위주)
   - 확인 대상 기록/증적 목록
   - 적합/부적합/해당없음 판정란
   - 관찰 기록란
3. 체크리스트는 표준 양식(F-812-02)을 기반으로 심사 범위에 맞게 조정한다.

## 7.3 질문 목록 준비

효과적인 심사를 위해 다음 유형의 질문을 준비한다:

| 질문 유형 | 목적 | 예시 |
|----------|------|------|
| 개방형 | 프로세스 이해 확인 | "이 프로세스를 어떻게 수행하십니까?" |
| 확인형 | 구체적 적합성 확인 | "이 기록은 어디에 보관됩니까?" |
| 추적형 | 증적 추적 | "이 승인 기록을 보여주시겠습니까?" |
| 상황형 | 예외 상황 대응 확인 | "부적합이 발생하면 어떻게 처리하십니까?" |

# 8. 심사 수행

## 8.1 Opening Meeting (시작회의)

1. 수석심사원이 시작회의를 주관한다.
2. 참석자: 심사팀 전원, 피심사부서 책임자, 관련 담당자
3. 시작회의 의제:

| 순서 | 내용 | 비고 |
|------|------|------|
| 1 | 심사팀 소개 및 역할 배정 | |
| 2 | 심사 목적, 범위, 기준 설명 | 개별 심사 계획 기반 |
| 3 | 심사 일정 및 방법 안내 | 현장 방문 경로 포함 |
| 4 | 부적합 분류 기준 설명 | Major/Minor/OFI |
| 5 | 비밀유지 원칙 안내 | |
| 6 | 피심사부서 안내자 지정 확인 | |
| 7 | 질의응답 | |

4. 시작회의 참석자 명단을 기록한다.

## 8.2 현장 심사 수행

### 8.2.1 증적 확인 (Evidence Gathering)

1. 체크리스트에 따라 문서, 기록, 현장 상태를 확인한다.
2. 증적 확인 방법:

| 방법 | 적용 상황 | 주의사항 |
|------|----------|----------|
| 문서 검토 | 절차서, 기록, 보고서 확인 | 최신판 여부 확인 |
| 기록 확인 | 양식, 데이터, 승인 기록 | 완전성, 추적성 확인 |
| 현장 관찰 | 작업 현장, 보관 상태, 표시 | 절차와 현장 일치 여부 |
| 인터뷰 | 담당자 면담 | 개방형 질문 활용 |
| 샘플링 | 대량 기록 중 표본 추출 | 표본 크기 적정성 확보 |

3. 확인된 증적은 구체적으로 기록한다 (문서번호, 일자, 내용 요약).

### 8.2.2 인터뷰 수행

1. 실제 업무 수행자를 대상으로 인터뷰한다.
2. 인터뷰 원칙:
   - 편안한 분위기를 조성한다.
   - 개방형 질문으로 시작하여 세부 사항으로 진행한다.
   - 답변 내용을 기록하고, 필요 시 확인 질문을 한다.
   - 인터뷰 내용은 문서/기록으로 교차 검증한다.
3. 인터뷰 결과를 체크리스트에 기록한다.

### 8.2.3 현장 관찰

1. 실제 작업 수행 상황을 관찰한다.
2. 관찰 포인트:
   - 작업자가 절차서에 따라 작업하는지
   - 작업장 정리정돈 및 안전 상태
   - 측정장비 교정 상태 표시
   - 부적합품 격리 상태
   - 문서의 최신판 사용 여부
3. 관찰 사항을 체크리스트에 즉시 기록한다.

### 8.2.4 일일 정리

1. 심사팀은 매일 심사 종료 후 일일 정리 회의를 갖는다.
2. 일일 정리 내용:
   - 당일 심사 결과 공유 (심사원 간)
   - 잠재 부적합 사항 논의
   - 추가 확인 필요 사항 정리
   - 다음 일 심사 계획 조정
3. 일일 정리 결과를 간략히 기록한다.

# 9. 부적합 작성 방법

## 9.1 부적합 유형 분류

| 유형 | 정의 | 판정 기준 |
|------|------|----------|
| **Major** (중부적합) | 시스템의 효과적 운영에 심각한 영향을 미치는 부적합 | 요구사항 전면 미이행, 시스템적 결함, 다수 부적합 누적 |
| **Minor** (경부적합) | 시스템의 효과적 운영에 경미한 영향을 미치는 부적합 | 요구사항 일부 미이행, 개별적 일탈, 기록 누락 |
| **OFI** (개선기회) | 부적합은 아니나 개선 시 효과성이 향상될 수 있는 사항 | 모범 사례 미적용, 효율성 개선 가능, 리스크 저감 가능 |

## 9.2 부적합 보고서 작성 요소

각 부적합 사항에 대해 다음을 명확히 기술한다:

### 9.2.1 근거 조항 (Requirement)

- 불충족된 AS9100D 조항 번호를 명시한다.
- 해당 조항의 요구사항을 인용 또는 요약한다.
- 관련 절차서 조항이 있는 경우 함께 명시한다.
- 예: "AS9100D 7.5.3.1에 따라 외부 출처 문서는 식별되고 배포가 관리되어야 한다."

### 9.2.2 객관적 증거 (Objective Evidence)

- 부적합을 뒷받침하는 구체적인 사실을 기술한다.
- 다음 정보를 포함한다:
  - **무엇을** (What): 어떤 문서/기록/활동이 부적합한지
  - **어디서** (Where): 어느 부서/공정/장소에서 발견되었는지
  - **언제** (When): 해당 기록의 일자, 관찰 일시
  - **얼마나** (How much): 검토한 표본 중 부적합 건수
- 예: "생산부 CNC 가공 공정에서 2026년 4월 기록 10건 중 3건의 공정검사 기록(QF-810-03)에 검사자 서명이 누락됨."

### 9.2.3 부적합 기술 방법 (Finding Statement)

부적합 기술은 다음 구조를 따른다:

```
[근거 조항 요약] + "그러나" + [객관적 증거에 의한 부적합 사실]
```

**올바른 예시**:
> "AS9100D 7.1.5.1에 따라 측정장비는 규정된 주기에 교정되어야 한다. 그러나 품질검사실의 마이크로미터 3대(장비번호: CAL-023, CAL-024, CAL-025)가 교정 유효기간(2026-03-15)이 만료된 상태로 사용 중이었다."

**잘못된 예시**:
> "교정이 제대로 관리되지 않고 있다." (주관적, 구체적 증거 없음)

## 9.3 부적합 작성 시 주의사항

1. 사실에 기반하여 작성하고, 추측이나 의견을 포함하지 않는다.
2. 하나의 부적합 보고서에는 하나의 부적합 사항만 기술한다.
3. 원인 분석은 부적합 보고서에 포함하지 않는다 (시정조치 단계에서 수행).
4. 부적합 등급(Major/Minor)은 심사팀 합의로 결정하고, 수석심사원이 최종 확인한다.
5. 피심사부서와 부적합 내용에 대해 사실 확인(Fact Confirmation)을 실시한다.

# 11. 심사 보고서 작성

## 11.1 심사 보고서(F-812-01) 구성

| 항목 | 기재 내용 |
|------|----------|
| 심사 보고서 번호 | IA-연도-일련번호 (예: IA-2026-001) |
| 심사 일자 | 심사 수행 기간 |
| 심사 목적 | 정기/특별 심사, 심사 목적 |
| 심사 범위 | 심사 대상 프로세스, 조항, 부서 |
| 심사 기준 | AS9100D, 절차서, 법규 등 |
| 심사팀 구성 | 수석심사원, 심사원 명단 |
| 심사 결과 요약 | 부적합 건수 (Major/Minor/OFI) |
| 개별 부적합 상세 | 각 건별 근거 조항, 증거, 기술 |
| 긍정적 발견사항 | 우수 사례, 모범 관행 |
| 종합 소견 | QMS 적합성/효과성에 대한 전반적 평가 |
| 시정조치 기한 | 각 부적합 건별 시정조치 완료 기한 |

## 11.2 보고서 작성 기한 및 배포

1. 심사 보고서는 심사 종료 후 **5영업일 이내**에 작성 완료한다.
2. 보고서 검토: 수석심사원 작성 → 품질팀장 검토
3. 배포 대상:
   - 피심사부서 책임자
   - 품질책임자(QMR)
   - 경영대리인 (경영검토 입력 자료로 활용)
4. 심사 보고서 원본은 품질팀에서 보관한다 (보존 기간: 5년).

# 12. 시정조치 추적 확인

## 12.1 시정조치 기한

| 부적합 유형 | 시정조치 기한 | 연장 승인 |
|------------|-------------|----------|
| Major | 30일 이내 | 품질책임자(QMR) 승인, 최대 1회 15일 연장 |
| Minor | 60일 이내 | 품질팀장 승인, 최대 1회 30일 연장 |
| OFI | 90일 이내 (권고) | 별도 추적 (의무 아님) |

## 12.2 Follow-up (추적 확인) 절차

1. 피심사부서는 기한 내 시정조치 계획 및 완료 보고서를 품질팀에 제출한다.
2. 시정조치 보고서에 포함할 사항:
   - 원인 분석 (근본 원인 포함)
   - 시정조치 내용 (즉시 조치 + 재발 방지 조치)
   - 조치 완료 증적
   - 효과성 확인 방법 및 일정
3. 수석심사원 또는 지정된 심사원이 시정조치의 적절성을 검증한다.
4. 검증 방법:
   - 제출된 증적 검토
   - 필요 시 현장 확인 (Follow-up Audit)
   - 효과성 확인 (조치 후 일정 기간 경과 후 재발 여부 확인)
5. 시정조치가 부적절한 경우 반려하고 보완을 요구한다.
6. 시정조치 완료 시 해당 부적합 건을 "종결(Closed)"로 처리한다.

## 12.3 미완료 시정조치 관리

1. 기한 초과 시정조치는 품질팀장에게 즉시 보고한다.
2. 2회 연속 기한 초과 시 품질책임자(QMR)에게 에스컬레이션한다.
3. 미완료 시정조치 현황은 월 1회 경영진에게 보고한다.
4. 미완료 시정조치는 다음 내부심사 시 중점 확인 대상으로 지정한다.

# 13. 주의사항

## 13.1 독립성 확보

1. 심사원은 자신의 업무 영역을 심사하지 않는다.
2. 심사원과 피심사 부서 간 직접적 보고 관계가 없어야 한다.
3. 독립성 확보가 불가능한 경우 품질책임자(QMR)가 대안을 결정한다.
4. 심사원은 심사 기간 중 피심사 부서의 업무를 겸직하지 않는다.

## 13.2 비밀유지

1. 심사 과정에서 취득한 정보는 심사 목적으로만 사용한다.
2. 심사 결과는 권한이 있는 인원에게만 공유한다.
3. 개인의 실수나 책임을 공개적으로 거론하지 않는다.
4. 심사 관련 문서는 기밀 등급에 따라 관리한다.

## 13.3 심사 윤리

1. 심사원은 객관적이고 공정하게 심사한다.
2. 개인적 감정이나 편견이 심사에 영향을 미치지 않도록 한다.
3. 심사 중 발견된 부적합은 사실에 기반하여 보고한다.
4. 피심사 부서에 대해 적대적이거나 위협적인 태도를 취하지 않는다.
5. 심사 결과에 대해 피심사 부서와 열린 대화를 유지한다.
