# AS9100D 정적 매뉴얼 사이트 구축 계획

## 목표
AS9100D 인증 관련 Markdown 문서를 브라우저에서 매뉴얼 형태로 볼 수 있는 정적 사이트 생성.
- 왼쪽: 리스크/문서 네비게이션 트리
- 오른쪽: 선택된 문서 내용
- 신규 파일 추가 시 재빌드만으로 자동 반영
- 한국어 완벽 지원 (UTF-8)

## 아키텍처

### 파일 구조
```
.site/
├── build.js          # Node.js 빌드 스크립트
├── PLAN.md           # 이 파일
└── dist/
    └── index.html    # 생성된 단일 정적 사이트 (모든 리소스 임베드)
```

### 빌드 스크립트 (`build.js`)
- Node.js fs를 사용하여 `05_산출물/`, `01_마스터일정/`, `02_문서관리/`, `03_실행계획/`, `04_대시보드/` 폴더 스캔
- 각 Markdown 파일을 읽어 JSON 데이터로 변환
- 파일 수정 시간(modified time)을 추적하여 "최근 업데이트" 자동 표시
- 단일 `index.html`에 모든 콘텐츠를 JavaScript 데이터로 임베드

### 정적 사이트 (`dist/index.html`)
**단일 파일로 CDN 기반 리소스 사용 (오프라인 불가 시 대비하여 핵심 라이브러리도 인라인)**

#### 기능
1. **트리 네비게이션**: 계층별(1계층/2계층/3계층/실행계획/대시보드) 문서 목록
2. **리스크 필터**: 고위험/중위험/저위험 필터 버튼
3. **텍스트 검색**: 문서명 + 내용 검색
4. **최근 수정**: 파일 수정 시간 기준 정렬 (상위 10건 자동 표시)
5. **마크다운 렌더링**: marked.js (CDN)로 마크다운 → HTML 변환
6. **코드 하이라이팅**: highlight.js (CDN) 코드 블록 지원
7. **한국어 지원**: UTF-8 charset, Pretendard 웹폰트 (CDN), 시스템 폴백 폰트
8. **반응형**: 데스크톱/태블릿 대응
9. **URL 해시**: `#doc=문서번호`로 직접 접근 가능

#### 기술 스택
- **마크다운 렌더링**: marked.js (CDN)
- **코드 하이라이트**: highlight.js (CDN)
- **스타일**: CSS-in-HTML (인라인)
- **폰트**: Pretendard (CDN) + Apple SD Gothic Neo, Malgun Gothic 폴백
- **빌드**: Node.js (v22, fs 모듈만 사용, 외부 패키지 불필요)

## 실행 계획

### Step 1: `build.js` 빌드 스크립트 작성
- Markdown 파일 스캔 (glob 패턴: `**/*.md`, `.site/` 제외)
- 각 파일에서 문서번호, 문서명, 계층, 위험등급 추출
- 파일 수정 시간 기록
- JSON 데이터 생성

### Step 2: `index.html` 템플릿 작성
- 사이드바 + 메인 레이아웃
- CSS 스타일링 (다크/라이트 모드 지원)
- JavaScript 로직: 네비게이션, 검색, 필터, 마크다운 렌더링
- UTF-8 charset 및 한국어 폰트 설정

### Step 3: 빌드 및 테스트
- `node .site/build.js` 실행
- 생성된 `.site/dist/index.html` 브라우저에서 열어 확인
- 한국어 출력, 네비게이션, 검색, 최근 수정 표시 동작 확인

### Step 4: 사용자 확인
- 브라우저에서 사이트 확인
- 필요 시 수정

## GitHub Pages 배포

GitHub Pages에서 `/docs` 폴더를 통해 서비스 가능합니다.

### 수동 설정
1. GitHub 저장소 Settings → Pages
2. Source: Deploy from a branch
3. Branch: `main`, Folder: `/docs`
4. Save

### 자동 배포 (GitHub Actions)
`.github/workflows/deploy-pages.yml` 워크플로우가 자동으로 배포합니다.

## 사용법
```bash
# 빌드
node .site/build.js

# 결과물 열기
start .site/dist/index.html
```

새 문서 추가 시 `node .site/build.js` 재실행만으로 자동 반영됩니다.
